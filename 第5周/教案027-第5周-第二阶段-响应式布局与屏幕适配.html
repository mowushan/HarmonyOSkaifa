🧩 知识讲解 · 让UI自动适应屏幕

1. 为什么要响应式布局？—— 衣服有尺码，屏幕也有

📱+📱+🖥️ 现实场景： 同一个App，在手机上可能是单列显示，在平板上可以是双列，在折叠屏展开时可以是三列。如果布局写死，在平板上就会显得空荡荡或挤成一团。

响应式布局核心思想：

  • 🌊 流动布局：元素宽度用百分比，随父容器变化
  • 📐 弹性网格：栅格系统自动调整列数
  • 🎯 断点切换：屏幕宽度超过某个值，布局自动“变身”

📣 鸿蒙ArkUI提供了强大的响应式能力，今天我们就来实战！

2. 鸿蒙响应式三件套：Grid + Breakpoint + 百分比

📐 Grid栅格布局

Grid() + GridItem()，设置 columnsTemplate 动态控制列数。
例如：columnsTemplate: '1fr 1fr' 表示两列等分。

🎯 媒体查询 @ohos.mediaquery

根据屏幕宽度动态改变状态变量，实现布局切换。
matchMediaSync('(min-width: 600vp)')

📏 相对单位 vp / %

vp (视口比例) 让字体和间距在不同设备上看起来大小合适。
width: '100%' 充满父容器。

3. 屏幕断点参考（华为推荐）

类型宽度范围典型设备建议列数
xs（超小）0 - 320vp小屏手机1列
sm（小）321 - 600vp普通手机1列或2列
md（中）601 - 840vp折叠屏展开、小平板2列
lg（大）841 - 1200vp平板横屏3列
xl（超大）>1200vp智慧屏、大屏4列以上

⚙️ 技能实训 · 手把手实现响应式资产列表

✅ 实训目标：将资产列表改造为手机1列、平板2列、大屏3列的自适应网格

📁 实训准备：新建响应式页面

1. 打开DevEco Studio，在 entry/src/main/ets/pages 目录下新建文件 ResponsiveAssetPage.ets

2. 先搭建基础结构，导入必要的模块。

🔧 实训一：创建基础数据模型（5分钟）

  • 定义资产数据接口 AssetItem
  • 创建模拟数据数组，包含图片、名称、数量、价格等信息
  • 在页面中声明 @State assets: AssetItem[] 存储数据
    • // 资产数据模型 interface AssetItem { id: number; name: string; quantity: number; price: number; icon: string; } // 模拟数据 const mockAssets: AssetItem[] = [ { id: 1, name: "笔记本电脑", quantity: 15, price: 5999, icon: "💻" }, { id: 2, name: "投影仪", quantity: 8, price: 2899, icon: "📽️" }, { id: 3, name: "实训开发板", quantity: 30, price: 399, icon: "🔌" }, { id: 4, name: "无线麦克风", quantity: 12, price: 599, icon: "🎤" }, { id: 5, name: "智慧屏", quantity: 5, price: 4999, icon: "📺" }, { id: 6, name: "平板电脑", quantity: 20, price: 2699, icon: "📱" } ];

    🔧 实训二：媒体查询获取屏幕宽度（15分钟）

  • 导入媒体查询模块：import mediaquery from '@ohos.mediaquery';
  • 创建状态变量 @State currentColumns: number = 1; 控制列数
  • 在 aboutToAppear() 中监听屏幕尺寸变化，动态修改列数
  • 根据宽度范围设置：宽度<600vp为1列，600-840为2列，≥840为3列
    • import mediaquery from '@ohos.mediaquery'; @Entry @Component struct ResponsiveAssetPage { @State assets: AssetItem[] = mockAssets; @State currentColumns: number = 1; private listener: mediaquery.MediaQueryListener; aboutToAppear() { // 创建媒体查询监听器 let media = mediaquery.matchMediaSync('(min-width: 0vp)'); this.updateColumns(media); media.on('change', (media: mediaquery.MediaQueryResult) => { this.updateColumns(media); }); this.listener = media; } updateColumns(media: mediaquery.MediaQueryResult) { let width = media.width; if (width < 600) { this.currentColumns = 1; } else if (width < 840) { this.currentColumns = 2; } else { this.currentColumns = 3; } console.info(`当前列数: ${this.currentColumns}, 宽度: ${width}`); } aboutToDisappear() { if (this.listener) { this.listener.off('change'); } } build() { Column() { // 标题栏 Text('响应式资产列表') .fontSize('24vp') .fontWeight(FontWeight.Bold) .width('100%') .padding({ left: '16vp', top: '16vp', bottom: '8vp' }) // TODO: 网格布局 } .width('100%') .height('100%') .backgroundColor('#F5F5F5') } }

    🔧 实训三：Grid网格布局 + 动态列数（20分钟）

  • 将 build() 中的占位替换为 Grid() 组件
  • 设置 columnsTemplate: `repeat(${this.currentColumns}, 1fr)`
  • 使用 ForEach 遍历资产数组，每个资产用 GridItem 包裹
  • 设计资产卡片样式：圆角、阴影、内边距，展示图标、名称、数量、价格
    • build() { Column() { Text('响应式资产列表') .fontSize('24vp') .fontWeight(FontWeight.Bold) .width('100%') .padding({ left: '16vp', top: '16vp', bottom: '8vp' }) // 显示当前列数提示（调试用） Text(`当前列数: ${this.currentColumns}`) .fontSize('12vp') .fontColor('#999') .width('100%') .padding({ left: '16vp', bottom: '8vp' }) Grid() { ForEach(this.assets, (item: AssetItem) => { GridItem() { this.AssetCard(item) } }, (item: AssetItem) => item.id.toString()) } .columnsTemplate(`repeat(${this.currentColumns}, 1fr)`) .columnsGap('12vp') .rowsGap('12vp') .padding({ left: '12vp', right: '12vp', top: '8vp' }) .width('100%') .layoutWeight(1) } .width('100%') .height('100%') .backgroundColor('#F5F5F5') } // 资产卡片组件 @Builder AssetCard(item: AssetItem) { Column() { Row() { Text(item.icon) .fontSize('32vp') Text(item.name) .fontSize('18vp') .fontWeight(FontWeight.Medium) .margin({ left: '8vp' }) } .width('100%') Divider().margin({ top: '8vp', bottom: '8vp' }) Row() { Text(`数量: ${item.quantity}`) .fontSize('14vp') .fontColor('#666') Blank() Text(`¥${item.price}`) .fontSize('16vp') .fontColor('#E6A23C') .fontWeight(FontWeight.Bold) } .width('100%') } .padding('16vp') .backgroundColor(Color.White) .borderRadius('16vp') .shadow({ radius: '8vp', color: '#CCCCCC' }) .width('100%') }

    🔧 实训四：百分比布局 + 相对单位优化（10分钟）

  • 将所有固定宽度改为百分比或vp单位
  • 卡片内图片/图标大小使用vp单位，确保在不同分辨率下比例协调
  • 测试不同屏幕尺寸下的显示效果

    • 💡 老师指导： vp (virtual pixel) 是鸿蒙推荐的适配单位，在不同ppi设备上会自动缩放，比px更友好。

    🔧 实训五：折叠屏模拟测试（10分钟）

    测试步骤：
