🧩 知识讲解 · 鸿蒙弹窗家族大揭秘

1. AlertDialog · 最常用的确认对话框

❓ 场景引入： 当用户点击“删除资产”时，不能直接删掉，应该弹出一个对话框问：“确定删除吗？” 让用户确认。

AlertDialog核心属性：

  • title：标题（如“提示”）
  • message：内容（如“确定要删除这条资产吗？”）
  • primaryButton：主要按钮（确定/删除）
  • secondaryButton：次要按钮（取消）
// 显示确认删除对话框
AlertDialog.show({
    title: '删除确认',
    message: '确定要删除这条资产记录吗？',
    primaryButton: {
        value: '删除',
        action: () => {
            // 这里执行删除操作
            console.log('用户确认删除');
        }
    },
    secondaryButton: {
        value: '取消',
        action: () => {
            console.log('用户取消删除');
        }
    }
});

2. Toast · 轻提示（自动消失）

🍞 Toast就像烤面包机弹出的提示，显示几秒后自动消失，不打扰用户操作。

使用场景： “保存成功”、“网络连接失败”、“已复制到剪贴板”。

import promptAction from '@ohos.promptAction';

// 显示简单文本提示
promptAction.showToast({
    message: '资产保存成功！',
    duration: 2000  // 显示2秒
});

3. ActionSheet · 操作菜单（多选项）

📋 ActionSheet从底部弹出，让用户选择多个操作选项，比如“拍照”、“从相册选择”、“取消”。

ActionSheet.show({
    title: '选择照片来源',
    message: '请选择添加图片的方式',
    sheets: [
        { title: '拍照', action: () => { /* 拍照逻辑 */ } },
        { title: '从相册选择', action: () => { /* 相册逻辑 */ } },
        { title: '取消', action: () => { } }
    ]
});

4. 自定义弹窗 · 打造专属对话框

🎨 如果系统弹窗不够用，可以用 @CustomDialog 自己做！比如带输入框的弹窗、带图标的弹窗。

// 自定义弹窗组件
@CustomDialog
struct MyInputDialog {
    controller: CustomDialogController;
    @State inputValue: string = '';

    build() {
        Column() {
            Text('请输入备注').fontSize(18).margin({ bottom: 12 });
            TextInput({ placeholder: '备注内容', text: this.inputValue })
                .onChange((value) => { this.inputValue = value; });
            Row() {
                Button('取消').onClick(() => { this.controller.close(); });
                Button('确定').onClick(() => {
                    // 处理输入内容
                    this.controller.close();
                }).backgroundColor('#0078d4');
            }.margin({ top: 16 }).justifyContent(FlexAlign.SpaceAround);
        }.padding(20);
    }
}

⚙️ 技能实训 · 动手做一个“有礼貌”的应用

✅ 实训目标：在资产管理Demo中集成三种弹窗，实现删除确认、保存提示、操作菜单

📁 准备工作：打开之前项目

1. 打开DevEco Studio，找到我们之前的“AssetManagement”项目（或新建一个测试项目）。

2. 在资产列表页（AssetList.ets）添加一个删除按钮，在资产录入页（AssetEntry.ets）添加保存按钮。

🧪 实训任务一：实现删除确认对话框（AlertDialog）

  • 在资产列表页的每个资产项旁边，添加一个“删除”图标或按钮。
  • 绑定点击事件，弹出AlertDialog询问用户是否删除。
  • 在“确定”回调里，删除该资产并刷新列表。
  • 测试： 点击删除 → 弹出“确定删除吗？” → 点“删除”后资产消失。
    • // 删除按钮点击事件
build() {
    ListItem() {
        Row() {
            Text(asset.name).fontSize(16).layoutWeight(1);
            Button('删除').onClick(() => {
                AlertDialog.show({
                    title: '删除确认',
                    message: `确定要删除“${asset.name}”吗？`,
                    primaryButton: {
                        value: '删除',
                        action: () => {
                            // 调用删除方法
                            this.deleteAsset(asset.id);
                            promptAction.showToast({ message: '删除成功' });
                        }
                    },
                    secondaryButton: { value: '取消' }
                });
            }).backgroundColor('#ff4444');
        }.padding(12);
    }
}
