🧩 知识讲解 · 让App穿上“统一制服”

1. 为什么需要统一主题？—— 用户体验的魔法

🎨 趣味小剧场： 想象一下，你的资产录入页是蓝色的，列表页是红色的，详情页又是绿色的……用户会以为用了三个不同的App！

统一主题的好处：

  • ✅ 品牌识别：一看颜色就知道是你的作品
  • ✅ 专业感：评委觉得你用心了
  • ✅ 维护方便：改一个颜色，全局都变，不用每个页面单独改
  • ✅ 用户体验：视觉统一，用户用起来更舒服

📣 鸿蒙官方推荐使用“主题色+辅助色+强调色”的配色方案！

2. 鸿蒙资源管理：`resources` 文件夹的秘密

📁 element/color.json

存放所有颜色变量，比如 `"color_primary": "#0078D4"`，全局引用。

📁 element/string.json

存放所有文本，支持多语言，方便国际化。

📁 element/float.json

存放尺寸、间距等数值，比如 `"spacing_large": 24`。

📁 项目结构示例：
entry/
  src/
    main/
      resources/
        base/
          element/
            color.json      ← 颜色定义
            string.json     ← 文字定义
            float.json      ← 尺寸定义
          media/            ← 图片资源

4. 复用样式 · @Styles 装饰器

// 定义全局样式函数
@Styles function cardStyle() {
  .width('100%')
  .backgroundColor($r('app.color.color_background'))
  .borderRadius(16)
  .padding(16)
  .margin({ bottom: 12 })
  .shadow({ radius: 8, color: '#1F2D3A20' })
}

@Styles function titleStyle() {
  .fontSize(18)
  .fontWeight(FontWeight.Medium)
  .color($r('app.color.color_text_primary'))
  .margin({ bottom: 8 })
}

// 在组件中使用
@Entry
@Component
struct AssetCard {
  build() {
    Column() {
      Text('资产名称')
        .titleStyle()
      Text('资产详情...')
        .fontSize(14)
        .color($r('app.color.color_text_secondary'))
    }
    .cardStyle()
  }
}

⚙️ 技能实训 · 给资产管理系统穿上“新衣”

✅ 本次实训目标：为资产管理系统设计统一主题，并应用到已有页面！

🔧 实训一：设计你的主题色板（15分钟）

  • 打开在线配色工具：推荐 https://colorhunt.co/ 或 https://coolors.co/，选择一套你喜欢的配色（主色+辅助色+强调色）。
  • 记录颜色值： 将选好的颜色 HEX 值（如 #0078D4）记录下来，准备应用到代码中。
  • 小组讨论： 资产管理App应该用什么颜色？蓝色代表科技？绿色代表资产？每组选一套方案并上台分享。

    • 🎨 推荐鸿蒙官方配色方案：

    主色 #0078D4（科技蓝）

    强调色 #FFB900（活力橙）

    成功色 #28A745（成功绿）

    错误色 #DC3545（警示红）

    🔧 实训二：创建颜色资源文件（20分钟）

  • 在 DevEco Studio 中打开项目，找到 `entry/src/main/resources/base/element/` 目录。
  • 创建或编辑 color.json，将实训一选好的颜色填入。
  • 同步创建 float.json（用于尺寸）和 string.json（用于文字）。
    • // 示例：float.json 定义间距
{
  "float": [
    { "name": "spacing_small", "value": "8vp" },
    { "name": "spacing_medium", "value": "16vp" },
    { "name": "spacing_large", "value": "24vp" },
    { "name": "radius_card", "value": "16vp" }
  ]
}

    🔧 实训三：定义全局样式函数（20分钟）

  • 在 `pages` 目录下新建一个文件 `styles.ets`（或在现有组件文件顶部定义）。
  • 使用 `@Styles` 装饰器定义卡片样式、标题样式、按钮样式等。
  • 在资产卡片组件中应用这些样式。
    • // styles.ets
@Styles export function cardStyle() {
  .width('100%')
  .backgroundColor($r('app.color.color_background'))
  .borderRadius($r('app.float.radius_card'))
  .padding($r('app.float.spacing_medium'))
  .margin({ bottom: $r('app.float.spacing_small') })
  .shadow({ radius: 8, color: '#00000020' })
}

@Styles export function primaryButtonStyle() {
  .backgroundColor($r('app.color.color_primary'))
  .borderRadius(24)
  .padding({ left: 24, right: 24, top: 12, bottom: 12 })
  .fontColor(Color.White)
  .fontSize(14)
}

    🔧 实训四：改造已有页面（20分钟）

  • 打开资产列表页，将原来的样式替换为刚才定义的样式函数。
  • 改造资产录入页，统一按钮颜色、输入框边框颜色等。
  • 改造资产详情页，让所有卡片风格一致。
  • 运行预览，看看所有页面是否风格统一！
    • // 改造后的资产卡片组件示例
import { cardStyle, titleStyle, primaryButtonStyle } from '../styles'

@Component
export struct AssetCard {
  @Prop assetName: string
  @Prop assetValue: string

  build() {
    Column() {
      Text(this.assetName)
        .titleStyle()
      Text(`价值：${this.assetValue}`)
        .fontSize(14)
        .color($r('app.color.color_text_secondary'))
        .margin({ top: 4 })
      Row() {
        Button('查看详情')
          .primaryButtonStyle()
          .onClick(() => {
            // 跳转逻辑
          })
      }
      .width('100%')
      .justifyContent(FlexAlign.End)
      .margin({ top: 12 })
    }
    .cardStyle()
  }
}

    🔧 实训五：深色模式适配（挑战任务 · 10分钟）

    🌙 挑战： 鸿蒙支持深色模式，只需在 `resources/dark/element/color.json` 中定义深色下的颜色值，系统会自动切换！

    步骤： 在 resources 下创建 `dark` 文件夹，复制 color.json 进去，将颜色调暗，运行试试切换系统深色模式。
