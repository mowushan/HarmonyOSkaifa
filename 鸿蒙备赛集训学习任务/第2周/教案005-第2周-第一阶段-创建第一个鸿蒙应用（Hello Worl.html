🧩 知识讲解 · 鸿蒙应用的“诞生记”

1. 鸿蒙应用长什么样？—— 项目结构大揭秘

🏗️ 趣味比喻： 一个鸿蒙项目就像一个“智能房屋”：

  • entry/ → 房子的大门（应用入口模块）
  • ets/ → 房间里的家具（代码文件）
  • resources/ → 墙纸和装饰（图片、文字资源）
  • module.json5 → 房屋说明书（模块配置文件）

📌 最重要文件： entry/src/main/ets/pages/Index.ets —— 这就是我们看到的第一个页面！

2. 第一个页面代码解读 —— 让屏幕显示文字

            // Index.ets - 这是我们的第一个鸿蒙页面
            @Entry      // 标记这是应用的入口页面
            @Component   // 标记这是一个自定义组件
            struct Index {
              build() {
                Column() {           // 垂直布局容器
                  Text('Hello World')   // 显示文字的组件
                    .fontSize(50)               // 设置字体大小
                    .fontWeight(FontWeight.Bold) // 设置粗体
                    .fontColor('#0078d4')       // 设置颜色
                }
                .width('100%')        // 宽度占满屏幕
                .height('100%')       // 高度占满屏幕
                .justifyContent(FlexAlign.Center) // 内容居中
              }
            }

🔍 逐行解读（老师边指边讲）：

  • @Entry → 告诉系统：“嘿，这个页面是应用的入口！”
  • @Component → 声明这是一个可复用的UI组件
  • build() → 描述这个页面长什么样子的函数
  • Column → 把东西竖着排（就像排队一样）
  • Text() → 显示文字，括号里写什么就显示什么
  • .fontSize(50) → 点语法给文字添加样式

✨ 小实验： 如果把 'Hello World' 改成 '你好，鸿蒙！' 会怎样？—— 等下我们就动手试试！

3. 模拟器 vs 真机 —— 在哪里看结果？

📱 本地模拟器

电脑上虚拟一个手机，不用真机也能运行。适合初学调试，启动稍慢但方便。

📲 真机调试

用自己的鸿蒙手机（或平板）直接运行，体验最真实。需要开启开发者模式。

👀 预览器

最快速的方式！右侧Previewer窗口，代码改完立刻看到效果，不用编译。

⚙️ 技能实训 · 手把手带你跑通Hello World

✅ 本次实训目标：每人成功运行第一个鸿蒙应用，并学会修改界面文字

🔧 实训准备：确认DevEco Studio已安装

  • 双击桌面DevEco Studio图标，打开软件。
  • 如果提示“SDK未安装”，按照向导完成SDK下载（约2GB，可课下提前完成）。
  • 确认左下角或欢迎页面显示“No Project”，准备创建新项目。

    • ⚠️ 如果SDK还没下载完，今天先使用“预览器”功能，也能看到效果。课后记得挂机下载。

    🧪 实训任务一：创建第一个鸿蒙项目（15分钟）

  • 点击“Create Project”： 在欢迎页面选择“Create Project”。
  • 选择模板： 选择“Empty Ability”（空模板），这是最基础的Hello World模板。
  • 配置项目：
    • Project name: MyFirstHarmonyApp
    • Package name: com.yourname.myfirstapp（可以换成自己的名字拼音）
    • Save location: 选择D:\HarmonyProjects（自己建一个文件夹）
    • Compatible API version: 选择最新的稳定版（如API 9或更高）
  • 点击“Finish”，等待项目构建完成（第一次会下载依赖，稍等1-2分钟）。

    • 🎯 检查点： 左侧项目目录中能看到 entry/src/main/ets/pages/Index.ets 文件，双击打开它，看到代码！

    🧪 实训任务二：使用预览器查看效果（10分钟）

  • 在打开 Index.ets 文件后，看右侧有没有“Previewer”窗口。
  • 如果没有，点击菜单栏：View → Tool Windows → Previewer。
  • 预览器里应该显示一个白色屏幕，中间有“Hello World”文字。
  • 互动挑战： 把代码中的 'Hello World' 改成 '你好，鸿蒙！我是XXX'（XXX换成自己名字），观察预览器里的文字是否立刻变化？

    • 🎉 恭喜！你已经体验了“实时预览”的魔力，这就是声明式UI的魅力——代码即界面！

    🧪 实训任务三：运行到模拟器/真机（15分钟）

  • 创建模拟器（如果没有）： 点击菜单 Tools → Device Manager → 点击“+”号 → 选择手机型号（如Pixel 6）→ 下载镜像 → 创建完成。
  • 启动模拟器： 在Device Manager中点播放按钮启动模拟器（第一次启动较慢，约2分钟）。
  • 运行项目： 点击工具栏绿色的三角形“Run”按钮（或按Shift+F10）。
  • 选择设备： 在弹出的窗口中选择刚才启动的模拟器，点击OK。
  • 等待编译安装： 下方Build窗口出现“Install successfully”后，模拟器屏幕上就会显示你的应用！

    • 📱 真机调试小贴士： 如果有鸿蒙手机，用USB线连接电脑，手机开启“开发者模式”和“USB调试”，点击运行后选择真机即可。老师会巡回指导。

    🧪 实训任务四：代码改造大闯关（15分钟）

    🎯 闯关1：改文字

    把文字改成“鸿蒙开发，我来了！” 并加一行副标题（复制一个Text组件）。

    🎯 闯关2：改颜色

    把字体颜色改成红色（.fontColor('#ff0000')）或者自己喜欢的颜色。

    🎯 闯关3：加背景

    给Column加个背景色：.backgroundColor('#f0f0f0')，看看效果。

    🏆 终极挑战

    显示一张图片！在resources/base/media里放一张图片，用Image组件显示出来。

                // 终极挑战参考代码
            Column() {
              Text('鸿蒙开发，我来了！')
                .fontSize(30)
                .fontColor('#ff6b6b')
              Text('第二行：这是我的第一个应用')
                .fontSize(20)
              Image('$r("app.media.icon")')  // 显示应用图标
                .width(100)
                .height(100)
            }
            .width('100%')
            .height('100%')
            .justifyContent(FlexAlign.Center)

    🔧 常见错误 · 别怕，我们帮你搞定

    ❌ 模拟器启动失败

    解决：检查电脑虚拟化是否开启（BIOS设置），或重启DevEco Studio。

    ❌ 编译报错“SDK not found”

    解决：菜单 File → Settings → SDK Manager，下载对应API版本的SDK。

    ❌ 预览器不显示

    解决：点击右上角刷新按钮，或关闭重新打开Index.ets文件。

    ❌ 真机连接不上

    解决：换一根数据线，检查USB调试是否开启，安装华为手机助手驱动。

    🤝 互助原则： 遇到报错先看红色提示，看不懂就截图发群，老师或同学帮忙。鸿蒙社区也很活跃，要学会搜索错误信息！
