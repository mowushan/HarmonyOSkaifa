🧩 知识讲解 · 面向对象の魔法

1. 类（Class）—— 制造对象的“饼干模具”

🥨 现实类比： 你想做很多个“学生信息卡”。如果每次都手写姓名、年龄、班级，太累了！不如先设计一个“学生卡模板”，里面预留姓名、年龄、班级的位置，然后批量印刷。这个模板就是类，印刷出来的每张卡就是对象。

// ArkTS 定义一个简单的“资产”类
class Asset {
    // 属性（特征）
    name: string
    price: number
    category: string
    
    // 构造函数：创建对象时自动调用
    constructor(name: string, price: number, category: string) {
        this.name = name
        this.price = price
        this.category = category
    }
    
    // 方法（行为）
    showInfo(): void {
        console.log(`资产: ${this.name}, 价值: ¥${this.price}, 类别: ${this.category}`)
    }
}

// 用类创建对象
let laptop = new Asset("华为MateBook", 6999, "电子设备")
laptop.showInfo()

🔑 关键点： this 指向当前对象本身，就像说“我自己的名字”。

2. 接口（Interface）—— 强制的“合同标准”

📄 现实类比： 公司招聘程序员，要求“必须会Python、会Git、能沟通”。这就是一个接口，它规定了你必须拥有哪些技能，但不规定你怎么学会的。任何应聘者（类）只要满足这些技能，就能入职。

// 定义一个“可打印”的接口，规定必须有一个 print() 方法
interface Printable {
    print(): void
}

// 定义一个“可序列化”的接口
interface Serializable {
    toJSON(): object
}

// 类可以同时实现多个接口（用 implements）
class AssetReport implements Printable, Serializable {
    data: string
    
    constructor(data: string) {
        this.data = data
    }
    
    // 必须实现 Printable 要求的 print 方法
    print(): void {
        console.log("📄 报告：" + this.data)
    }
    
    // 必须实现 Serializable 要求的 toJSON 方法
    toJSON(): object {
        return { report: this.data }
    }
}

🤔 为什么需要接口？ 有了接口，不同类可以“统一标准”。比如任何实现了 Printable 的类，都可以被同一个函数调用 print()，代码更灵活！

3. 访问修饰符 · 公开的秘密与隐私保护

public（公开）

默认就是 public，任何地方都能访问。比如 asset.name 随便看。

private（私有）

只有类内部能访问，外部不能直接改。用 getter/setter 间接操作，保护数据安全。

protected（受保护）

类内部和子类可以访问，就像“家族内部秘密”。

class Asset {
    // ... 其他属性
    private _price: number     // 私有属性，习惯用下划线开头
    
    constructor(id: number, name: string, price: number, purchaseDate: string) {
        // ...
        this._price = price

    🧪 实训任务三：定义接口并让类实现（20分钟）

  • 定义 Exportable 接口： 要求实现 exportToCSV(): string 方法。
  • 让 Asset 类实现 Exportable： 在 class 后加 implements Exportable，并实现 exportToCSV 方法，返回 CSV 格式的资产数据（如 "101,华为MateBook,6999,2026-02-10"）。
  • 定义 Searchable 接口： 要求实现 matches(keyword: string): boolean，判断资产名称是否包含关键字。
  • 让 Asset 实现 Searchable： 实现 matches 方法（用 includes 判断）。
  • 测试： 创建一个资产列表，用循环筛选出包含“华为”的资产，并导出其 CSV 字符串。
    • // 定义接口
interface Exportable {
    exportToCSV(): string
}

interface Searchable {
    matches(keyword: string): boolean
}

// 类实现多个接口
class Asset implements Exportable, Searchable {
    // ... 原有属性和方法
    
    exportToCSV(): string {
        return `${this.id},${this.name},${this._price},${this.purchaseDate}`
    }
    
    matches(keyword: string): boolean {
        return this.name.includes(keyword)
    }
}

// 实战应用
let assets = [
    new Asset(1, "华为笔记本", 6999, "2026-02-01"),
    new Asset(2, "小米电视", 2999, "2026-02-05"),
    new Asset(3, "华为平板", 3499, "2026-02-10")
]

for (let item of assets) {
    if (item.matches("华为")) {
        console.log("📦 符合搜索：" + item.exportToCSV())
    }
}

    🧪 实训任务四：综合实战 —— 资产统计工具（15分钟）

    任务： 编写一个静态方法 calculateTotalPrice(assets: Asset[]): number，计算资产总价值。再编写一个方法 getExpensiveAssets(assets: Asset[], threshold: number): Asset[]，筛选出价格超过阈值的资产。

    挑战： 利用接口和类，创建一个资产管理器类 AssetManager，包含添加、删除、统计、搜索等功能。

    // 拓展：资产管理器
class AssetManager {
    private assetList: Asset[] = []
    
    add(asset: Asset): void {
        this.assetList.push(asset)
    }
    
    getTotalValue(): number {
        return this.assetList.reduce((sum, a) => sum + a.getPrice(), 0)
    }
    
    searchByKeyword(keyword: string): Asset[] {
        return this.assetList.filter(a => a.matches(keyword))
    }
}

// 演示
let manager = new AssetManager()
manager.add(new Asset(1, "鸿蒙开发板", 1280, "2026-02-01"))
manager.add(new Asset(2, "智能手表", 1999, "2026-02-03"))
console.log("资产总值：" + manager.getTotalValue())
console.log("搜索鸿蒙：" + manager.searchByKeyword("鸿蒙").length + "条")
