  • 核心代码提示（教师可展示骨架）： 
    @Entry
@Component
struct CardPage {
  @State slogan: string = "我的鸿蒙梦，从此刻开始！"

  build() {
    Column() {
      Text("😊").fontSize(50).margin({top: 30})
      Text("王小明").fontSize(24).fontWeight(FontWeight.Bold)
      Text("鸿蒙备赛1班").fontSize(16).fontColor("#666")
      Text(this.slogan).fontSize(18).margin(20)
      Button("加油！")
        .onClick(() => {
          this.slogan = "我是鸿蒙战士！🔥"
        })
        .backgroundColor("#0078d4")
        .borderRadius(20)
        .width(120)
    }
    .width('100%')
    .height('100%')
    .backgroundColor("#e6f4ff")
    .borderRadius(24)
    .padding(20)
  }
}

    • ⚠️ 避坑指南 · 第一阶段常见错误诊所

    🔴 错误1：装饰器漏写

    @Entry 和 @Component 缺一不可，否则页面无法显示。

    🔴 错误2：build() 里写多个根组件

    build() 只能有一个根组件，外面必须包一层 Column/Row/Stack。

    🔴 错误3：状态变量未加@State

    变量改变UI不更新？检查是否忘了 @State 装饰器。

    🔴 错误4：点击事件箭头函数

    .onClick(() => { ... }) 内不能用 this 丢失问题，正确写法是箭头函数。

    🛠️ 诊所实操： 老师故意展示一个“错误代码”，让学生分组找bug，最先找到并修正的小组加分。在实战中提升排错能力。

    // 请找出下面代码的3处错误 @Entry struct BugDemo { @State name = "小明" build() { Text(this.name) Button("改名").onClick(() => { name = "小红" }) } }

    答案提示：缺少 @Component、name 前面需加 this、Button应在Text下方且需包在容器内。
