🎯 学习目标 · 这节课你能掌握什么？

📘 知识点1 UIAbility是什么

学到： UIAbility是鸿蒙应用中承载UI的“能力单元”，类似于Android的Activity，但更轻量、更分布式。

🔍 类比：UIAbility就像一个“演员”，不同时刻（生命周期）演绎不同剧情。

🛠️ 技能点1 生命周期回调

学会： 重写 onCreate()、onForeground()、onBackground()、onDestroy() 等回调，并在控制台打印日志。

📘 知识点2 前后台切换

学到： 应用切到后台时触发 onBackground()，回到前台触发 onForeground()，理解Android与鸿蒙的异同。

🛠️ 技能点2 日志观察实战

学会： 使用 hilog 打印生命周期日志，通过“设备调试”观察应用行为。

📘 知识点3 销毁与资源释放

学到： 在 onDestroy() 中释放数据库连接、取消定时器等，避免内存泄漏。

🛠️ 技能点3 实战改造资产应用

学会： 在资产管理系统启动时加载数据，切后台时暂存状态，销毁时关闭资源。

💡 老师的话： 生命周期是每个鸿蒙开发者的必修课。掌握它，你就知道“应用什么时候初始化、什么时候保存数据、什么时候释放资源”，备赛作品的稳定性就靠它！

🧩 知识讲解 · UIAbility的五种状态

1. UIAbility的生命周期图谱

鸿蒙的 UIAbility 有5个核心状态：Create → Foreground → Background → Destroy，另外还有 WindowStageCreate/Destroy。

📌 生命周期回调执行顺序（重要！）

1. onCreate // 能力创建，做初始化操作（绑定变量，加载资源） 2. onWindowStageCreate // 窗口创建，在这里 setUIContent 加载页面 3. onForeground // 界面可见，进入前台（类似 onResume） ... 应用运行中 ... 4. onBackground // 切到后台（按Home键或跳转其他应用） 5. onWindowStageDestroy // 窗口销毁 6. onDestroy // UIAbility销毁，释放资源

🎯 实战理解：当用户打开你的“资产管理系统”时，依次触发 onCreate → onWindowStageCreate → onForeground；按Home键切后台，触发 onBackground；再次点击图标回到前台，触发 onForeground（不会重新走onCreate）。

2. 各个回调的作用详解

onCreate
初始化数据、创建单例、读取Preferences。适合加载本地资产列表。
onWindowStageCreate
通过 windowStage.loadContent 设置页面入口，类似“舞台搭好，演员上台”。
onForeground
应用获取焦点，开始交互。适合恢复实时数据或动画。
onBackground
用户暂时离开，适合保存草稿、暂停音视频、释放临时资源。
onDestroy
应用销毁时调用（如划掉后台卡片）。一定要释放数据库、取消网络请求！

⚙️ 技能实训 · 动手监听生命周期

✅ 本次实训目标：在原有的资产EntryAbility中增加生命周期日志，并观察前后台切换

🔧 实训一：修改EntryAbility，打印生命周期日志（30分钟）

  • 打开项目：在DevEco Studio中打开你的“AssetManagement”项目（或新建一个空白工程，注意包名）。
  • 定位 EntryAbility.ets：路径 entry/src/main/ets/entryability/EntryAbility.ets。
  • 导入 hilog：在文件顶部导入日志库
    import hilog from '@ohos.hilog';
  • 定义日志domain和tag：在类外定义
    const DOMAIN = 0x0001;
    const TAG = '[AssetAbility]';
  • 重写生命周期：在对应回调中添加 hilog 输出（参考下方代码）
    • import AbilityConstant from '@ohos.app.ability.AbilityConstant'; import hilog from '@ohos.hilog'; import UIAbility from '@ohos.app.ability.UIAbility'; import Want from '@ohos.app.ability.Want'; import window from '@ohos.window'; const DOMAIN = 0x0001; const TAG = '[AssetAbility]'; export default class EntryAbility extends UIAbility { onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void { hilog.info(DOMAIN, TAG, '✅ onCreate 执行 - 应用初始化'); // 这里可以加载Preferences、初始化全局数据 } onWindowStageCreate(windowStage: window.WindowStage): void { hilog.info(DOMAIN, TAG, '✅ onWindowStageCreate - 窗口创建，准备加载页面'); windowStage.loadContent('pages/Index', (err, data) => { if (err.code) { hilog.error(DOMAIN, TAG, '❌ 加载页面失败: ' + JSON.stringify(err)); return; } hilog.info(DOMAIN, TAG, '✅ 页面加载成功'); }); } onForeground(): void { hilog.info(DOMAIN, TAG, '🟢 onForeground - 应用进入前台，用户可交互'); // 恢复实时数据、刷新资产列表 } onBackground(): void { hilog.info(DOMAIN, TAG, '🔘 onBackground - 应用切到后台，保存临时状态'); // 比如保存当前正在编辑的资产草稿 } onWindowStageDestroy(): void { hilog.info(DOMAIN, TAG, '🗑️ onWindowStageDestroy - 窗口销毁'); } onDestroy(): void { hilog.info(DOMAIN, TAG, '💥 onDestroy - 应用被销毁，释放数据库连接等'); // 关闭数据库、移除监听器 } }

    📌 小组协作： 每人按照步骤修改代码，确保没有语法错误。老师巡视，助教协助解决导入问题。

    🔧 实训二：运行并观察日志（30分钟）

  • 连接真机或模拟器，点击运行按钮（或 Shift+F10）。
  • 打开 Log 面板：View → Tool Windows → Log，过滤器选择你的设备，搜索TAG [AssetAbility]。
  • 观察启动日志：你会看到依次打印 onCreate → onWindowStageCreate → onForeground。
  • 按Home键（模拟器或手机）：查看日志变化，应该出现 onBackground。
  • 从后台重新切回应用：观察 onForeground 再次被调用（不会再执行onCreate）。
  • 划掉应用卡片（杀掉进程）：观察 onWindowStageDestroy 和 onDestroy 执行，释放资源。

    • 📱 实训挑战： 改造你的资产管理首页，在 onForeground 中自动刷新资产列表，在 onBackground 中把当前列表滚动位置保存到 Preferences（预习思路，下节课正式实现）。

    核心代码参考（数据库初始化）：

    // 假设你有一个 DatabaseHelper 类 import database from '@ohos.data.relationalStore'; private dbHelper: database.RdbStore | null = null; async onCreate(want, launchParam) { hilog.info(DOMAIN, TAG, 'onCreate: 初始化数据库'); const STORE_CONFIG = { name: 'asset.db', securityLevel: database.SecurityLevel.S1 }; this.dbHelper = await database.getRdbStore(this.context, STORE_CONFIG); // 建表语句 const sql = `CREATE TABLE IF NOT EXISTS asset ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, price REAL )`; await this.dbHelper.executeSql(sql); } onBackground(): void { hilog.info(DOMAIN, TAG, '后台运行，暂存草稿'); // 后续扩展：保存未提交的资产名称、价格 }

    🎮 互动环节 · 生命周期侦探挑战赛

    「找出生命周期的Bug」

    情景： 以下代码存在两处生命周期相关的严重错误。请以小组为单位找出并修改。

    // 错误写法示例 ❌ onCreate(want, launchParam) { // 错误1：在onCreate里调用了loadContent（应该在onWindowStageCreate里） windowStage.loadContent('pages/Index'); } onWindowStageCreate(windowStage) { // 错误2：没有调用任何loadContent，页面永远不会显示 console.log("窗口创建了"); } onForeground() { // 正确做法：打印日志，但不影响主流程 hilog.info(DOMAIN, TAG, '前台'); }
    🔍 点击查看参考答案
    // 错误1修正：onCreate中不能使用windowStage，windowStage参数仅在onWindowStageCreate可用 onCreate(want, launchParam) { console.log("只做初始化工作，不涉及窗口"); } onWindowStageCreate(windowStage) { windowStage.loadContent('pages/Index'); // 修正2：必须调用setUIContent }
