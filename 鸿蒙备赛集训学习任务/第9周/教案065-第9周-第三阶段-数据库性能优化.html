🧩 知识讲解 · 数据库为什么慢？怎么让它快？

1. 索引 —— 数据库的“导航仪”

🗺️ 痛点场景： 资产表里存了10000条数据，你要找“笔记本”类别的所有资产。如果没有索引，数据库会一行一行扫描（全表扫描），就像你在图书馆一本一本地翻书，累死！

索引原理： 索引是一种有序的数据结构（B-Tree），类似于书籍的目录。有了索引，数据库能直接定位到数据所在位置，时间复杂度从O(N)降到O(logN)。

-- 创建索引的SQL语法 CREATE INDEX idx_asset_category ON assets(category); -- 查看索引（SQLite） PRAGMA index_list(assets);

🎯 什么时候该用索引？ 查询频繁的字段、用于WHERE/JOIN/ORDER BY的字段。但注意：索引会占用空间，且会减慢插入/更新速度，所以不是越多越好！

2. 查询优化 — 只拿你需要的数据

❌ 坏习惯

SELECT * FROM assets;
一次性拉取所有列，浪费内存和网络。

✅ 好习惯

SELECT id, name, price FROM assets WHERE category='笔记本';
只取需要的列，加上WHERE过滤。

-- 使用EXPLAIN QUERY PLAN分析查询计划 EXPLAIN QUERY PLAN SELECT * FROM assets WHERE category = '笔记本'; -- 如果看到“SCAN TABLE assets”表示全表扫描（慢） -- 如果看到“SEARCH TABLE assets USING INDEX idx_...”表示用索引查找（快）

3. 事务批量操作 —— 极速插入的秘密

🚀 为什么批量插入慢？ 默认每条INSERT都是一个独立事务，每次都要写磁盘、刷日志。

解决方案： 用 BEGIN TRANSACTION 和 COMMIT 包裹所有插入，让所有操作一次性提交，速度提升10~100倍！

// ArkTS 伪代码示例（后续实训会写） let db = await relationalStore.getRdbStore(context, config); await db.beginTransaction(); // 开启事务 try { for (let i = 0; i < 1000; i++) { await db.insert(table, values); } await db.commit(); // 提交事务 } catch (err) { await db.rollBack(); // 回滚 }

✅ 实训目标：亲自体验索引威力、事务加速，让查询提升10倍以上

🔧 实训准备：打开项目 + 准备测试数据

  • 打开VSCode，打开我们的鸿蒙项目 AssetManager。
  • 在 entry/src/main/ets/database 目录下新建文件 PerformanceTest.ets（如果没有，就临时创建一个测试页面）。
  • 确保已有资产表（id, name, category, price, quantity, createTime）。如果没有，先执行建表语句。

    • 🧪 实训任务一：对比有无索引的查询速度（20分钟）

    目标： 先插入5000条测试数据，然后分别查询“category='电子产品'”的数据，对比有索引和无索引的耗时。

  • 插入测试数据（使用事务）：编写函数批量插入5000条随机资产数据。
  • 第一次查询（无索引）：执行 SELECT * FROM assets WHERE category = '电子产品'，记录耗时（毫秒）。
  • 创建索引：执行 CREATE INDEX idx_category ON assets(category);
  • 第二次查询（有索引）：再次执行同样的查询，观察耗时变化。
  • 📊 记录结果：小组内共享数据，看看索引让查询变快了多少倍。
    • // 计时工具函数 async function testQuerySpeed() { let start = new Date().getTime(); let result = await queryAssetsByCategory('电子产品'); let end = new Date().getTime(); console.info(`查询耗时：${end - start} ms`); }

    🔥 预期结果： 无索引时可能耗时200~500ms，有索引后可能降到20ms以内！这就是索引的威力。

    🧪 实训任务二：批量插入性能对比 —— 事务 vs 无事务（20分钟）

  • 无事务插入：写一个循环，每次单独insert一条数据（插入500条），计时。
  • 有事务插入：用 beginTransaction() 开启事务，循环插入500条，最后 commit()，计时。
  • 对比结果：计算两种方式的耗时，算出加速比（例如无事务8秒，有事务0.2秒，加快40倍）。
  • 📌 小组讨论：为什么事务能这么快？原理是什么？（引导：减少了磁盘Sync次数）
    • // 事务批量插入示例（ArkTS） async function batchInsertWithTransaction(valuesList: Array<ValuesBucket>) { let db = await getRdbStore(); await db.beginTransaction(); try { for (let values of valuesList) { await db.insert('assets', values); } await db.commit(); console.info('批量插入成功'); } catch (e) { await db.rollBack(); console.error('插入失败，已回滚'); } }

    • 🧪 实训任务四（挑战）：模拟真实场景 —— 资产搜索优化（10分钟）

    需求： 用户需要一个“模糊搜索”功能，根据资产名称的关键词查找。例如输入“笔记”，能搜出“笔记本电脑”、“笔记本支架”等。

    挑战： 观察 LIKE '%笔记本%' 能否使用索引？如何优化？（提示：考虑全文搜索，或者限制搜索范围）

    实战： 写出查询语句，并尝试用 EXPLAIN 分析，看是否全表扫描。

    -- LIKE 以通配符开头会导致索引失效（全表扫描） SELECT * FROM assets WHERE name LIKE '%笔记本%'; -- 慢 -- 如果能改成"前缀匹配"，就能用上索引 SELECT * FROM assets WHERE name LIKE '笔记本%'; -- 快

    教师总结： 如果必须做全文模糊搜索，可以考虑引入第三方全文索引库，或限制搜索范围和分页。
