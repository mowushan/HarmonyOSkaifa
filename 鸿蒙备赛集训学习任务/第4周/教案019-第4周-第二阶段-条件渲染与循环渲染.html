🧩 知识讲解 · 让组件随数据而动

1. 条件渲染 —— 组件界的“开关灯”

💡 场景引入： 点击“显示详情”按钮，展开一段文字；再点击“隐藏”，文字消失。这就是条件渲染的典型应用。

ArkUI 条件渲染两种方式：

  • 方式一：if/else（推荐） — 组件会被完全创建/销毁，性能好，适合内容差异大的场景。
  • 方式二：visibility属性 — 组件保留在页面但不可见，适合频繁切换且结构简单的场景。
// 使用 if 条件渲染
@State showDetail: boolean = false;

build() {
    Column() {
        Button("切换详情")
            .onClick(() => {
                this.showDetail = !this.showDetail;
            })
        
        if (this.showDetail) {
            Text("这是详细内容，只有点击后才显示！")
                .fontSize(16)
                .padding(10)
        }
    }
}

2. 循环渲染 —— 批量生产组件

📋 场景引入： 假如有100条新闻数据，难道要写100次Text组件吗？当然不！用 `ForEach` 循环搞定！

ForEach语法结构：

ForEach(
    // 1. 数据源：数组
    this.newsList,
    // 2. 生成组件的函数
    (item: NewsItem, index: number) => {
        Text(item.title)
            .fontSize(16)
            .margin({ bottom: 8 })
    },
    // 3. key生成器（必须！）
    (item: NewsItem) => item.id
)

⚠️ 关键点： `key` 帮助系统高效识别每个组件，避免重新渲染时出现混乱。通常用唯一的 `id` 或 `index`（不推荐用index做key，除非数据不变）。

3. 组合拳：条件 + 循环 = 动态清单

🎯 实战思维： 我们的资产管理系统需要展示资产列表，每条资产可以“显示/隐藏备注”，还需要根据“资产状态”显示不同颜色。这就需要条件渲染 + 循环渲染联手！

🔧 实训一：热身练习——开关显示（15分钟）

  • 新建文件： 在 `entry/src/main/ets/pages` 下新建 `ConditionDemo.ets`。
  • 基础代码： 创建一个 `@Entry` 组件，包含一个按钮和一个Text，用 `@State showText` 控制Text显示/隐藏。
  • 挑战： 增加第二个按钮，控制显示不同的文本（如“内容A”和“内容B”）。

@Entry
@Component
struct ConditionDemo {
    @State showText: boolean = false;
    @State content: string = "默认内容";

    build() {
        Column({ space: 20 }) {
            Button("切换显示")
                .onClick(() => {
                    this.showText = !this.showText;
                })
            
            if (this.showText) {
                Text("我是被条件渲染出来的！")
                    .fontSize(18)
                    .fontColor('#0078d4')
            }
            
            Button("切换内容A/B")
                .onClick(() => {
                    this.content = this.content === "内容A" ? "内容B" : "内容A";
                })
            Text(this.content).fontSize(16)
        }
        .width('100%')
        .padding(20)
    }
}

    ✅ 运行观察：点击按钮，Text出现/消失；内容切换正常。教师巡视，确保每位同学都能独立完成。

    🔧 实训二：循环渲染——动态新闻列表（20分钟）

  • 定义数据模型： 创建 `NewsItem` 接口，包含 id 和 title。
  • 定义状态数组： `@State newsList: NewsItem[] = [...]`
  • 使用 ForEach 渲染： 在 Column 或 List 组件中遍历生成多个 Text。
  • 增加点击事件： 点击某条新闻，弹出 Toast 提示（或打印日志）。
    • // ForEachDemo.ets 核心代码
interface NewsItem {
    id: number;
    title: string;
}

@Entry
@Component
struct ForEachDemo {
    @State newsList: NewsItem[] = [
        { id: 1, title: "鸿蒙生态发展迅猛" },
        { id: 2, title: "DevEco Studio 5.0发布" },
        { id: 3, title: "元服务卡片大赛启动" },
        { id: 4, title: "零基础也能做鸿蒙开发" }
    ];

    build() {
        Column() {
            Text("📰 新闻速递")
                .fontSize(20)
                .fontWeight(FontWeight.Bold)
                .margin({ bottom: 15 })
            
            ForEach(this.newsList, (item: NewsItem, index: number) => {
                Row() {
                    Text(`${index + 1}. ` + item.title)
                        .fontSize(16)
                        .padding(10)
                        .onClick(() => {
                            console.info(`点击了：${item.title}`);
                            // 模拟提示，实际可弹Dialog
                        })
                }
                .width('100%')
                .borderRadius(8)
                .backgroundColor('#f5f5f5')
                .margin({ bottom: 8 })
            }, (item: NewsItem) => item.id.toString())
        }
        .padding(20)
        .width('100%')
    }
}

    🔑 关键讲解： ForEach 的第三个参数（key）至关重要！如果用 index 作为 key，当数组顺序改变时界面会混乱。这里用唯一的 id 做 key 最稳定。

    🔧 实训三：综合大挑战——待办清单（30分钟）

    📌 项目需求： 实现一个待办清单，具备以下功能：

    • ✅ 循环渲染显示所有待办事项（标题 + 完成状态）
    • ✅ 每个事项前有一个“完成”复选框，点击后该事项文字变为灰色加删除线（条件渲染样式）
    • ✅ 底部有一个“显示已完成/显示全部”按钮，切换列表显示范围（条件渲染 + 循环配合）
    • ✅ 新增输入框和按钮，可以添加新的待办事项（动态修改数组）

    💪 难度分层： 基础好的同学挑战全部功能，基础一般的至少完成循环渲染+复选框切换样式。

    // TodoList.ets 核心骨架（带完整逻辑）
interface Todo {
    id: number;
    title: string;
    completed: boolean;
}

@Entry
@Component
struct TodoList {
    @State todos: Todo[] = [
        { id: 1, title: "学习条件渲染", completed: false },
        { id: 2, title: "掌握ForEach循环", completed: false },
        { id: 3, title: "完成实训三挑战", completed: false }
    ];
    @State showOnlyCompleted: boolean = false;
    @State newTodoTitle: string = "";
    private nextId: number = 4;

    build() {
        Column() {
            Text("📝 我的待办清单")
                .fontSize(24)
                .fontWeight(FontWeight.Bold)
                .margin({ bottom: 20 })
            
            // 新增区域
            Row({ space: 10 }) {
                TextInput({ placeholder: "输入新任务...", text: this.newTodoTitle })
                    .onChange((value: string) => { this.newTodoTitle = value; })
                    .layoutWeight(1)
                Button("添加")
                    .onClick(() => {
                        if (this.newTodoTitle.trim() !== "") {
                            this.todos.push({
                                id: this.nextId++,
                                title: this.newTodoTitle,
                                completed: false
                            });
                            this.newTodoTitle = "";
                        }
                    })
            }
            .width('100%')
            .margin({ bottom: 20 })
            
            // 筛选按钮
            Button(this.showOnlyCompleted ? "显示全部" : "只显示已完成")
                .onClick(() => {
                    this.showOnlyCompleted = !this.showOnlyCompleted;
                })
                .margin({ bottom: 15 })
            
            // 循环渲染待办列表
            ForEach(
                this.todos.filter(todo => !this.showOnlyCompleted || todo.completed),
                (todo: Todo) => {
                    Row() {
                        Checkbox({ name: todo.id.toString() })
                            .select(todo.completed)
                            .onChange((value: boolean) => {
                                todo.completed = value;
                                // 触发UI更新
                                this.todos = [...this.todos];
                            })
                        Text(todo.title)
                            .fontSize(16)
                            .decoration({
                                type: todo.completed ? TextDecorationType.LineThrough : TextDecorationType.None
                            })
                            .fontColor(todo.completed ? '#999' : '#333')
                            .margin({ left: 10 })
                            .layoutWeight(1)
                    }
                    .padding(8)
                    .borderRadius(8)
                    .backgroundColor('#fafafa')
                    .margin({ bottom: 8 })
                },
                (todo: Todo) => todo.id.toString()
            )
        }
        .padding(20)
        .width('100%')
    }
}

    🎯 教师指导： 强调 `this.todos = [...this.todos];` 这一行的作用——触发状态更新，让UI刷新。这是ArkUI响应式更新的关键技巧。
