🧩 知识讲解 · 页面跳转与数据传递

1. 鸿蒙页面路由基础

🚦 什么是路由？ 路由就是页面之间的“导航系统”。鸿蒙提供了 @ohos.router 模块来实现页面跳转。

两种常用跳转方式：

  • pushUrl()：跳转到新页面，并保留当前页面（可返回）。适合详情页、设置页等。
  • replaceUrl()：替换当前页面，无法返回上一页。适合登录页跳转主页等。
// 跳转到详情页，并传递参数 router.pushUrl({ url: 'pages/AssetDetail', params: { id: 1001, name: '笔记本电脑', price: 5999 } })

2. 参数传递与接收

📤 发送方

// 在列表项的点击事件中 router.pushUrl({ url: 'pages/AssetDetail', params: { asset: this.assetItem } })

📥 接收方

// 在详情页 aboutToAppear() 中 const params = router.getParams(); this.assetData = params.asset;

3. 详情页UI设计思路

一个专业的详情页通常包含：

  • 🔙 返回按钮（可用系统导航栏，也可自定义）
  • 📷 资产图片（占位图或网络图片）
  • 📝 基本信息：名称、分类、购买日期、价格、状态等
  • ⚙️ 操作按钮：编辑、删除（后续课程实现）

⚙️ 技能实训 · 手把手实现资产详情页

✅ 本次实训目标：从列表跳转详情页，并正确显示资产详情

📁 准备阶段：创建详情页文件

  • 在 DevEco Studio 中，右键点击 pages 目录，选择 New → Page，命名为 AssetDetail。
  • 系统会自动生成 AssetDetail.ets 文件和路由配置。
  • 打开 AssetDetail.ets，清空默认代码，准备从头构建。

    • 🧪 实训任务一：编写详情页基础结构（15分钟）

    在 AssetDetail.ets 中编写以下代码，构建详情页框架：

    import router from '@ohos.router'; // 定义资产数据接口 interface AssetItem { id: number; name: string; category: string; purchaseDate: string; price: number; status: string; description?: string; } @Entry @Component struct AssetDetail { // 用于存储接收到的资产数据 @State assetData: AssetItem | null = null; aboutToAppear() { // 获取路由传递的参数 const params = router.getParams(); if (params && params['asset']) { this.assetData = params['asset'] as AssetItem; } else { // 测试数据：如果没有传参，显示默认示例 this.assetData = { id: 0, name: "示例资产", category: "电子设备", purchaseDate: "2025-01-01", price: 2999, status: "使用中", description: "这是一个示例资产，用于演示详情页布局" }; } } build() { Column() { // 标题栏 this.TitleBar() // 内容区域 // 资产卡片 this.AssetCard() } .padding(16) } .layoutWeight(1) .scrollBar('auto') } .width('100%') .height('100%') .backgroundColor('#F5F5F5') } // 标题栏组件 @Builder TitleBar() { Row() { Button({ symbol: $r('sys.symbol.arrow_left'), type: ButtonType.Circle }) .width(36) .height(36) .backgroundColor(Color.White) .onClick(() => { router.back(); }) Text("资产详情") .fontSize(20) .fontWeight(FontWeight.Bold) .layoutWeight(1) .textAlign(TextAlign.Center) // 占位保持对称 Row().width(36) } .width('100%') .padding({ left: 16, right: 16, top: 12, bottom: 12 }) .backgroundColor(Color.White) } // 资产信息卡片组件（待完善） @Builder AssetCard() { Column() { Text("资产信息将在这里显示") .fontSize(14) .fontColor('#999') } .width('100%') .backgroundColor(Color.White) .borderRadius(16) .padding(20) } }

    ✅ 此时运行项目，点击列表（如果还没修改列表页跳转）会看到基础框架。先确保编译通过！

    🧪 实训任务二：完善 AssetCard 组件，展示资产详情（20分钟）

    替换 AssetCard() 构建函数，将接收到的 assetData 展示出来：

    @Builder AssetCard() { Column() { // 资产名称 Text(this.assetData?.name || '未命名资产') .fontSize(24) .fontWeight(FontWeight.Bold) .width('100%') .margin({ bottom: 16 }) // 信息网格（两列布局） Grid() { GridItem() { this.InfoRow("📁 分类", this.assetData?.category || '--') } GridItem() { this.InfoRow("💰 价格", "¥" + (this.assetData?.price || 0)) } GridItem() { this.InfoRow("📅 购入日期", this.assetData?.purchaseDate || '--') } GridItem() { this.InfoRow("🔖 状态", this.assetData?.status || '--') } } .columnsTemplate('1fr 1fr') .rowsGap(16) .columnsGap(16) .margin({ bottom: 24 }) // 分割线 Divider().margin({ bottom: 16 }) // 详细描述 Column() { Text("📝 详细描述") .fontSize(16) .fontWeight(FontWeight.Medium) .margin({ bottom: 8 }) Text(this.assetData?.description || "暂无描述信息") .fontSize(14) .fontColor('#666') .lineHeight(22) } .alignItems(HorizontalAlign.Start) .width('100%') } .width('100%') .backgroundColor(Color.White) .borderRadius(16) .padding(20) } // 辅助组件：单行信息展示 @Builder InfoRow(label: string, value: string) { Column() { Text(label) .fontSize(12) .fontColor('#8a8a8a') .margin({ bottom: 4 }) Text(value) .fontSize(16) .fontWeight(FontWeight.Medium) .fontColor('#1a1a1a') } .alignItems(HorizontalAlign.Start) .width('100%') }

    📱 预期效果： 详情页会展示资产名称、分类、价格、购入日期、状态和描述，形成一个漂亮的卡片布局。

    🧪 实训任务三：改造资产列表页，实现点击跳转（20分钟）

    在资产列表页（AssetList.ets）中，为每个列表项添加点击事件：

    // 在 ListItem 上添加 onClick 事件 ListItem() { // 原有的列表项UI... Text(asset.name) } .onClick(() => { router.pushUrl({ url: 'pages/AssetDetail', params: { asset: asset } }).catch((err) => { console.error('跳转失败：' + JSON.stringify(err)); }) })

    🔧 注意事项： 需要 import router from '@ohos.router'。如果资产数据字段不全，可以在详情页的 aboutToAppear 中做默认值处理。
