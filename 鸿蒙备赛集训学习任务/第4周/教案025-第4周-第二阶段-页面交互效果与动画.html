🧩 知识讲解 · 鸿蒙动画三兄弟

1. 属性动画 —— 让组件“变形”的魔法

🎯 核心概念： 属性动画就是让组件的某个属性（宽、高、透明度、旋转角度等）在一段时间内平滑变化。

常用属性动画参数：

  • animation：配置动画时长、曲线、延迟等
  • scale：缩放比例 (scaleX, scaleY)
  • opacity：透明度 (0-1)
  • rotate：旋转角度
  • translate：位移

📌 生活类比： 就像你按下手机相机的快门键，按钮会有一个“弹跳”的反馈，这就是属性动画！

// 示例：一个点击放大的按钮 @State scaleValue: number = 1; Button('点击我变大') .width(120) .height(40) .scale({ x: this.scaleValue, y: this.scaleValue }) .animation({ duration: 300, // 动画时长300毫秒 curve: Curve.EaseOut, // 缓动曲线：先快后慢 iterations: 1 // 播放次数 }) .onClick(() => { this.scaleValue = this.scaleValue === 1 ? 1.2 : 1; })

2. 转场动画 —— 页面切换的“入场仪式”

// 路由跳转时添加转场动画 router.pushUrl({ url: 'pages/AssetDetailPage', params: { id: 1 } }, router.RouterType.Default) .then(() => { // 跳转成功后的回调 }) // 在目标页面的 aboutToAppear 中可以配置进入动画

💡 鸿蒙提供了丰富的转场效果：SlideEffect（滑动）、FadeEffect（淡入淡出）等。

3. 手势触发 —— 让用户“玩”出动画

👆 点击事件

.onClick(() => { ... })
最常用，按钮、卡片点击触发动画。

🤚 长按事件

.onLongPress(() => { ... })
长按出现删除或编辑动画。

👆👇 滑动事件

.onTouch() 或 .gesture()
滑动删除、滑动菜单。

⚙️ 技能实训 · 手把手实现四大动画特效

✅ 本次实训目标：给资产管理系统添加交互动效，让应用更有质感

🔧 实训一：按钮心跳动画（15分钟）

  • 新建文件： 在 entry/src/main/ets/pages/ 下新建 AnimationDemo.ets
  • 编写代码： 实现一个“添加资产”按钮，点击后有“心跳”缩放效果
    • // AnimationDemo.ets @Entry @Component struct AnimationDemo { @State heartScale: number = 1; @State isAnimating: boolean = false; build() { Column({ space: 30 }) { Text('❤️ 心跳按钮').fontSize(20).fontWeight(FontWeight.Bold) Button('➕ 添加资产') .width(150) .height(50) .fontSize(18) .backgroundColor('#0078D4') .scale({ x: this.heartScale, y: this.heartScale }) .animation({ duration: 400, curve: Curve.EaseOut, iterations: 1, onFinish: () => { this.heartScale = 1; // 动画结束后恢复原始大小 } }) .onClick(() => { this.heartScale = 1.15; // 放大15% console.log('心跳动画触发，准备添加资产'); }) Text('点击按钮会有一个“怦然心动”的放大效果') .fontSize(14) .fontColor('#666') } .width('100%') .height('100%') .padding(20) .justifyContent(FlexAlign.Center) } }

    🎯 挑战升级： 修改动画时长到200ms，再试试 Curve.Sharp（急停效果），感受不同曲线的区别。

    🔧 实训二：卡片悬浮阴影 + 抬起效果（20分钟）

  • 任务： 给资产卡片添加鼠标/手指悬浮效果，抬起时卡片“浮起来”。
  • 核心知识点： 动态改变阴影深度和Y轴位移。
    • // 资产卡片悬浮动画 @State cardElevation: number = 2; @State cardTranslateY: number = 0; build() { Column() { // ... 卡片内容 } .width('90%') .backgroundColor(Color.White) .borderRadius(16) .padding(15) .shadow({ radius: this.cardElevation, color: '#cccccc' }) .translate({ y: this.cardTranslateY }) .animation({ duration: 200, curve: Curve.EaseOut }) .onTouch((event: TouchEvent) => { if (event.type === TouchType.Down) { // 手指按下：卡片浮起 this.cardElevation = 12; this.cardTranslateY = -4; } else if (event.type === TouchType.Up) { // 手指抬起：恢复 this.cardElevation = 2; this.cardTranslateY = 0; } }) }

    🎨 提示：可以把这个效果应用到“资产列表页”的每个卡片上，让整个应用更有质感！

    🔧 实训三：页面转场动画（25分钟）

  • 目标： 从资产列表页跳转到资产详情页时，添加滑动进入动画。
  • 步骤： 修改路由配置，使用 router.pushUrl 并设置动画选项。
    • // 在资产列表页的跳转方法中 import router from '@ohos.router'; // 跳转到详情页，带转场动画 goToDetail(assetId: number) { router.pushUrl({ url: 'pages/AssetDetailPage', params: { id: assetId } }, router.RouterType.Default) } // 在 AssetDetailPage.ets 中配置页面进入动画 @Entry @Component struct AssetDetailPage { @State slideOffset: number = 100; aboutToAppear() { // 页面出现时执行动画：从右侧滑入 this.slideOffset = 0; } build() { Column() { // 详情内容 } .translate({ x: this.slideOffset }) .animation({ duration: 300, curve: Curve.FastOutSlowIn }) } }

    📌 扩展知识： 鸿蒙还提供了更高级的 PageTransition API，可以定制页面切换时的入场和离场动画。

    🔧 实训四：组合动画 —— 删除资产时的“消失特效”（20分钟）

  • 场景： 删除资产时，先缩小、再淡出，最后从列表中移除。
  • 技术点： 多个动画串行执行，用 onFinish 回调。

    🐛 常见问题 & 调试技巧

    ❌ 动画不执行

    原因： 忘记在组件的属性变化后添加 .animation() 修饰符。

    解决： 确保动画属性写在被动画修饰的组件上。

    ❌ 动画生硬、跳变

    原因： 动画时长太短或曲线不合适。

    解决： 调整 duration 和 curve，试试 Curve.EaseInOut。

    ❌ 转场动画无效

    原因： 页面路由方式错误或目标页面未配置。

    解决： 检查 router.pushUrl 的路径是否正确。
