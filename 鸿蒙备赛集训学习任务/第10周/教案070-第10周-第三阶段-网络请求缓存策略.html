🧩 知识讲解 · 缓存是什么？为什么需要它？

1. 缓存的故事：从“每次问老师”到“记在小本本上”

📖 趣味引入： 假设你每天都要问同桌“今天星期几”。第一天，同桌查了日历告诉你。第二天，你又问，同桌又查一遍……太累了！如果同桌把答案记在小本本上，以后直接看本子，是不是快多了？

网络请求缓存就是这个道理：

  • 🌐 第一次请求：从服务器获取数据，同时存到本地缓存。
  • ⚡ 第二次打开：先从缓存读取，瞬间显示，后台再偷偷更新。
  • 📱 效果：用户感知到的加载时间从2秒变成0.1秒！

🎯 这就是为什么很多优秀App第一次打开慢，后面每次都快——因为它们用了缓存！

2. 鸿蒙中的数据持久化与缓存方案

📦 Preferences

轻量级键值对存储，适合小数据量场景（如缓存用户设置、少量API返回数据）。

🗄️ 关系型数据库

适合大量结构化数据，可以配合缓存使用（如缓存新闻列表）。

📁 文件存储

适合图片、大文件缓存。我们后续再学。

今天我们先掌握最简单的方案：Preferences + 网络请求实现缓存。

3. 缓存策略的核心思想

缓存策略伪代码：
1. 发起请求时，先从缓存中读取数据
2. 如果有缓存且未过期 → 立即显示缓存数据
3. 同时发起网络请求获取最新数据
4. 获取到最新数据后更新界面 + 更新缓存
5. 如果没有缓存或缓存已过期 → 只显示网络数据

这样用户永远先看到“旧数据”（秒开），然后短暂等待后刷新成最新内容。体验最好！

🔧 实训一：创建带缓存的网络请求工具类（25分钟）

  • 在项目中新建文件 CacheUtil.ets，我们将封装缓存操作。
  • 编写Preferences工具代码： 
    import preferences from '@ohos.data.preferences';

export class CacheUtil {
    private static PREFERENCES_NAME = 'app_cache';
    private static preferencesInstance: preferences.Preferences | null = null;

    // 初始化Preferences
    static async init(context: any) {
        try {
            this.preferencesInstance = await preferences.getPreferences(context, this.PREFERENCES_NAME);
            console.info('缓存工具初始化成功');
        } catch (err) {
            console.error('缓存工具初始化失败：' + JSON.stringify(err));
        }
    }

    // 存入缓存（带过期时间，单位：毫秒）
    static async put(key: string, value: string, expireMs: number = 30 * 60 * 1000) {
        if (!this.preferencesInstance) return;
        let cacheData = {
            data: value,
            timestamp: Date.now(),
            expireMs: expireMs
        };
        await this.preferencesInstance.put(key, JSON.stringify(cacheData));
        await this.preferencesInstance.flush();
    }

    // 读取缓存（如果过期返回null）
    static async get(key: string): Promise<string | null> {
        if (!this.preferencesInstance) return null;
        let cacheStr = await this.preferencesInstance.get(key, '') as string;
        if (!cacheStr) return null;
        
        try {
            let cacheData = JSON.parse(cacheStr);
            let now = Date.now();
            if (now - cacheData.timestamp > cacheData.expireMs) {
                // 缓存过期
                return null;
            }
            return cacheData.data;
        } catch (e) {
            return null;
        }
    }

    // 清除指定缓存
    static async clear(key: string) {
        if (!this.preferencesInstance) return;
        await this.preferencesInstance.delete(key);
        await this.preferencesInstance.flush();
    }
}
  • 解释代码： 缓存不仅存数据，还存“时间戳”和“过期时长”，过期后自动失效。
  • 在EntryAbility的onCreate中初始化： CacheUtil.init(this.context)

    • 🔧 实训二：封装带缓存的网络请求（25分钟）

  • 创建 AssetApi.ets，专门处理资产相关的网络请求。
  • 编写带缓存功能的获取资产列表方法： 
    import http from '@ohos.net.http';
import { CacheUtil } from './CacheUtil';

export class AssetApi {
    private static BASE_URL = 'https://your-api.com'; // 替换为实际API地址

    // 获取资产列表（带缓存）
    static async getAssetList(forceRefresh: boolean = false): Promise<string> {
        const cacheKey = 'asset_list';
        
        // 1. 如果不强制刷新，先尝试读缓存
        if (!forceRefresh) {
            let cachedData = await CacheUtil.get(cacheKey);
            if (cachedData) {
                console.info('使用缓存数据');
                return cachedData;
            }
        }
        
        // 2. 发起网络请求
        console.info('发起网络请求获取最新数据');
        let httpRequest = http.createHttp();
        let response = await httpRequest.request(`${this.BASE_URL}/assets`, {
            method: http.RequestMethod.GET,
            header: { 'Content-Type': 'application/json' }
        });
        
        if (response.responseCode === 200) {
            let data = response.result as string;
            // 3. 存入缓存（过期时间30分钟）
            await CacheUtil.put(cacheKey, data, 30 * 60 * 1000);
            return data;
        } else {
            throw new Error(`请求失败：${response.responseCode}`);
        }
    }
    
    // 刷新资产列表（强制获取最新，并更新缓存）
    static async refreshAssetList(): Promise<string> {
        return await this.getAssetList(true);
    }
}
  • 核心逻辑讲解：
    • 普通调用：先缓存 → 秒开
    • 强制刷新：跳过缓存 → 获取最新
    • 缓存过期后自动重新请求

    • 🔧 实训三：在页面中使用缓存功能（20分钟）

  • 修改资产管理页面，实现“先缓存后刷新”的效果： 
    import { AssetApi } from '../api/AssetApi';

@Entry
@Component
struct AssetListPage {
    @State assetList: string = '';
    @State isLoading: boolean = false;

    aboutToAppear() {
        this.loadData();
    }

    async loadData() {
        this.isLoading = true;
        try {
            // 第一步：先加载缓存（如果有，立即显示）
            let cachedData = await AssetApi.getAssetList(false);
            if (cachedData) {
                this.assetList = cachedData;
            }
            
            // 第二步：后台获取最新数据（不阻塞界面）
            let freshData = await AssetApi.refreshAssetList();
            this.assetList = freshData;
        } catch (err) {
            console.error('加载失败：' + JSON.stringify(err));
        } finally {
            this.isLoading = false;
        }
    }

    build() {
        Column() {
            if (this.isLoading && !this.assetList) {
                LoadingProgress().width(50).height(50)
            } else {
                Text(this.assetList || '暂无数据').fontSize(16)
            }
            
            Button('刷新').onClick(() => {
                this.loadData(); // 点击刷新时强制获取最新
            }).margin(20)
        }
        .width('100%')
        .height('100%')
        .padding(20)
    }
}
  • 演示效果：
    1. 第一次打开 → 转圈等待网络返回（0-2秒）
    2. 第二次打开 → 瞬间显示缓存内容（0.1秒），然后自动更新
    3. 点击刷新按钮 → 强制获取最新数据
