🧩 知识讲解 · WebSocket：让服务器主动找上门

1. 为什么需要WebSocket？—— 告别“手动刷新”

📡 生活场景： 你在刷微博，每隔几秒就要手动刷新才能看到新消息——这就是HTTP的极限：客户端主动问，服务器被动答。

WebSocket的魔法： 客户端和服务器握手成功后，两者随时可以互相发消息！服务器有新消息能立刻推给你，就像微信收到新消息一样实时。

对比项HTTP（传统）WebSocket
通信方式请求-响应（一问一答）双向实时（随时说话）
连接开销每次请求重新建立连接一次握手，持久连接
实时性需轮询（定时问）服务器主动推送
适用场景网页加载、数据查询聊天、游戏、通知、协作

2. WebSocket在鸿蒙中怎么用？—— 核心API

🔌 创建连接

import webSocket from '@ohos.net.webSocket'
创建WebSocket实例，调用connect()方法。

🎧 监听事件

on('open')连接成功
on('message')收到消息
on('close')连接关闭
on('error')发生错误

📤 发送消息

send(String) 或 send(ArrayBuffer)

🔒 关闭连接

close() 主动断开连接

3. WebSocket 协议格式小揭秘（趣味版）

🔬 WebSocket连接地址格式：ws://服务器地址:端口 或 wss://（加密版）。

💬 例如测试服务器：ws://echo.websocket.org —— 它会原样返回你发送的任何消息（回显服务器），非常适合练习！

let ws = webSocket.createWebSocket();

ws.connect('ws://echo.websocket.org', (err, value) => {
  if (!err) {
    console.info('WebSocket连接成功！');
  } else {
    console.error('连接失败：' + JSON.stringify(err));
  }

ws.on('open', () => {
  console.info('onOpen回调：连接已打开');
});

ws.on('message', (data) => {
  console.info('收到消息：' + data);
});

    🔧 实训三：发送消息与接收回显（20分钟）

  • 在界面上添加一个TextInput输入框和一个Button发送按钮。
  • 点击发送按钮时调用ws.send(inputText)发送消息。
  • 观察on('message')回调，收到的回显消息显示在屏幕上。
  • 完成一个简单的“聊天式”UI：显示发送记录和收到的回复。
    • @Entry
@Component
struct WebSocketPage {
  @State inputMsg: string = '';
  @State outputMsg: string = '等待消息...';

  build() {
    Column() {
      Text('WebSocket 回显测试').fontSize(20).margin(10);
      TextArea({ text: $$this.inputMsg })
        .width('90%')
        .height(80)
        .backgroundColor('#f5f5f5')
        .margin(10);
      Button('发送消息').onClick(() => {
        if (this.inputMsg.trim() !== '') {
          ws.send(this.inputMsg, (err) => {
            if (err) console.error('发送失败');
            else console.info('发送成功');
          });
        }
      }).margin(10);
      Text('服务器回显：').fontWeight(FontWeight.Bold).margin(10);
      Text(this.outputMsg).padding(10).backgroundColor('#e0f2fe')
        .width('90%').borderRadius(12);
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center);
  }
}

    🔧 实训四：心跳保活机制（10分钟）

    为什么需要心跳？ 网络不稳定或空闲时，连接可能被中间路由器断开。心跳（定期发送ping/pong）能保持连接活跃。

    实现思路： 连接成功后启动定时器（setInterval），每30秒发送一个“ping”消息，服务器通常会回复“pong”。

    let heartBeatInterval: number;

ws.on('open', () => {
  // 每30秒发送一次心跳
  heartBeatInterval = setInterval(() => {
    ws.send('ping');
  }, 30000);
});

ws.on('close', () => {
  if (heartBeatInterval) clearInterval(heartBeatInterval);
});

    🐛 常见Bug & 调试小贴士

    ❌ 连接失败

    检查网络权限、服务器地址是否正确（ws:// 不是 http://），真机需要联网。

    ❌ 收不到消息

    确认on('message')回调已注册，并且服务器确实推送了消息。用echo服务器验证。

    ❌ 连接自动断开

    加入心跳机制；检查防火墙或代理设置；尝试wss加密连接。

    ✅ 调试技巧

    使用console.info打印关键步骤；在DevEco Studio的Log面板筛选“WebSocket”标签。
