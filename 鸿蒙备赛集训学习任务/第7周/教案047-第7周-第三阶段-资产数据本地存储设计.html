🧩 知识讲解 · 数据存储三板斧

1. 为什么需要本地存储？—— 告别“重启就丢”

🗂️ 场景小剧场： 小张辛苦录入了50条资产，一关App再打开，全部消失！😭 这就是因为没有做数据持久化。

鸿蒙提供了三种主要存储方式：

  • 📝 Preferences：轻量级键值对，适合存设置、用户名等少量数据。
  • 🗄️ 关系型数据库 (relationalStore)：功能强大，支持复杂查询，适合资产清单这种结构化数据。
  • 📁 文件存储：存图片、日志等大文件。

🎯 我们今天重点攻克Preferences + SQLite数据库，为资产管理系统打牢地基。

2. Preferences 快速入门 —— 存“设置”利器

// 1. 导入模块 import dataPreferences from '@ohos.data.preferences'; // 2. 获取Preferences实例（异步） let pref = await dataPreferences.getPreferences(this.context, 'my_settings'); // 3. 写入数据 await pref.put('user_name', '张三'); await pref.put('is_first_launch', false); await pref.flush(); // 必须调用flush持久化 // 4. 读取数据 let userName = await pref.get('user_name', '默认值');

📌 应用场景： 保存“是否记住密码”、“主题颜色”、“上次登录账号”等。

3. 关系型数据库 (SQLite) —— 存储资产清单

// 创建数据库 & 建表 import relationalStore from '@ohos.data.relationalStore'; const SQL_CREATE_TABLE = ` CREATE TABLE IF NOT EXISTS ASSETS ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, category TEXT, purchase_date TEXT, price REAL, description TEXT ) `;

💡 我们使用SQLite存储每一条资产，可以按类别、日期查询，后面还能实现搜索和排序！

4. 数据模型 (Entity) —— 让代码更规范

// 定义AssetItem接口，对应数据库的一条记录 export interface AssetItem { id?: number; // 可选，新增时自动生成 name: string; category: string; purchaseDate: string; price: number; description: string; }

✅ 有了这个模型，我们在代码中操作资产数据时会更清晰，TypeScript还会自动检查类型错误！

    ✅ 验证：代码无红色波浪线，接口定义正确。

    🧪 实训任务二：创建数据库工具类 DBHelper（25分钟）

  • 打开 DBHelper.ets，导入必要模块：
    • import relationalStore from '@ohos.data.relationalStore'; import { AssetItem, NewAssetItem } from './AssetItem'; import common from '@ohos.app.ability.common';
  • 编写 DBHelper 类，实现初始化、插入资产方法：
    • export class DBHelper { private store: relationalStore.RdbStore | null = null; private context: common.Context; constructor(context: common.Context) { this.context = context; } // 初始化数据库（建表） async init(): Promise<void> { const config: relationalStore.StoreConfig = { name: 'asset.db', // 数据库文件名 securityLevel: relationalStore.SecurityLevel.S1 }; this.store = await relationalStore.getRdbStore(this.context, config); // 建表SQL const createTableSql = ` CREATE TABLE IF NOT EXISTS ASSETS ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, category TEXT, purchase_date TEXT, price REAL, description TEXT ) `; await this.store.executeSql(createTableSql); console.log('数据库初始化成功，表已创建'); } // 插入一条新资产 async insertAsset(asset: NewAssetItem): Promise<number> { if (!this.store) { throw new Error('数据库未初始化，请先调用 init()'); } const valueBucket: relationalStore.ValuesBucket = { name: asset.name, category: asset.category, purchase_date: asset.purchaseDate, price: asset.price, description: asset.description }; const rowId = await this.store.insert('ASSETS', valueBucket); console.log(`插入成功，行ID: ${rowId}`); return rowId; } // 查询所有资产（下周详细实现） async queryAllAssets(): Promise<AssetItem[]> { // 预留，下周做 return []; } }

    ⚠️ 常见错误提醒：

    • 忘记调用 await this.store.executeSql() 建表会导致插入失败。
    • 字段名必须和SQL中的列名一致（如 purchase_date 而不是 purchaseDate）。
    • 插入前务必确保 store 已初始化（if判断）。

    🧪 实训任务三：在录入页集成数据库插入（25分钟）

  • 打开 pages/AddAssetPage.ets（资产录入页面）。
  • 在页面顶部导入 DBHelper 和 AssetItem：
    • import { DBHelper } from '../common/database/DBHelper'; import { NewAssetItem } from '../common/database/AssetItem';
  • 声明一个成员变量 dbHelper：
    • @Entry @Component struct AddAssetPage { private dbHelper: DBHelper = new DBHelper(getContext(this)); // ... 其余状态变量 }
  • 在 aboutToAppear 生命周期中初始化数据库：
    • async aboutToAppear() { await this.dbHelper.init(); }
  • 修改“保存”按钮的逻辑，调用 insertAsset：
    • // 保存按钮点击事件 async onSave() { if (!this.nameInput || !this.categoryInput) { promptAction.showToast({ message: '请填写资产名称和分类' }); return; } const newAsset: NewAssetItem = { name: this.nameInput, category: this.categoryInput, purchaseDate: this.dateInput, price: Number(this.priceInput) || 0, description: this.descInput }; try { const rowId = await this.dbHelper.insertAsset(newAsset); promptAction.showToast({ message: `保存成功！记录ID: ${rowId}` }); // 清空表单或返回上一页 this.clearForm(); } catch (err) { promptAction.showToast({ message: '保存失败：' + JSON.stringify(err) }); console.error('插入错误', err); } }

    🎉 阶段性成果： 现在运行应用，填写资产信息并点击保存，如果看到“保存成功”提示，说明数据已经写入SQLite数据库！
