🧩 知识讲解 · 数据库是什么？为什么需要它？

1. 为什么需要数据库？—— 告别“一重启就丢数据”

📦 趣味引入： 之前我们用Preferences存简单的设置，但资产数据有几十上百条，需要快速查询、排序、修改。Preferences就像一个小抽屉，只能放几样东西；数据库就像一个大仓库，有货架、有标签、还能用“SQL语言”快速找到任何东西！

SQLite三大优势：

  • ⚡ 零配置：不需要安装数据库服务器，鸿蒙自带支持。
  • 📁 单文件：一个.db文件就是一个完整数据库，方便备份和迁移。
  • 🔍 SQL查询：用类似英语的SQL语句操作数据，比如“SELECT * FROM assets WHERE price > 1000”。

2. SQL基础语法 · 五分钟学会“数据库普通话”

📋 CREATE TABLE

CREATE TABLE assets ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, price REAL, category TEXT );

创建资产表，id自增主键，name不能为空。

➕ INSERT

INSERT INTO assets (name, price, category) VALUES ('笔记本电脑', 5999, '电子设备');

插入一条资产记录。

🔍 SELECT

SELECT * FROM assets; SELECT name, price FROM assets WHERE price > 1000;

查询所有资产，或条件筛选。

✏️ UPDATE

UPDATE assets SET price = 5500 WHERE name = '笔记本电脑';

更新特定资产的价格。

🗑️ DELETE

DELETE FROM assets WHERE id = 3;

删除id为3的资产。

📖 SQL不区分大小写，但习惯把关键字大写，表名/字段名小写，更易阅读。

3. 鸿蒙关系型数据库API · 三步走

  • 获取数据库管理实例：relationalStore.getRdbStore(context, config, callback)
  • 执行SQL语句：rdbStore.executeSql(sql, bindArgs, callback) 或专用增删改查方法
  • 关闭数据库：rdbStore.close() 释放资源
    • // 鸿蒙API示例（Stage模型） import relationalStore from '@ohos.data.relationalStore'; const STORE_CONFIG = { name: "AssetDB.db", securityLevel: relationalStore.SecurityLevel.S1 }; relationalStore.getRdbStore(this.context, STORE_CONFIG, (err, rdbStore) => { if (err) { console.error('获取数据库失败' + err); return; } // 建表SQL rdbStore.executeSql(`CREATE TABLE IF NOT EXISTS assets ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, price REAL, category TEXT )`, null, (err) => { if (err) console.error('建表失败'); else console.log('数据库初始化成功'); }); });

    ⚙️ 技能实训 · 从零到一构建资产数据库

    ✅ 实训目标：独立完成数据库创建、建表、插入和查询操作

    🔧 实训一：创建鸿蒙工程并配置数据库模块（15分钟）

  • 打开DevEco Studio，在之前的备赛项目中新建一个Ability或在现有页面中添加数据库逻辑。
  • 在module的oh-package.json5 确保已包含关系型数据库权限（无需额外添加，基础SDK已包含）。
  • 在EntryAbility或自定义DatabaseHelper中，编写初始化代码（参考上方示例）。
  • 添加日志输出：在回调中打印“数据库创建成功”，运行应用查看HiLog。

    • 💡 常见问题： 如果报错“getRdbStore fail”，检查context是否正确传入（this.context）。

    🔧 实训二：编写建表语句 · 为资产设计“表格” （15分钟）

  • 在数据库初始化成功后，执行建表SQL：
    • // 建表SQL字符串 const CREATE_TABLE_SQL = ` CREATE TABLE IF NOT EXISTS assets ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, price REAL, category TEXT, purchase_date TEXT, location TEXT ) `; rdbStore.executeSql(CREATE_TABLE_SQL, null, (err) => { if (err) { console.error('建表失败：' + err); } else { console.info('assets表创建成功'); } });

    📌 使用 IF NOT EXISTS 防止重复建表报错，让代码更健壮。

    🔧 实训三：插入数据 · 给仓库“进货” （20分钟）

  • 准备插入数据：定义几个资产对象，例如“实训台”、“示波器”。
  • 编写插入SQL：使用参数化查询防止SQL注入（使用?占位符）。
  • 执行插入并验证：插入成功后查询显示。
    • // 参数化插入 const insertSql = `INSERT INTO assets (name, price, category, location) VALUES (?, ?, ?, ?)`; const values = ["数字示波器", 3200, "测量仪器", "实训室101"]; rdbStore.executeSql(insertSql, values, (err) => { if (err) { console.error('插入失败：' + err); } else { console.info('插入成功，示波器已入库'); } });

    每位同学至少插入3条不同资产，然后互相交换验证。

    🔧 实训四：查询与展示 · 让数据“说话” （20分钟）

  • 编写查询SQL：查询所有资产，按价格降序排列。
  • 使用rdbStore.querySql() 获取结果集。
  • 遍历结果集并打印到控制台，观察输出。
  • 挑战任务： 添加条件查询，只显示价格大于2000的资产。
    • const querySql = `SELECT * FROM assets ORDER BY price DESC`; rdbStore.querySql(querySql, null, (err, resultSet) => { if (err) { console.error('查询失败：' + err); return; } console.info(`共查询到 ${resultSet.rowCount} 条资产`); while (resultSet.goToNextRow()) { let id = resultSet.getLong(resultSet.getColumnIndex("id")); let name = resultSet.getString(resultSet.getColumnIndex("name")); let price = resultSet.getDouble(resultSet.getColumnIndex("price")); console.info(`ID:${id}, 名称:${name}, 价格:${price}`); } resultSet.close(); // 记得关闭结果集 });

    🧪 小组互动： 相邻小组交换数据库文件（拷贝.db文件到对方设备），互相查询对方插入的数据，验证数据持久化效果。
