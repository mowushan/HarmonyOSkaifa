🧩 知识讲解 · 错误处理的“防弹衣”

1. 异常与崩溃 —— 程序也会“生病”

🤒 趣味例子： 你让程序计算“10 / 0”，数学上不允许，程序就会“抛异常”；如果不处理，应用直接闪退——这就是崩溃！

常见异常类型（ArkTS/JS）：

  • SyntaxError：语法错误（写错单词、少括号）
  • TypeError：类型错误（对数字调用方法）
  • ReferenceError：引用错误（变量未定义）
  • NetworkError：网络请求失败

✅ 学会捕获异常，就能在“摔倒前接住自己”。

2. try-catch-finally —— 程序的“安全气囊”

// 基本语法 try { // 可能出错的代码 let result = 10 / 0; console.info("结果：" + result); } catch (error) { // 出错时执行的代码 console.error("捕获到异常：" + error.message); } finally { // 无论是否出错都会执行（比如关闭资源） console.info("执行最终清理"); }

执行流程： try代码块 → 如果出错跳转到catch → 最后一定会执行finally。

3. hilog日志系统 —— 开发者的“侦察兵”

鸿蒙系统提供hilog日志API，可以分级输出，方便在调试窗口中查看。

import hilog from '@ohos.hilog'; // 日志级别： DEBUG < INFO < WARN < ERROR < FATAL hilog.debug(0x0000, "资产模块", "这是一条调试信息"); hilog.info(0x0000, "资产模块", `用户ID: ${userId}`); hilog.warn(0x0000, "资产模块", "网络连接较慢"); hilog.error(0x0000, "资产模块", `请求失败: ${errorMsg}`);

📌 小窍门： 在DevEco Studio的Log窗口中可以按“domain”和“tag”过滤，快速找到你自己的日志。

4. 错误上报思路 —— 崩溃也要留痕

📡 真实场景中，应用发布后用户遇到崩溃，开发者无法现场调试。因此需要“错误上报”：把崩溃信息（堆栈、设备信息、操作步骤）发送到服务器。

本节课我们做简化版：

  • 捕获异常后，将错误信息写入本地文件（后面可扩展成网络上报）。
  • 模拟“主动上报”按钮，向控制台输出上报内容。

    🔧 实训三：模拟错误上报（25分钟）

    场景模拟： 资产管理系统在添加资产时，如果数据校验失败，除了提示用户，还要把错误记录到本地文件，模拟上报服务器。

  • 封装错误上报工具类： 新建 `utils/ErrorReporter.ets`。
  • 写入文件： 使用文件管理接口将错误信息追加到本地 error.log。
  • 模拟网络上报： 封装 `reportToServer(errorInfo)` 函数，目前只打印“上报成功：”+错误信息。
  • 集成到添加资产逻辑： 当用户输入非法数据（如名称为空）时，调用上报方法。
    • // utils/ErrorReporter.ets import fs from '@ohos.file.fs'; import hilog from '@ohos.hilog'; export class ErrorReporter { private static LOG_FILE_PATH = '/data/storage/el2/base/haps/entry/files/error.log'; static async saveToLocal(errorMsg: string) { try { let file = fs.openSync(this.LOG_FILE_PATH, fs.OpenMode.READ_WRITE | fs.OpenMode.CREATE); let content = `${new Date().toISOString()} | ${errorMsg}\n`; fs.writeSync(file.fd, content); fs.closeSync(file); hilog.info(0x0001, 'Reporter', '本地记录完成'); } catch (err) { hilog.error(0x0001, 'Reporter', `本地写入失败: ${err.message}`); } } static async reportToServer(errorMsg: string) { // 模拟网络请求 hilog.info(0x0001, 'Reporter', `【模拟上报】错误内容: ${errorMsg}`); // 实际项目可用 http 模块发送 } }

    📁 重要提醒： 真机调试时需要申请文件读写权限（`ohos.permission.WRITE_USER_STORAGE`），模拟器默认可用。注意在module.json5中配置权限。
