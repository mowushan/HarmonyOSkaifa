🧩 知识讲解 · 异常处理三板斧

1. 异常是什么？—— 程序也会“生病”

🤒 趣味引入： 想象一下，你让程序做“5除以0”，这在数学上无意义。程序就会“生病”——抛出一个ZeroDivisionError异常！

两类错误对比：

  • 🔴 语法错误（SyntaxError）： 代码写错了，比如少了个冒号，程序根本跑不起来。
  • 🟡 运行时异常（Exception）： 语法没错，但运行时出状况了，比如文件不存在、网络断开、除数为0。
# 语法错误示范：这行代码根本跑不起来
# if 5 > 2      # 少了冒号，SyntaxError

# 运行时异常示范：语法没问题，但运行报错
# result = 5 / 0   # ZeroDivisionError: division by zero

2. try-except — 程序的“安全气囊”

📖 基本语法

try:
    # 可能会出错的代码
     risky_code()
except:
    # 出错后执行的代码
     handle_error()

🎯 捕获特定异常

try:
    num = int(input("输入数字: "))
    result = 10 / num
except ZeroDivisionError:
    print("不能除以0！")
except ValueError:
    print("请输入有效数字！")

3. 完整的异常处理结构

try:
    # 尝试执行的代码
    file = open("data.txt", "r")
    content = file.read()
except FileNotFoundError:
    # 文件不存在时的处理
    print("文件不存在，将创建新文件")
    file = open("data.txt", "w")
except Exception as e:
    # 捕获其他所有异常
    print(f"未知错误: {e}")
else:
    # 没有异常时执行（可选）
    print("文件读取成功")
finally:
    # 无论是否异常都会执行（可选）
    if file:
        file.close()
    print("资源已释放")

📌 记忆口诀： try试探，except接盘，else立功，finally扫尾！

4. 主动抛出异常 — raise

有时候，我们自己发现数据不合理，可以主动“叫停”并抛出异常：

def add_asset(price):
    if price < 0:
        raise ValueError("资产价格不能为负数！")
    # 正常处理...
    print(f"添加资产，价格：{price}")

# 调用
add_asset(-100)   # 会抛出ValueError

def add_asset():
    try:
        name = input("资产名称: ").strip()
        if not name:
            raise ValueError("资产名称不能为空")
        
        price = float(input("资产价格: "))
        if price <= 0:
            raise ValueError("资产价格必须为正数")
        
        quantity = int(input("资产数量: "))
        if quantity <= 0:
            raise ValueError("资产数量必须为正整数")
        
        asset = {"name": name, "price": price, "quantity": quantity}
        assets.append(asset)
        print(f"✅ 成功添加资产：{name}")
        
    except ValueError as e:
        print(f"❌ 输入错误：{e}")
    except Exception as e:
        print(f"❌ 未知错误：{e}")
