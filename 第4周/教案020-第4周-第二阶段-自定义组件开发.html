🧩 知识讲解 · 为什么需要自定义组件？

1. 从重复代码说起

😵 糟糕的代码长这样： 资产列表里有10个资产卡片，每个卡片都是一大段重复的 Column + Text + Button。

修改一个样式？ 要改10个地方！
添加新功能？ 要加10次！
这就是“复制粘贴工程师”的噩梦。

✅ 解决方案： 把“资产卡片”封装成一个组件，以后只需要写一行 AssetCard()，传入不同数据即可！

2. 鸿蒙自定义组件语法 · 三件套

// 1️⃣ 使用 @Component 装饰器标记这是一个自定义组件 @Component export struct AssetCard { // 2️⃣ 定义组件参数（接收父组件传值） @Prop name: string = '' @Prop amount: number = 0 @Prop type: string = '电子设备' // 3️⃣ build() 方法描述UI结构 build() { Column() { Text(this.name) .fontSize(18) .fontWeight(FontWeight.Bold) Text(`¥ ${this.amount}`) .fontSize(16) .fontColor('#FF6600') Text(this.type) .fontSize(14) .fontColor('#666666') } .width('100%') .padding(12) .backgroundColor('#F5F5F5') .borderRadius(12) .margin({ bottom: 10 }) } }

💡 关键点： @Prop 让组件可以接收外部数据，就像函数的参数一样！

3. 父组件如何使用？—— 一行代码搞定

// 在父组件（如资产列表页）中使用 @Component export struct AssetList { build() { Column() { // 像用普通组件一样使用AssetCard，传入不同的数据 AssetCard({ name: 'MacBook Pro', amount: 12999, type: '电脑' }) AssetCard({ name: 'iPad Air', amount: 4799, type: '平板' }) AssetCard({ name: 'iPhone 15', amount: 5999, type: '手机' }) } } }

✨ 是不是瞬间清爽了很多？以后增加新的资产，只需要加一行 AssetCard！

4. 组件事件：让子组件“喊话”父组件

🔔 场景需求

点击卡片时，父组件需要知道“哪个资产被点了”，然后跳转到详情页。

解决方案： 子组件定义回调函数，父组件传入实现。

📦 代码示例（子组件）

@Component struct AssetCard { @Prop name: string // 定义回调属性，类型是函数 onCardClick?: () => void build() { Column() { Text(this.name) } .onClick(() => { // 点击时调用父组件传进来的函数 if (this.onCardClick) { this.onCardClick() } }) } }

⚙️ 技能实训 · 手把手封装资产卡片组件

✅ 本次实训目标：从零封装AssetCard组件，并在列表页使用

🔧 实训准备 · 打开项目

  • 打开 DevEco Studio，找到之前的“资产管理”项目（若没有则新建一个空白工程）。
  • 在 entry/src/main/ets/pages 目录下新建文件夹 components。
  • 在 components 文件夹下新建文件 AssetCard.ets（这就是我们的自定义组件文件）。

    • 🧪 实训任务一：编写 AssetCard 组件（15分钟）

  • 打开 AssetCard.ets，先编写组件结构，完整代码如下（教师可投影）：
    • // AssetCard.ets @Component export struct AssetCard { // 接收父组件传入的数据 @Prop name: string = '' @Prop amount: number = 0 @Prop type: string = '其他' @Prop icon: string = '📦' // 事件回调（可选） onCardClick?: () => void build() { Row() { // 左侧图标 Text(this.icon) .fontSize(32) .width(50) .height(50) .textAlign(TextAlign.Center) // 中间信息区 Column() { Text(this.name) .fontSize(18) .fontWeight(FontWeight.Bold) .width('100%') Text(this.type) .fontSize(12) .fontColor('#999999') .width('100%') } .layoutWeight(1) .alignItems(HorizontalAlign.Start) // 右侧金额 Text(`¥${this.amount.toFixed(2)}`) .fontSize(18) .fontWeight(FontWeight.Bold) .fontColor('#FF6600') } .width('100%') .height(70) .padding({ left: 12, right: 12 }) .backgroundColor('#FFFFFF') .borderRadius(12) .margin({ bottom: 8 }) .onClick(() => { if (this.onCardClick) { this.onCardClick() } }) } }

    📌 学生动手： 逐行敲入代码，注意大小写和符号。老师巡视，检查是否有语法错误。

    💡 要点讲解： Row水平布局 + Column垂直布局，让卡片更美观；toFixed(2)保留两位小数。

    🧪 实训任务二：在资产列表页中使用自定义组件（20分钟）

  • 打开 Index.ets（或之前的资产列表页），清空原来的重复代码。
  • 在文件顶部导入组件：import { AssetCard } from './components/AssetCard'
  • 编写资产列表数据（模拟数据）和列表渲染代码：
    • // Index.ets import { AssetCard } from './components/AssetCard' @Entry @Component struct AssetListPage { // 模拟资产数据 @State assets: Array<{ name: string, amount: number, type: string, icon: string }> = [ { name: 'MacBook Pro', amount: 12999, type: '电脑', icon: '💻' }, { name: 'iPad Air', amount: 4799, type: '平板', icon: '📱' }, { name: 'iPhone 15', amount: 5999, type: '手机', icon: '📱' }, { name: '华为手表', amount: 1999, type: '穿戴', icon: '⌚' }, { name: '投影仪', amount: 2899, type: '办公', icon: '📽️' } ] build() { Column() { Text('我的资产') .fontSize(24) .fontWeight(FontWeight.Bold) .margin({ top: 20, bottom: 16 }) List() { ForEach(this.assets, (item, index) => { ListItem() { AssetCard({ name: item.name, amount: item.amount, type: item.type, icon: item.icon, onCardClick: () => { console.info(`点击了：${item.name}`) // 这里可以跳转到详情页（后续课程实现） } }) } }) } .width('100%') .height('100%') } .width('100%') .height('100%') .padding(16) .backgroundColor('#F8F9FA') } }

    🎯 运行效果： 预览器中应该出现5张漂亮的资产卡片，点击卡片会在日志中输出信息。

    📢 成就感时刻： 原来几十行的重复代码，现在只用几行循环 + 组件调用！

    🧪 实训任务三：添加“新增资产”功能（15分钟）

  • 在页面顶部加一个按钮“+ 新增资产”。
  • 点击按钮时，向 assets 数组 push 一个新资产（使用 @State 会自动刷新UI）。
    • // 在 Index.ets 的 build() 中，标题下方添加按钮 Button('+ 新增资产') .width('100%') .height(44) .backgroundColor('#0078D4') .fontColor('#FFFFFF') .borderRadius(22) .margin({ bottom: 16 }) .onClick(() => { const newAsset = { name: `新资产${this.assets.length + 1}`, amount: Math.floor(Math.random() * 10000), type: '新增', icon: '✨' } this.assets = [...this.assets, newAsset] // 触发UI更新 })

    🤔 思考： 为什么用 [...this.assets, newAsset] 而不是 this.assets.push()？因为 @State 需要新数组引用才能检测变化并刷新界面！

    🧪 实训任务四：组件参数默认值与校验（10分钟）

    挑战： 如果调用 AssetCard 时忘记传 amount，能不能给个默认值 0？

    ✅ 我们已经做了！@Prop amount: number = 0 就是默认值。还可以添加更智能的展示：如果金额>10000，金额颜色变成红色（高价值资产）。

    // 在 AssetCard.ets 的 build() 中，动态设置金额颜色 Text(`¥${this.amount.toFixed(2)}`) .fontSize(18) .fontWeight(FontWeight.Bold) .fontColor(this.amount > 10000 ? '#D32F2F' : '#FF6600')

    ✨ 这样，超过1万元的资产会用红色显示，更加醒目！
