🔗 组件间通信 · @Prop与@Link数据同步

🏗️ 父传子 | 子传父 | 双向同步
📅 第4周 · 星期二 第5-6节 (12:30-14:00) ⏱️ 2课时 (80分钟) | 💻 理实一体化 · 实训为主 🎯 授课对象：鸿蒙集训队 · 已完成基础组件学习

🎯 学习目标 · 这堂课我们要拿下什么？

📘 知识点1 @Prop装饰器

学到： @Prop用于父组件向子组件传递数据，子组件接收后可以修改，但修改不会同步回父组件（单向传递）。

💡 类比：就像老师发给大家一份资料，你可以做笔记，但不会改掉老师的原稿。

🛠️ 技能点1 实现@Prop传值

学会： 在子组件中用@Prop定义变量，父组件通过参数传递数据，子组件展示并修改（不影响父组件）。

📘 知识点2 @Link装饰器

学到： @Link实现双向同步，子组件修改数据会同步回父组件，就像两个人共用一个记事本。

🛠️ 技能点2 实现@Link双向绑定

学会： 父组件用$符号传递状态，子组件用@Link接收，任何一方的修改都会同步到另一方。

📘 知识点3 选择策略

学到： 什么时候用@Prop，什么时候用@Link？取决于是否需要子组件修改影响父组件。

🛠️ 技能点3 资产卡片通信实战

学会： 在资产管理系统项目中，用@Link实现“编辑资产”弹窗与主页面数据实时同步。

💡 老师暖心提示： 组件通信是鸿蒙应用开发的“核心内功”，掌握@Prop和@Link，你就能让页面间数据流动起来！

🧩 知识讲解 · 组件间的“情报传递”

1. 场景引入：为什么需要组件通信？

🏢 趣味类比： 把App想象成一家公司，每个组件就是一个部门。

  • 📊 资产列表组件（总经办）：显示所有资产数据。
  • ✏️ 编辑弹窗组件（财务部）：修改某个资产的金额。
  • 💬 需求： 财务部改完数据，总经办的表格要立刻刷新！这就需要“组件通信”。

鸿蒙ArkUI提供了@Prop和@Link两种装饰器来解决这个问题。

2. @Prop装饰器 —— 单向数据流

// 子组件：Child.ets @Component export struct ChildComponent { // 用@Prop接收父组件传过来的值 @Prop message: string = ''; build() { Column() { Text(`子组件收到：${this.message}`) .fontSize(18) Button(`修改消息`) .onClick(() => { // @Prop可以修改，但只影响子组件自己，父组件不会变 this.message = '子组件改过的消息'; }) } } } // 父组件：Parent.ets @Component export struct ParentComponent { @State parentMsg: string = '父组件的原始消息'; build() { Column() { Text(`父组件消息：${this.parentMsg}`) ChildComponent({ message: this.parentMsg }) // 传值 Button(`修改父组件消息`) .onClick(() => { this.parentMsg = '父组件新消息'; }) } } }
特性@Prop 数据流向父 → 子（单向） 子组件能否修改可以，但修改只影响子组件自身，不会传回父组件 适用场景展示型数据、配置项、不需要回传的临时修改 语法要点父组件直接传值，子组件用@Prop接收

3. @Link装饰器 —— 双向同步

// 子组件：LinkChild.ets @Component export struct LinkChild { // 用@Link建立双向绑定 @Link linkMsg: string; build() { Column() { Text(`子组件看到：${this.linkMsg}`) Button(`子组件修改`) .onClick(() => { // 修改会同步回父组件！ this.linkMsg = '子组件改的，父组件也变了'; }) } } } // 父组件：LinkParent.ets @Component export struct LinkParent { @State parentLinkMsg: string = '父组件原始内容'; build() { Column() { Text(`父组件内容：${this.parentLinkMsg}`) // 注意：传递时使用 $ 符号建立双向绑定 LinkChild({ linkMsg: $parentLinkMsg }) Button(`父组件修改`) .onClick(() => { this.parentLinkMsg = '父组件主动修改'; }) } } }
特性@Link 数据流向双向同步（父⇄子） 子组件能否修改可以，且修改会同步回父组件，父组件自动刷新 适用场景表单输入、编辑弹窗、需要双向反馈的交互 语法要点父组件用 $变量名 传递，子组件用 @Link 接收

⚡ 核心口诀： 单向传值用@Prop，双向同步用@Link。父传子，子改父，记得加个$符号！

⚙️ 技能实训 · 动手实现组件通信

✅ 本次实训目标：完成@Prop和@Link两个实战任务，并应用到资产管理项目

🔧 实训一：@Prop单向通信 · 展示学生卡片（15分钟）

  • 新建文件： 在项目中创建 StudentCard.ets 和 PropDemo.ets。
  • 编写子组件 StudentCard： 接收 @Prop name 和 @Prop score，展示姓名和分数，并提供一个“加分”按钮（只改变子组件显示，不影响父组件）。
  • 编写父组件 PropDemo： 定义 @State 学生数据，渲染 StudentCard，并展示父组件的原始分数。
  • 运行验证： 点击子组件的“加分”按钮，观察子组件分数变化，父组件分数是否改变？
    • // StudentCard.ets @Component export struct StudentCard { @Prop name: string; @Prop score: number; build() { Column() { Text(`${this.name} 当前分数：${this.score}`) Button(`+5分（仅子组件变）`) .onClick(() => { this.score += 5; // 只改子组件自己的副本 }) } .padding(10) .border({ width: 1, color: '#ccc' }) .margin(5) } } // PropDemo.ets @Component export struct PropDemo { @State stuName: string = '王小明'; @State stuScore: number = 80; build() { Column() { Text(`【父组件】${this.stuName} 原始分数：${this.stuScore}`) StudentCard({ name: this.stuName, score: this.stuScore }) Button(`父组件改分数`) .onClick(() => { this.stuScore += 10; }) } .padding(20) } }

    🔍 观察结果： 子组件点击“+5分”后，子组件显示的分数增加了，但父组件的分数没变。点击父组件按钮，父组件分数增加，子组件也会同步更新（因为@Prop会接收新值）。这就是“单向”的含义。

    🔧 实训二：@Link双向通信 · 资产编辑弹窗（20分钟）

  • 创建资产编辑弹窗组件： AssetEditDialog.ets，接收 @Link assetName 和 @Link assetValue。
  • 实现编辑界面： 显示两个输入框，修改资产名称和金额，修改实时同步到父组件。
  • 父组件 AssetListPage： 定义一个资产对象（名称、金额），用 $ 语法传递，并实时显示当前资产信息。
  • 验证双向效果： 在弹窗中修改，父组件数据立即刷新！
    • // AssetEditDialog.ets @Component export struct AssetEditDialog { @Link assetName: string; @Link assetValue: number; @Link isVisible: boolean; build() { if (this.isVisible) { Column() { Text(`编辑资产信息`).fontSize(20).fontWeight(FontWeight.Bold) TextInput({ text: this.assetName }) .onChange((value) => { this.assetName = value; }) TextInput({ text: this.assetValue.toString() }) .onChange((value) => { this.assetValue = parseInt(value) || 0; }) Button(`关闭`) .onClick(() => { this.isVisible = false; }) } .padding(20) .backgroundColor('#f5f5f5') .borderRadius(16) } } } // AssetListPage.ets（父组件） @Component export struct AssetListPage { @State assetName: string = '笔记本电脑'; @State assetValue: number = 5999; @State showDialog: boolean = false; build() { Column() { Text(`资产：${this.assetName} 价值：¥${this.assetValue}`) .fontSize(18) Button(`编辑资产`) .onClick(() => { this.showDialog = true; }) if (this.showDialog) { AssetEditDialog({ assetName: $assetName, assetValue: $assetValue, isVisible: $showDialog }) } } .padding(20) } }

    ✨ 关键点： 父组件传递时用 $assetName，子组件用 @Link assetName: string 接收，修改任意一端，另一端都会自动更新。

    🔧 实训四：综合挑战 · 资产清单批量编辑（15分钟）

    🏆 挑战任务： 创建一个资产列表，每个资产项是一个子组件，点击“编辑”按钮弹出对话框，用@Link实现资产数据修改后列表自动刷新。

    提示： 父组件维护一个资产数组，子组件接收 @Link item 对象（需要将整个对象用@Link传递），对话框内修改 item 的字段。
