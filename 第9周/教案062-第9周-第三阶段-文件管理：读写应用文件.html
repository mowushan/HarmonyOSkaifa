🧩 知识讲解 · 文件系统与读写接口

1. 鸿蒙应用沙箱 —— 每个App的“独立小仓库”

🏠 鸿蒙应用运行在沙箱环境中，每个App只能访问自己的私有目录，不能随便翻别人的文件。这样既安全又干净！

常用目录（通过context获取）：

  • context.filesDir → 应用文件目录，存放用户数据，会备份
  • context.cacheDir → 缓存目录，系统可能随时清理，适合放临时文件
  • context.tempDir → 临时目录，应用退出可能被清
  • context.databaseDir → 数据库专用目录（之前我们用的关系型数据库就在这）

📌 实际路径类似：/data/storage/el2/base/haps/entry/files/ 但我们不需要记忆，直接用API获取即可。

2. fileio模块 —— 鸿蒙版“文件操作瑞士军刀”

import fileio from '@ohos.fileio';

// 推荐异步写法，避免阻塞UI线程
try {
    // 打开文件（需要先导入'@ohos.fileio'）
    let file = await fileio.open(path, 0o2);  // 0o2 表示只写模式
    await fileio.write(file.fd, 'hello 鸿蒙');
    await fileio.close(file.fd);
} catch (err) {
    console.error('文件写入失败', err);
}

常用模式常量：

  • 0o0 只读 0o2 只写 0o2 | 0o100 读写+创建(若不存在)
  • 0o100 如果文件不存在则创建

💡 小技巧： 建议使用异步API + try-catch，防止因权限或路径问题导致应用闪退。

3. 读写文本文件完整流程

// ========= 写入文本文件 =========
async function writeToFile(fileName: string, content: string) {
    let context = getContext(this);
    let filePath = context.filesDir + '/' + fileName;
    try {
        let file = await fileio.open(filePath, 0o2 | 0o100);  // 读写+创建
        await fileio.write(file.fd, content);
        await fileio.close(file.fd);
        console.info('写入成功', filePath);
    } catch (e) {
        console.error('写入失败', e);
    }
}

// ========= 读取文本文件 =========
async function readFromFile(fileName: string): Promise<string> {
    let context = getContext(this);
    let filePath = context.filesDir + '/' + fileName;
    try {
        let file = await fileio.open(filePath, 0o0); // 只读
        let stat = await fileio.stat(filePath);
        let buf = new ArrayBuffer(stat.size);
        await fileio.read(file.fd, buf);
        await fileio.close(file.fd);
        return new TextDecoder().decode(buf);
    } catch (e) {
        console.error('读取失败', e);
        return '';
    }
}

🔧 实训一：新建文件操作工具模块

  • 在DevEco Studio中，打开之前的集训项目（若还没有，请新创建一个Empty Ability项目，包名自己定义）。
  • 在 entry/src/main/ets/common/ 目录下新建 FileHelper.ets 文件。
  • 参考上述知识讲解，封装三个方法：
      - writeTextFile(fileName: string, content: string): Promise<void>
      - readTextFile(fileName: string): Promise<string>
      - appendToFile(fileName: string, content: string): Promise<void> (附加写入)
  • 添加必要的import: import fileio from '@ohos.fileio'; 和 import { BusinessError } from '@ohos.base';
    • // FileHelper.ets 关键代码示例
import fileio from '@ohos.fileio';
import { BusinessError } from '@ohos.base';
import { common } from '@kit.AbilityKit';

export class FileHelper {
    private context: common.UIAbilityContext;

    constructor(context: common.UIAbilityContext) {
        this.context = context;
    }

    async writeTextFile(fileName: string, content: string): Promise<void> {
        let filePath = this.context.filesDir + '/' + fileName;
        try {
            let file = await fileio.open(filePath, 0o2 | 0o100);
            await fileio.write(file.fd, content);
            await fileio.close(file.fd);
            console.info(`写入成功: ${filePath}`);
        } catch (err) {
            let error = err as BusinessError;
            console.error(`写入失败 code ${error.code}: ${error.message}`);
            throw err;
        }
    }

    async readTextFile(fileName: string): Promise<string> {
        let filePath = this.context.filesDir + '/' + fileName;
        try {
            let file = await fileio.open(filePath, 0o0);
            let stat = await fileio.stat(filePath);
            let buf = new ArrayBuffer(stat.size);
            await fileio.read(file.fd, buf);
            await fileio.close(file.fd);
            let decoder = new TextDecoder('utf-8');
            return decoder.decode(buf);
        } catch (err) {
            let error = err as BusinessError;
            if (error.code === 13900002) { // 文件不存在
                console.warn('文件不存在，返回默认内容');
                return '';
            }
            console.error(`读取失败: ${error.message}`);
            throw err;
        }
    }

    async appendToFile(fileName: string, content: string): Promise<void> {
        let filePath = this.context.filesDir + '/' + fileName;
        let file = await fileio.open(filePath, 0o2 | 0o100);
        await fileio.write(file.fd, content, {
            offset: (await fileio.stat(filePath)).size
        });
        await fileio.close(file.fd);
    }
}

    🔧 实训二：在资产页面中读写配置（偏好设置）

  • 在主页面的aboutToAppear中初始化FileHelper，并尝试读取配置文件 app_config.txt，若不存在则写入默认配置。
  • 配置文件内容可以设计为JSON格式，比如：{"theme":"light","lastSyncTime":"2025-04-09"}。
  • 添加一个设置按钮，点击后可以修改主题配置（例如切换深色/浅色模式的概念演示），并重新写入文件。
  • 再次打开App时，从文件读取配置并应用（展示日志或弹窗提示）。
    • // Index.ets 中使用 FileHelper
import { FileHelper } from '../common/FileHelper';
import { common } from '@kit.AbilityKit';

@Entry
@Component
struct Index {
    private fileHelper: FileHelper = new FileHelper(getContext(this) as common.UIAbilityContext);
    @State config: string = '';

    async aboutToAppear() {
        try {
            let content = await this.fileHelper.readTextFile('app_config.txt');
            if (content === '') {
                let defaultConfig = JSON.stringify({ theme: 'light', lastSync: '无' });
                await this.fileHelper.writeTextFile('app_config.txt', defaultConfig);
                this.config = defaultConfig;
            } else {
                this.config = content;
            }
            console.info('当前配置：' + this.config);
        } catch (e) {
            console.error('初始化配置文件出错' + e);
        }
    }

    build() {
        Column() {
            Text('当前配置文件内容:').fontSize(18).margin(10)
            Text(this.config).fontSize(14).fontColor('#0078d4').padding(10)
            Button('更新配置（清空同步时间）').onClick(async () => {
                let newConfig = JSON.stringify({ theme: 'light', lastSync: new Date().toLocaleString() });
                await this.fileHelper.writeTextFile('app_config.txt', newConfig);
                this.config = newConfig;
            }).margin(10)
        }.width('100%').height('100%')
    }
}

    🔧 实训三：资产数据导出功能（核心亮点）

    📎 结合之前数据库中的资产数据，实现“导出资产清单”文本文件功能，把数据库里所有资产的名称、价格、购买日期写入到 assets_export.txt 文件。

  • 在我们已有的资产管理页面添加一个“导出到文件”按钮。
  • 从数据库中查询所有资产（使用关系型数据库的query方法）。
  • 将查询结果拼装为可读文本（例如每行格式：“笔记本 | 5999 | 2025-01-01”）。
  • 调用FileHelper的writeTextFile，存储为 assets_export.txt，并弹出提示“导出成功，文件位于应用files目录”。
  • 附加挑战：实现追加导出（不覆盖原文件），每次导出时在文件末尾加上时间戳和新的数据。
    • // 伪代码示例
async exportAssetsToFile() {
    // 假设已有数据库helper
    let assets = await this.dbHelper.queryAllAssets();  // 返回Asset[]
    let content = '资产清单导出时间：' + new Date().toLocaleString() + '\n';
    content += '-----------------------------------\n';
    assets.forEach(asset => {
        content += `名称：${asset.name} | 价格：${asset.price} | 日期：${asset.date}\n`;
    });
    await this.fileHelper.writeTextFile('assets_export.txt', content);
    promptAction.showToast({ message: '导出成功！' });
}
