🧩 知识讲解 · App的“上网许可证”

1. 权限是什么？—— App的“通行证”

🔑 趣味类比： 想象你的手机是一个小区，每个App都是访客。权限就是“门禁卡”，网络权限就是“小区大门的钥匙”。没有钥匙，App只能待在自家屋里，无法访问外面的世界（互联网）。

鸿蒙权限分类（简单记）：

  • 🟢 normal（普通权限）：风险较低，安装时自动授权，比如震动、访问网络（部分版本）。
  • 🟡 system_basic（基础敏感）：需要用户弹窗同意，比如访问相册、位置。
  • 🔴 system_core（核心敏感）：只有系统应用可用，我们暂时不用管。

📌 记住： 网络权限在HarmonyOS NEXT中属于normal权限，但必须在配置文件中声明！

2. 如何声明网络权限？—— module.json5 配置

每个鸿蒙应用都有一个“身份证”文件 module.json5，权限声明就写在这里面。

{
  "module": {
    "name": "entry",
    "type": "entry",
    "srcEntry": "./ets/pages/Index.ets",
    "requestPermissions": [
      {
        "name": "ohos.permission.INTERNET",
        "reason": "$string:internet_reason",
        "usedScene": {
          "abilities": ["EntryAbility"],
          "when": "inuse"
        }
      }
    ]
  }
}

重点解释：

  • ohos.permission.INTERNET：网络权限的名称，固定写法。
  • reason：告诉用户为什么要这个权限（需要在资源文件里定义字符串）。
  • usedScene：在哪些Ability中使用，何时使用。

3. 发起网络请求 —— 使用 http 模块

// 导入http模块
import http from '@ohos.net.http';

// 创建一个http请求对象
let httpRequest = http.createHttp();

// 发起GET请求
httpRequest.request(
  "https://api.example.com/data",  // 请求地址
  {
    method: http.RequestMethod.GET,
    header: { 'Content-Type': 'application/json' },
    connectTimeout: 60000,
    readTimeout: 60000
  },
  (err, data) => {
    if (!err) {
      // 请求成功，data.responseCode是状态码，data.result是返回的数据
      console.info('请求成功，状态码：' + data.responseCode);
      console.info('返回数据：' + data.result);
    } else {
      console.error('请求失败：' + JSON.stringify(err));
    }
  }
);

⚠️ 注意： 网络请求是异步操作，需要在回调里处理结果。新版鸿蒙推荐使用Promise或async/await，今天先掌握回调方式。

⚙️ 技能实训 · 手把手实现联网功能

✅ 实训目标：给资产管理系统添加“获取云端公告”功能

📁 准备工作

1. 打开之前集训的资产管理系统项目（或新建一个Demo项目）。

2. 确保DevEco Studio已打开，真机/模拟器已连接。

3. 本实训我们将调用一个免费的公共API：https://jsonplaceholder.typicode.com/posts/1（返回一篇模拟文章）。

🧪 实训任务一：声明网络权限（10分钟）

  • 在项目目录中找到 entry/src/main/module.json5 文件。
  • 在 module 对象内，找到（或添加）requestPermissions 数组。
  • 添加网络权限配置，参考上面代码块。如果没有 string.json 中的 internet_reason，可以先删掉 reason 字段（简化版）。
  • ✅ 验证： 编译项目，没有报错即成功。

    • 🔧 常见错误：忘记加权限直接请求网络，应用会闪退或请求失败。权限声明是第一步！

    🧪 实训任务二：编写联网工具类（20分钟）

  • 在 ets 目录下新建一个文件夹 common，新建文件 HttpUtil.ets。
  • 编写一个简单的GET请求函数： 
    import http from '@ohos.net.http';

export class HttpUtil {
  static get(url: string): Promise<string> {
    return new Promise((resolve, reject) => {
      let httpRequest = http.createHttp();
      httpRequest.request(url, {
        method: http.RequestMethod.GET,
        connectTimeout: 10000,
        readTimeout: 10000
      }, (err, data) => {
        if (!err && data.responseCode === 200) {
          resolve(data.result);
        } else {
          reject(err || `HTTP ${data.responseCode}`);
        }
        httpRequest.destroy(); // 及时销毁
      });
    });
  }
}
  • 解释： 封装成Promise方便后续使用 async/await。

    • 🧪 实训任务三：在页面中调用并显示结果（25分钟）

  • 打开资产列表页 Index.ets 或新建一个测试页 NetworkTestPage.ets。
  • 添加状态变量存储返回数据：@State responseData: string = '';
  • 添加一个按钮，点击后发起请求： 
    Button('获取云端公告')
  .onClick(async () => {
    try {
      let result = await HttpUtil.get('https://jsonplaceholder.typicode.com/posts/1');
      this.responseData = result;
      console.info('请求成功', result);
    } catch (err) {
      this.responseData = '请求失败：' + JSON.stringify(err);
      console.error('请求失败', err);
    }
  })
  • 添加一个 Text 组件显示 responseData。
  • 运行测试： 点击按钮，观察是否出现返回的JSON数据。如果请求失败，检查设备网络和权限。

    • 🎉 恭喜！ 你的App已经能上网了！哪怕只是获取一条公告，都意味着我们的资产管理系统可以对接云端数据了。
