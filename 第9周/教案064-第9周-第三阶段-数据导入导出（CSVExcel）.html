🧩 知识讲解 · 数据导入导出秘籍

1. CSV格式 —— 数据界的“普通话”

📄 趣味引入： 想象一下，如果每个软件都说自己的“方言”，数据就没法交流了。CSV就是大家都懂的“普通话”！

CSV长这样：

资产编号,资产名称,分类,购买日期,金额
A001,笔记本电脑,电子设备,2025-01-10,5999
A002,投影仪,办公设备,2025-02-15,3200
  • 第一行叫“表头”，告诉每列是什么。
  • 后面每行是一条资产记录。
  • 用英文逗号分隔，所以叫Comma-Separated Values。

📌 优点：简单、通用、任何表格软件都能打开（Excel/WPS/记事本）。

2. 鸿蒙中操作文件 · 核心API

鸿蒙提供了强大的文件管理模块，今天主要用到：

API作用
fileIo.openSync()打开文件
fileIo.writeSync()写入数据
fileIo.readText()读取文本文件
fileIo.accessSync()检查文件是否存在

🔑 关键点： 需要申请ohos.permission.WRITE_USER_STORAGE和ohos.permission.READ_USER_STORAGE权限，并在module.json5中配置。

3. 导入导出流程图解

📤 导出流程

查询数据库 → 组装CSV字符串 → 创建文件 → 写入 → 提示成功

📥 导入流程

选择文件 → 读取内容 → 解析CSV → 校验数据 → 插入数据库

    • 🔧 实训二：导出CSV功能实现（25分钟）

    需求： 点击“导出备份”按钮，将所有资产导出到 /data/storage/el2/base/haps/entry/files/assets_备份.csv

  • 在数据库工具类（例如 DatabaseHelper.ets）中添加查询所有资产的方法。
  • 编写导出函数 exportToCSV(context: Context)： 
    import { fileIo } from '@kit.CoreFileKit';
import { BusinessError } from '@kit.BasicServicesKit';

async exportToCSV(context: Context) {
  // 1. 查询所有资产
  let assets = await this.getAllAssets();
  if (assets.length === 0) {
    console.info("没有数据可导出");
    return;
  }
  
  // 2. 构建CSV表头
  let csvContent = "资产编号,资产名称,分类,购买日期,金额\n";
  
  // 3. 遍历资产，构建数据行
  for (let item of assets) {
    csvContent += `${item.code},${item.name},${item.category},${item.date},${item.amount}\n`;
  }
  
  // 4. 获取应用私有目录
  let filesDir = context.filesDir;
  let filePath = filesDir + "/assets_backup.csv";
  
  // 5. 写入文件
  try {
    let file = fileIo.openSync(filePath, fileIo.OpenMode.CREATE | fileIo.OpenMode.WRITE_ONLY);
    fileIo.writeSync(file.fd, csvContent);
    fileIo.closeSync(file);
    console.info("导出成功！文件路径：" + filePath);
  } catch (err) {
    let error = err as BusinessError;
    console.error("导出失败：" + error.message);
  }
}
  • 在页面中添加“导出备份”按钮，调用 exportToCSV，并用promptAction.showToast提示成功。

    • ✅ 学生完成后，互相验证：导出文件后在手机文件管理器中能否找到.csv文件，用文本打开是否格式正确。

    🔧 实训三：导入CSV功能实现（30分钟）

  • 使用 fileIo.readText 读取CSV文件内容。
  • 编写CSV解析函数，处理逗号分隔和换行问题。
  • 编写导入函数 importFromCSV(context: Context, fileUri: string)： 
    async importFromCSV(context: Context, fileUri: string) {
  try {
    // 1. 读取文件内容
    let content = await fileIo.readText(fileUri);
    let lines = content.split('\n');
    if (lines.length < 2) {
      console.info("文件为空");
      return;
    }
    
    // 2. 跳过表头（第0行）
    let successCount = 0;
    for (let i = 1; i < lines.length; i++) {
      let line = lines[i].trim();
      if (line === "") continue;
      
      let parts = line.split(',');
      if (parts.length >= 5) {
        let asset = {
          code: parts[0],
          name: parts[1],
          category: parts[2],
          date: parts[3],
          amount: parseFloat(parts[4])
        };
        // 3. 插入数据库（需要先查重，避免重复导入）
        await this.insertAsset(asset);
        successCount++;
      }
    }
    console.info(`导入完成，成功导入 ${successCount} 条记录`);
  } catch (err) {
    let error = err as BusinessError;
    console.error("导入失败：" + error.message);
  }
}

    🐛 常见Bug与排错指南

    ❌ 导出失败

    原因： 权限未配置或未申请。
    解决： 检查module.json5权限，运行时动态弹窗授权。

    ❌ 导入中文乱码

    原因： CSV文件编码不是UTF-8。
    解决： 导出时强制使用UTF-8编码，或用记事本另存为UTF-8。

    ❌ 数据重复导入

    原因： 没有按资产编号查重。
    解决： 导入前先查询数据库，若已存在则跳过或更新。
