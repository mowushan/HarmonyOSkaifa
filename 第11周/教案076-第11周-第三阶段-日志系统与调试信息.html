🧩 知识讲解 · 让程序“开口说话”

1. 为什么需要日志？—— 程序员的“行车记录仪”

🎬 趣味场景： 你的App突然闪退，用户只看到“应用无响应”。你不知道哪里出错，就像蒙着眼睛走路。日志就是程序的“行车记录仪”，记录每一帧发生了什么。

没有日志的调试： 猜谜 → 改代码 → 运行 → 还是错 → 崩溃。

有日志的调试： 看日志 → 定位行号 → 修复 → 搞定！

2. 鸿蒙hilog系统 · 官方标准日志

📊 日志级别（由低到高）

  • DEBUG - 调试细节，开发时用
  • INFO - 一般信息，如“登录成功”
  • WARN - 警告，可能有问题但不影响运行
  • ERROR - 错误，功能异常
  • FATAL - 致命错误，应用可能崩溃

📦 hilog基本用法

import hilog from '@ohos.hilog'; // 领域ID: 0x0001, 标签: "MyApp" hilog.info(0x0001, "MyApp", "用户登录成功，userId=%{public}d", userId); hilog.error(0x0001, "MyApp", "网络请求失败，错误码=%{public}d", errCode);

3. console对象 · 快速调试小助手

// 最简单直接的方法，适合快速验证 console.log("普通信息: " + message); console.warn("警告: 数组长度为空"); console.error("错误: 数据格式不正确"); console.debug("调试: 变量值=", JSON.stringify(obj));

⚡ 区别对比： console简单方便，适合临时调试；hilog更强大，支持级别过滤、领域分类、生产环境保留，正式项目推荐用hilog。

4. 调试策略 · 高手秘诀

  • 📍 关键节点埋点：页面生命周期（onPageShow/onHide）、按钮点击、网络请求前后、数据存储成功/失败。
  • 📍 变量快照：条件分支入口打印关键变量值。
  • 📍 性能日志：记录耗时操作开始/结束，计算时间差。
  • 📍 异常捕获日志：try-catch中打印错误栈。

⚙️ 技能实训 · 动手打造调试利器

✅ 实训目标：在资产管理项目中植入完整日志体系，能通过日志快速定位问题

🔧 实训一：hilog初体验（15分钟）

  • 打开之前的资产管理项目，在 entry/src/main/ets/pages/Index.ets 顶部导入hilog：
    import hilog from '@ohos.hilog';
  • 在aboutToAppear()生命周期中添加日志：
    aboutToAppear() { hilog.info(0x0001, "AssetApp", "页面加载完成，当前时间戳=%{public}d", Date.now()); console.log("Index页面已显示"); }
  • 运行应用，打开DevEco Studio底部的“Log”面板，选择自己的设备，筛选标签“AssetApp”。观察输出的日志。
  • 尝试不同级别：将info改成warn/error，观察日志颜色变化（warn黄色，error红色）。

    • 💡 老师巡导： 提醒学生注意日志区域可以通过“Level”下拉菜单过滤级别，查看特定级别日志。

    🔧 实训二：资产录入页面完整埋点（25分钟）

  • 打开资产录入页（AssetEntry.ets），在以下位置添加日志：
    • 页面初始化：打印“资产录入页打开”
    • 输入框变化：打印当前输入内容（可选）
    • 点击“保存”按钮：打印“准备保存资产数据”
    • 数据验证通过后：打印“验证通过，开始存储”
    • 存储成功/失败：分别打印info/error日志
  • 参考代码片段：
    // 保存按钮点击事件 saveAsset() { hilog.info(0x0001, "AssetEntry", "用户点击保存，资产名称=%{public}s", this.assetName); if (!this.assetName) { hilog.warn(0x0001, "AssetEntry", "资产名称为空，提示用户"); promptAction.showToast({ message: "请输入资产名称" }); return; } try { // 模拟存储逻辑 hilog.info(0x0001, "AssetEntry", "开始存储资产到数据库"); // ... 实际存储代码 hilog.info(0x0001, "AssetEntry", "存储成功"); } catch (err) { hilog.error(0x0001, "AssetEntry", "存储失败：%{public}s", JSON.stringify(err)); } }
  • 故意制造一个错误（比如数据库未初始化），运行并观察错误日志如何显示。

    • 📝 小技巧： 使用%{public}s格式化字符串，敏感信息可以用%{private}s自动脱敏。

    🔧 实训三：Log过滤器与搜索挑战（15分钟）

  • 在资产列表页添加循环打印10条日志（模拟大量日志输出）。
  • 打开Log面板，尝试以下过滤方式：
    • 按标签过滤：输入“AssetApp”只显示你的应用日志
    • 按级别过滤：只显示ERROR级别
    • 搜索关键词：如“存储成功”
  • 小组PK： 每组在代码里故意埋一个带特殊标记的日志（比如 // FLAG_20241111），让其他组最快找到这个日志特征，最快找到的获胜。

    • 🔧 实训四：性能日志 · 监控应用启动耗时（15分钟）

  • 在EntryAbility的onCreate()和onWindowStageCreate()开始结束位置记录时间戳。
    // EntryAbility.ets onCreate(want, launchParam) { hilog.info(0x0001, "Performance", "onCreate开始"); AppStorage.setOrCreate('appStartTime', Date.now()); } onWindowStageCreate(windowStage) { let startTime = AppStorage.get('appStartTime'); let cost = Date.now() - startTime; hilog.info(0x0001, "Performance", "启动耗时=%{public}d ms", cost); }
  • 多次冷启动（杀掉进程重新打开），观察启动耗时变化，记录下来与队友讨论哪些阶段最耗时。
