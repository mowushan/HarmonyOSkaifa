🧩 知识讲解 · 优化就像给App“健身”

1. 为什么App会卡？—— 找对“病根”

🐌 趣味引入： 想象一下，如果你要从1000本书里找一本书，没有目录（索引），只能一本一本翻，是不是要累死？App也一样！

三大常见“卡顿元凶”：

  • 📜 一次性加载太多数据：比如你的资产列表有1万条，全加载到内存，手机直接“喘不过气”。
  • 🔍 数据库查询慢：没有索引，每次查询都要“扫描全表”。
  • 🖼️ 内存管理不当：图片不释放、变量不回收，像垃圾越堆越多。

✅ 解决思路： 分页加载 + 索引优化 + 及时释放！

2. 列表懒加载（LazyForEach）—— 省着用才流畅

// 错误示范：一次性加载所有数据（❌ 卡！） List() { ForEach(this.allAssets, (item: Asset) => { ListItem() { Text(item.name) } }) } // 正确示范：懒加载（✅ 丝滑！） // 1. 定义数据源类，实现IDataSource接口 class AssetDataSource implements IDataSource { private dataArray: Asset[] = []; constructor(assets: Asset[]) { this.dataArray = assets; } totalCount(): number { return this.dataArray.length; } getData(index: number): Asset { return this.dataArray[index]; } // 注册数据监听（实现懒加载刷新） registerDataChangeListener(listener: DataChangeListener): void {} unregisterDataChangeListener(listener: DataChangeListener): void {} } // 2. 使用LazyForEach代替ForEach List() { LazyForEach(this.assetDataSource, (item: Asset, index: number) => { ListItem() { Text(`${index+1}. ${item.name}`) .fontSize(16) .padding(12) } }, (item: Asset, index: number) => index.toString()) }

📌 重点： LazyForEach 只渲染屏幕上显示的那几条数据，滚动时动态创建销毁，内存占用极低！

3. 数据库索引 —— 让查询快100倍

📖 索引原理

索引就像书的目录，帮你快速定位数据。没有索引，数据库只能“全表扫描”。

🎯 什么时候加索引？

经常用于 WHERE、ORDER BY、GROUP BY 的字段。

// SQLite 创建索引语法 CREATE INDEX idx_asset_name ON assets (name); CREATE INDEX idx_asset_category_price ON assets (category, price); // 在鸿蒙关系型数据库中执行SQL let sql = "CREATE INDEX IF NOT EXISTS idx_asset_name ON assets (name)"; let result = await rdbStore.executeSql(sql);

📊 效果对比： 10万条数据，不加索引查询需2-5秒，加了索引只需0.05秒！

4. 内存优化小技巧

  • 图片回收： 使用完大图后调用 imagePixelMap?.release()。
  • 及时释放监听器： 页面销毁时移除事件监听。
  • 避免循环引用： 匿名函数里使用变量要注意闭包。
  • 使用DevEco Profiler： 打开DevEco Studio → View → Tool Windows → Profiler，实时看内存曲线。
