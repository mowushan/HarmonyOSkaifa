🧩 知识讲解 · 测试与调试心法

1. 为什么需要测试？—— 预防针比感冒药更管用

🏥 趣味比喻： 如果医院没有体检中心，病人都是病倒了才来，医生会累死。测试就是代码的“体检中心”，提前发现问题。

三种测试级别（我们重点做前两个）：

  • 🔬 单元测试： 测试一个函数、一个方法，比如测试“保存资产”函数单独运行是否正确。
  • 🔗 集成测试： 测试多个模块配合，比如“录入资产 → 保存到数据库 → 刷新列表”整个流程。
  • 🌐 端到端测试： 模拟用户真实操作，后续再学。

2. 调试工具箱 —— 日志 + 断点

📝 日志大法

在代码关键位置打印变量值，观察程序执行轨迹。

// ArkTS 日志示例 console.info('开始保存资产，数据：' + JSON.stringify(assetData)); console.error('保存失败，错误码：' + err.code);

⏸️ 断点调试

让程序运行到指定行暂停，逐行执行，实时查看变量。

// 在DevEco Studio中点击行号左侧设置红点 let result = await database.insert(asset); // ← 断点 console.info('插入结果：' + result);

3. 鸿蒙数据模块常见Bug及解决方案

Bug表现可能原因解决方法
保存资产后列表不刷新忘记重新查询数据或状态未更新保存成功后重新调用loadData()，并更新@State变量
编辑资产后数据未修改SQL语句写错或者主键条件遗漏检查UPDATE语句的WHERE条件，打日志对比ID
删除资产点取消也删除了弹窗异步回调处理错误检查AlertDialog的confirm/取消逻辑，只在confirm时删除
数据库报错“no such table”数据库版本升级未执行建表语句检查onCreate或onUpgrade，确保表已创建
字符串转数字失败导致排序错误价格、数量字段存储为字符串存储时转为number，或查询时用CAST转换

🧪 实训任务一：日志调试 · 给代码装上“行车记录仪”（15分钟）

  • 任务1.1 添加日志： 在保存资产函数开头和结尾添加console.info，打印“开始保存”和“保存成功，ID=xxx”。
  • 任务1.2 错误日志： 在try-catch块中捕获异常，用console.error输出错误信息。
  • 任务1.3 观察输出： 运行应用，录入一条资产，切换到HiLog或终端面板查看日志输出。
    • // 示例：增加日志的保存函数 async saveAsset(asset: Asset) { console.info(`[测试] 开始保存资产，名称：${asset.name}`); try { let id = await dataService.insert(asset); console.info(`[测试] 保存成功，生成ID：${id}`); return id; } catch (err) { console.error(`[测试] 保存失败：${JSON.stringify(err)}`); return -1; } }

    🧪 实训任务二：断点调试实战（20分钟）

  • 设置断点： 在loadAssetList()函数的第一行左侧单击，出现红点。
  • 启动调试： 点击甲虫图标“Debug”，应用启动后在界面触发加载列表（例如进入主页）。
  • 单步执行： 程序停在断点处，按F10逐行执行，观察右侧“变量”面板中数据库返回的数据。
  • 条件断点（进阶）： 右键断点，设置条件“assetId===5”，只有ID为5时才中断，方便排查特定数据。

    • 🎯 小组挑战：每组找出一条“有问题的数据”（比如名称特别长、价格为负数），用断点观察程序如何处理，并讨论是否合理。
