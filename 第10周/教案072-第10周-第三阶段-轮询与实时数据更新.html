🧩 知识讲解 · 轮询让数据“活”起来

1. 为什么要轮询？—— 资产更新场景

📊 真实场景： 资产管理系统中，可能多名管理员同时录入/借出资产。如果不刷新，A同学录入了新资产，B同学看不到，就会造成数据不一致。

轮询方案： 每隔3~5秒，自动向服务器请求一次最新资产列表，有变化就更新界面。

🔁 轮询比喻：就像你每隔几秒刷新一次班级群，看有没有新消息。

2. 鸿蒙中的定时器：setInterval / setTimeout

// 鸿蒙（ArkTS）中使用定时器 let timerId: number = setInterval(() => { console.info("每3秒执行一次"); // 在这里请求最新资产数据 }, 3000); // 清除定时器（非常重要！） clearInterval(timerId);

关键点： 页面销毁时一定要清除，否则定时器会一直在后台运行，浪费资源甚至导致崩溃。

3. 轮询完整流程 + 生命周期配合

aboutToAppear
页面即将显示时 → 启动轮询定时器
定时器回调
触发网络请求 → 获取数据 → 更新@State变量 → 界面自动刷新
onPageHide
页面不可见时（如跳转详情）→ 清除定时器，停止轮询
onPageShow
页面重新可见时 → 重新启动轮询

⚙️ 技能实训 · 一步步实现资产实时刷新

✅ 实训目标：给资产列表页增加轮询，每隔3秒自动更新

🔧 实训一：准备模拟数据接口（无后端模拟）

  • 在项目中创建模拟数据服务：新建文件 mockAssetService.ets，模拟从服务器获取资产列表。
  • 编写一个返回随机资产数量的函数，模拟数据变化。
  • 参考代码：↓
    • // mockAssetService.ets export class MockAssetService { // 模拟从服务器获取资产列表，每次调用可能数据不同 static async fetchAssetList(): Promise> { // 模拟网络延迟 await new Promise(resolve => setTimeout(resolve, 300)); // 模拟随机增加/修改资产 const baseAssets = [ { id: '1', name: '笔记本电脑', status: '在库' }, { id: '2', name: '投影仪', status: '借用中' }, { id: '3', name: '开发板套件', status: '在库' } ]; // 随机改变某个资产的状态，模拟其他用户操作 const randomIndex = Math.floor(Math.random() * baseAssets.length); const statuses = ['在库', '借用中', '维修中']; baseAssets[randomIndex].status = statuses[Math.floor(Math.random() * statuses.length)]; // 随机增加一条新资产（模拟新增） if (Math.random() > 0.7) { const newId = (baseAssets.length + 1).toString(); baseAssets.push({ id: newId, name: `临时资产${newId}`, status: '在库' }); } return baseAssets; } } interface AssetItem { id: string; name: string; status: string; }

    💡 这个模拟服务会每隔几秒返回不同的数据，就跟真实的后端一样！等以后有真后端，只需替换请求URL即可。

    🔧 实训二：编写资产列表页，增加轮询

  • 打开 AssetListPage.ets（上一节课已经完成列表展示）。
  • 导入模拟服务：import { MockAssetService } from './mockAssetService';
  • 定义状态变量 @State assetList: AssetItem[] = [];
  • 定义定时器ID：private timerId?: number;
  • 编写加载数据函数 loadAssets()，调用Mock服务并更新assetList。
  • 在aboutToAppear中启动轮询：this.startPolling();
  • 实现startPolling: 先清除旧定时器，再用setInterval每3秒调用loadAssets。
  • 在onPageHide中清除定时器，在onPageShow中重启。
    • // AssetListPage.ets 核心代码片段 import { MockAssetService, AssetItem } from './mockAssetService'; @Entry @Component struct AssetListPage { @State assetList: AssetItem[] = []; private timerId?: number; aboutToAppear() { this.startPolling(); } onPageHide() { this.stopPolling(); } onPageShow() { this.startPolling(); } startPolling() { if (this.timerId) { clearInterval(this.timerId); } // 立即加载一次 this.loadAssets(); // 每隔3秒轮询 this.timerId = setInterval(() => { this.loadAssets(); }, 3000); } stopPolling() { if (this.timerId) { clearInterval(this.timerId); this.timerId = undefined; } } async loadAssets() { try { const newList = await MockAssetService.fetchAssetList(); this.assetList = newList; console.info(`轮询刷新成功，共 ${newList.length} 条资产`); } catch (error) { console.error(`加载资产失败: ${error}`); } } build() { Column() { Text(`资产总数: ${this.assetList.length}`) .fontSize(18) .margin(12); List() { ForEach(this.assetList, (item: AssetItem) => { ListItem() { Row() { Text(item.name) .fontSize(16) .layoutWeight(1); Text(item.status) .fontSize(14) .fontColor(item.status === '在库' ? '#28a745' : '#dc3545') } .padding(12) .width('100%') } }) } .layoutWeight(1) } .width('100%') .height('100%') } }

    🧪 动手验证： 运行应用，观察资产列表每3秒自动变化一次（状态或数量变化）。打开“控制台”可以看到打印的刷新日志。

    🔧 实训三：优化——只在页面可见时轮询

  • 上面代码已经利用 onPageShow / onPageHide 控制了轮询的启停。
  • 让学生测试：跳转到详情页再返回，观察轮询是否恢复。
  • 挑战改进： 增加“手动刷新”按钮，让用户也可以主动下拉刷新。
    • // 手动刷新按钮 Button("手动刷新") .onClick(() => { this.loadAssets(); }) .margin(10)

    🔧 实训四：避开轮询陷阱 · 防止重复请求

    坑点1： 如果上一次请求还没回来，下一次轮询又发起，会造成并发冲突。

    解决方案： 加一个标志位 isLoading，请求中时跳过本次轮询。

    private isLoading: boolean = false; async loadAssets() { if (this.isLoading) { console.info("上一次请求未完成，跳过本次轮询"); return; } this.isLoading = true; try { const newList = await MockAssetService.fetchAssetList(); this.assetList = newList; } finally { this.isLoading = false; } }
