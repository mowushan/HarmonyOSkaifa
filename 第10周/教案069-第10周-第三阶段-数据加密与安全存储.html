🧩 知识讲解 · 为什么需要加密？

1. 真实风险 & 鸿蒙安全解决方案

⚠️ 情景剧场： 假如我们的资产管理系统存储了设备价值、购买日期、甚至维修合同。如果没有加密，手机被root或数据文件被拷贝，敏感信息就泄露了！

鸿蒙提供了两大武器：

  • 🔑 cryptoFramework —— 提供AES、RSA、SHA等主流加解密算法。
  • 🔒 安全存储区(Keystore) —— 用于保存密钥，防止逆向提取。

今天我们重点掌握AES对称加密+Preferences安全存储。

2. AES对称加密 + 哈希原理

🔐 AES对称加密

加密和解密使用同一个密钥，适合大量数据加密。我们选用AES-128-GCM模式（兼具加密+完整性校验）。

🪄 哈希（SHA-256）

单向函数，不能解密。用于存储密码hash，登录时比对hash而不是明文密码。

⚙️ 技能实训 · 一步一步实现加密模块

✅ 实训目标：手写通用加解密工具类 + 集成至资产管理设置页

📁 准备：新建加密工具文件

  • 打开鸿蒙项目（DevEco Studio），在 entry/src/main/ets/common/utils/ 目录下新建文件 CryptoHelper.ets。
  • 如果目录不存在，手动创建 utils 包。
  • 在 oh-package.json5 确保已引入必要的SDK模块（cryptoFramework 为系统内置，无需额外安装）。

    • 🔧 实训一：编写CryptoHelper工具类（完整代码）

    // CryptoHelper.ets import cryptoFramework from '@ohos.security.cryptoFramework'; import util from '@ohos.util'; export class CryptoHelper { // AES-GCM 模式固定参数 private static readonly AES_ALG = 'AES128|GCM|NoPadding'; private static readonly GCM_TAG_LEN = 16; /** * 生成随机密钥（用于AES加密，实际项目中也可以固定密钥并从密钥库读取） * 本demo为了演示，生成固定密钥后base64保存（生产环境应使用安全密钥库） */ static async generateAesKey(): Promise<string> { let symKeyGenerator = cryptoFramework.createSymKeyGenerator('AES128'); let symKey = await symKeyGenerator.generateSymKey(); let encodedKey = await symKey.getEncoded(); let base64Helper = new util.Base64Helper(); return base64Helper.encodeToString(encodedKey.data); } /** * AES-GCM 加密 * @param plainText 明文 * @param keyBase64 base64格式密钥 * @returns 返回 { cipherText, iv, authTag } 的base64串拼接 */ static async encrypt(plainText: string, keyBase64: string): Promise<string> { let base64Helper = new util.Base64Helper(); let keyData = base64Helper.decodeSync(keyBase64); let symKey = await cryptoFramework.createSymKeyGenerator('AES128') .convertKey({ data: keyData }); // 生成随机IV (12字节) let ivArray = new Uint8Array(12); for (let i = 0; i < ivArray.length; i++) { ivArray[i] = Math.floor(Math.random() * 256); } let ivData = { data: ivArray }; let alg = 'AES128|GCM|NoPadding'; let cipher = cryptoFramework.createCipher(alg); let gcmParams: cryptoFramework.GcmParamsSpec = { iv: ivData, aad: { data: new Uint8Array(0) }, authTag: { data: new Uint8Array(16) }, tagLen: this.GCM_TAG_LEN }; await cipher.init(cryptoFramework.CryptoMode.ENCRYPT_MODE, symKey, gcmParams); let plainData = { data: new Uint8Array(base64Helper.decodeSync(plainText)) }; let encryptOut = await cipher.update(plainData); let authTag = await cipher.doFinal(null); // 拼接 iv(base64) + authTag(base64) + cipherText(base64) let ivB64 = base64Helper.encodeToString(ivArray); let tagB64 = base64Helper.encodeToString(authTag.data.data); let cipherB64 = base64Helper.encodeToString(encryptOut.data.data); return `${ivB64}:${tagB64}:${cipherB64}`; } /** * AES-GCM 解密 * @param encryptedPack 格式： iv:authTag:cipherText * @param keyBase64 密钥 * @returns 明文 */ static async decrypt(encryptedPack: string, keyBase64: string): Promise<string> { let parts = encryptedPack.split(':'); if (parts.length !== 3) throw new Error('Invalid encrypted pack'); let base64Helper = new util.Base64Helper(); let ivData = base64Helper.decodeSync(parts[0]); let authTagData = base64Helper.decodeSync(parts[1]); let cipherData = base64Helper.decodeSync(parts[2]); let keyRaw = base64Helper.decodeSync(keyBase64); let symKey = await cryptoFramework.createSymKeyGenerator('AES128') .convertKey({ data: keyRaw }); let gcmParams: cryptoFramework.GcmParamsSpec = { iv: { data: ivData }, aad: { data: new Uint8Array(0) }, authTag: { data: authTagData }, tagLen: this.GCM_TAG_LEN }; let cipher = cryptoFramework.createCipher('AES128|GCM|NoPadding'); await cipher.init(cryptoFramework.CryptoMode.DECRYPT_MODE, symKey, gcmParams); let decryptData = await cipher.update({ data: cipherData }); await cipher.doFinal(null); let plainB64 = base64Helper.encodeToString(decryptData.data.data); return base64Helper.decodeSync(plainB64).toString(); } /** * 哈希（SHA-256）密码 */ static async hashPassword(password: string): Promise<string> { let md = cryptoFramework.createMd('SHA256'); await md.update({ data: new Uint8Array(new util.TextEncoder().encode(password)) }); let hashResult = await md.digest(); let base64Helper = new util.Base64Helper(); return base64Helper.encodeToString(hashResult.data); } }
    📎 代码说明：上述类封装了AES-GCM加解密和SHA256哈希，支持我们后续对敏感字段加密存储。

    🔧 实训二：在资产设置页集成加密（保存用户Token示例）

  • 在 pages/SettingsPage.ets 中导入 CryptoHelper。
  • 添加一个“安全测试”按钮：加密字符串“我的资产总额10000”，解密并Toast显示。
  • 将加密后的结果存储到Preferences中，重启应用后读取并解密验证。
    • // SettingsPage.ets 片段 import { CryptoHelper } from '../common/utils/CryptoHelper'; import preferences from '@ohos.data.preferences'; @Entry @Component struct SettingsPage { @State encryptResult: string = ''; @State decryptResult: string = ''; private prefs: preferences.Preferences; async aboutToAppear() { this.prefs = await preferences.getPreferences(this.context, 'secure_store'); } async testEncrypt() { let plain = "admin:123456"; // 实际项目中应提前存储或生成密钥 let base64Key = "dGVzdEtleTEyMzQ1Njc4OTBhYmNkZWY="; // 示例base64密钥（32字节解码后16字节） let encrypted = await CryptoHelper.encrypt(plain, base64Key); this.encryptResult = encrypted; await this.prefs.put('encrypted_cred', encrypted); await this.prefs.flush(); } async testDecrypt() { let base64Key = "dGVzdEtleTEyMzQ1Njc4OTBhYmNkZWY="; let encrypted = await this.prefs.get('encrypted_cred', ''); if (encrypted) { let plain = await CryptoHelper.decrypt(encrypted, base64Key); this.decryptResult = plain; } } build() { Column() { Button('加密并存储敏感数据').onClick(() => this.testEncrypt()) Text('加密结果: ' + this.encryptResult) Button('解密验证').onClick(() => this.testDecrypt()) Text('解密结果: ' + this.decryptResult) }.padding(20) } }

    🔧 实训三：密码哈希校验（不存明文）

    在用户登录/设置密码时，不要直接存密码字符串，而应该存SHA256哈希值。验证时对比哈希值。

    async function savePasswordHash(pwd: string) { let hash = await CryptoHelper.hashPassword(pwd); // 存入 Preferences 或数据库 await prefs.put('pwd_hash', hash); } async function verifyPassword(inputPwd: string): Promise<boolean> { let inputHash = await CryptoHelper.hashPassword(inputPwd); let storedHash = await prefs.get('pwd_hash', ''); return inputHash === storedHash; }
