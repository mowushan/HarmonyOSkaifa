🧩 知识讲解 · 让页面“动”起来

1. 为什么需要滚动容器？—— 一屏放不下怎么办

📱 生活场景： 刷抖音、看朋友圈、逛淘宝……是不是都要上下滑动？因为内容太多，屏幕装不下！

在鸿蒙ArkUI中，我们使用 Scroll 组件来实现滚动效果。

  • 📦 Scroll容器：它是一个“可滚动的盒子”，把内容放进去，内容超出高度/宽度时自动出现滚动条。
  • ↕️ 滚动方向：默认垂直滚动，也可以设置为水平滚动。
  • ⚡ 性能注意：如果滚动列表里有几百上千条数据，要用 LazyForEach 懒加载，不然会卡顿。

🤔 思考： 为什么手机App很少一屏显示完所有内容？（因为屏幕高度有限，滚动是刚需）

2. Scroll组件基本用法 —— 5分钟学会

// 最基本的垂直滚动 Scroll() { Column() { Text('第1行内容') Text('第2行内容') // ... 很多内容 Text('第20行内容') } } .width('100%') .height('100%')

重要属性：

  • scrollable(ScrollDirection.Vertical) — 垂直滚动（默认）
  • scrollable(ScrollDirection.Horizontal) — 水平滚动
  • scrollBar(BarState.Auto) — 滚动条自动显示

3. 水平滚动：横向卡片效果

Scroll() { Row() { ForEach([1,2,3,4,5,6,7,8], (item: number) => { Card() { Text(`卡片${item}`) } }) } } .scrollable(ScrollDirection.Horizontal) // 水平滚动 .width('100%') .height(200)

✨ 常用于：商品推荐栏、热门话题横向滑动、图片画廊。

4. 性能优化：LazyForEach（懒加载）

⚡ 如果资产列表有1000条数据，一次性渲染会很卡。用 LazyForEach 只渲染屏幕上看得见的几条，滑动时动态加载，丝般顺滑！

今天我们先用普通ForEach，下节课再深入LazyForEach。先掌握Scroll的用法。

⚙️ 技能实训 · 在DevEco Studio中动手实现滚动页面

✅ 本次实训目标：掌握Scroll组件，改造资产列表为可滚动页面

📁 准备工作

1. 打开DevEco Studio，打开我们之前创建的鸿蒙项目。

2. 找到 AssetList.ets（资产列表页），我们今天要把它改造成可滚动的列表。

3. 如果还没有这个文件，新建一个页面，命名为 AssetList.ets。

🧪 实训任务一：垂直滚动 · 实现资产列表滚动（20分钟）

  • 创建基础结构： 在 AssetList.ets 中编写以下代码框架：
    import { Column, Text, Scroll } from '@ohos.arkui'; @Entry @Component struct AssetList { @State assets: string[] = []; aboutToAppear() { // 模拟加载50条资产数据 for (let i = 1; i <= 50; i++) { this.assets.push(`资产${i}：设备编号DEV-${i}`); } } build() { Column() { Text('我的资产列表') .fontSize(24) .fontWeight(FontWeight.Bold) .margin({ bottom: 16 }) Scroll() { Column() { ForEach(this.assets, (item: string) => { Text(item) .width('100%') .height(60) .backgroundColor('#f5f5f5') .borderRadius(12) .margin({ bottom: 8 }) .padding({ left: 16 }) .fontSize(16) }) } .width('100%') } .scrollable(ScrollDirection.Vertical) // 垂直滚动 .scrollBar(BarState.Auto) .width('100%') .height('100%') } .width('100%') .height('100%') .padding(16) } }
  • 运行预览： 在模拟器或真机上预览，尝试上下滑动，观察滚动效果。
  • 修改数据量： 把循环改为100条，看看滚动是否依然流畅。
  • 小组讨论： 如果不加Scroll组件，页面会怎样？（提示：内容会被截断，无法滑动）

    • 💡 常见错误： 记得给Scroll组件设置 .height('100%')，否则滚动区域可能不生效！

    🧪 实训任务二：水平滚动 · 实现横向分类卡片（20分钟）

  • 在资产列表页顶部添加横向滚动分类栏： 在Text标题下方、Scroll上方增加以下代码：
    // 横向滚动分类栏 Text('热门分类').fontSize(18).fontWeight(FontWeight.Medium).margin({ top: 8, bottom: 8 }) Scroll() { Row() { ForEach(['电脑', '手机', '平板', '显示器', '键盘', '鼠标', '耳机', '路由器'], (category: string) => { Text(category) .width(80) .height(40) .backgroundColor('#e6f4ff') .borderRadius(20) .textAlign(TextAlign.Center) .margin({ right: 12 }) .fontSize(14) }) } .padding({ left: 8, right: 8 }) } .scrollable(ScrollDirection.Horizontal) .scrollBar(BarState.Off) .height(60) .width('100%')
  • 运行查看： 分类栏可以左右滑动，点击分类暂时只做展示，后续再添加筛选功能。
  • 挑战任务： 把分类数据提取到 @State categories: string[] 中，方便后续动态修改。

    • 🧪 实训任务三：嵌套滚动 & 滚动事件（15分钟）

  • 添加滚动监听： 在Scroll组件上添加 .onScroll() 事件，打印当前滚动偏移量。
    .onScroll((xOffset: number, yOffset: number) => { console.info(`滚动偏移: x=${xOffset}, y=${yOffset}`); })
  • 观察控制台输出： 在HiLog或预览窗口的Log中查看滚动时的输出。
  • 嵌套滚动注意事项： 如果Scroll里面再套Scroll，会出现滚动冲突。今天我们只做一层Scroll。

    • ⚠️ 注意：Scroll组件内通常只能有一个直接子组件（Column或Row），把其他内容放进去即可。
