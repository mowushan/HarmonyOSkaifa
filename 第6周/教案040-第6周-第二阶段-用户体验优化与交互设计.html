🧩 知识讲解 · 好体验是怎样炼成的

1. 用户体验(UX)是什么？—— 让用户“哇塞”的秘密

😊 趣味引入： 想象一下，你点外卖时：

  • ❌ 坏体验：按钮太小总点错，加载转圈半天没反应，不知道点没点成功。
  • ✅ 好体验：按钮很大，点击有震动/颜色变化，加载有进度条，完成后有“叮”一声提示。

UX五大黄金法则（简化版）：

  1. 即时反馈： 用户操作后，0.1秒内要有视觉/触觉/听觉反馈。
  2. 一致性： 相同功能的按钮在App里长得一样，位置也类似。
  3. 防错： 危险操作前询问确认，按钮够大不容易误触。
  4. 高效： 常用功能一步直达，减少点击次数。
  5. 美观： 舒服的颜色、间距、字体，让用户愿意停留。

2. 鸿蒙ArkUI中的交互反馈实现

🔘 按钮点击态

使用.stateStyles配置不同状态下的样式。

💬 Toast弹窗

promptAction.showToast() 轻提示，短暂显示操作结果。

⏳ 加载动画

LoadingProgress 组件，网络请求时展示。

✨ 页面转场

使用PageTransitionEnter/Exit实现滑动/淡入淡出效果。

// 示例1：按钮点击态 + Toast反馈
Button('保存资产', { type: ButtonType.Normal })
  .backgroundColor('#0078D4')
  .stateStyles({
    pressed: { .backgroundColor('#005A9E') },   // 按下时颜色变深
    normal: { .backgroundColor('#0078D4') }
  })
  .onClick(() => {
    // 保存成功后弹出提示
    promptAction.showToast({
      message: '保存成功！',
      duration: 2000
    });
  })

// 示例2：加载中状态
if (this.isLoading) {
  LoadingProgress()
    .width(40).height(40)
    .color('#0078D4')
} else {
  // 正常内容显示
}

3. 动效让界面“活”起来

🎬 动效不是花里胡哨，而是为了：

  • ✅ 引导注意力（新功能高亮闪一下）
  • ✅ 反馈操作结果（删除时卡片飞走）
  • ✅ 过渡更自然（页面切换不突兀）

鸿蒙动画API： animateTo() 和 transition()

// 示例3：添加资产时的弹入动画
animateTo({ duration: 300, curve: Curve.EaseInOut }, () => {
  this.showAddDialog = true;
})

// 示例4：列表项滑动删除动效（简化）
ListItem() {
  // ... 内容
}
.swipeAction({ end: this.buildDeleteButton() })

⚙️ 技能实训 · 动手优化我们的资产管理系统

✅ 本次实训目标：将第二阶段完成的资产录入/列表页进行UX大改造

🔧 实训一：按钮交互升级（15分钟）

  • 找到资产录入页的“提交/保存”按钮，为它添加 .stateStyles 按下态（颜色变深或缩放效果）。
  • 在保存成功的回调里，添加Toast提示“添加成功”或“更新成功”。
  • 测试： 点击按钮，观察按下效果+松手后出现短暂提示。
    • // 参考代码片段
Button('添加资产')
  .width('90%').height(44)
  .backgroundColor('#0078D4')
  .stateStyles({
    pressed: { .backgroundColor('#005A9E'), .scale(0.98) }
  })
  .onClick(() => {
    // 执行保存逻辑
    if (this.saveData()) {
      promptAction.showToast({ message: '资产已添加', duration: 1500 });
    }
  })

    🔧 实训二：加载状态 + 空状态设计（15分钟）

  • 在资产列表页添加 isLoading 状态变量（默认为true）。
  • 在aboutToAppear中模拟网络请求（setTimeout 1秒后加载数据），期间显示LoadingProgress。
  • 如果列表为空，显示“暂无资产，点击+号添加”的空状态插图+文案。
    • // 参考代码：加载+空状态
build() {
  Column() {
    if (this.isLoading) {
      LoadingProgress().width(50).height(50)
    } else if (this.assetList.length === 0) {
      Column() {
        Image($r('app.media.ic_empty')).width(80).height(80)
        Text('暂无资产，去添加一条吧~').fontSize(14).fontColor('#999')
      }
    } else {
      List() { /* 正常列表 */ }
    }
  }
}

    🔧 实训三：页面转场动画（15分钟）

  • 在路由跳转时添加转场动画：从详情页返回列表页时，使用滑动效果。
  • 修改 pageTransition() 方法，设置进场/退场动画。
  • 测试： 点击资产项进入详情，再返回，观察页面切换是否平滑。
    • // 在自定义组件中添加转场动画
pageTransition() {
  PageTransitionEnter({ duration: 300, curve: Curve.EaseOut })
    .slide(SlideEffect.Right)
  PageTransitionExit({ duration: 300, curve: Curve.EaseIn })
    .slide(SlideEffect.Left)
}

    🔧 实训四：小组UX评审会（20分钟）

    📋 评审清单（每组互相交换设备体验5分钟）：

    • ✅ 按钮是否够大（手指触摸区域≥44x44pt）？
    • ✅ 操作后是否有明确反馈（颜色/提示/动画）？
    • ✅ 加载时用户是否知道正在加载？
    • ✅ 字体大小是否清晰（正文≥14sp）？
    • ✅ 颜色对比度是否足够（灰色字在浅灰背景上是否看得清）？
    • ✅ 错误提示是否友好（不是“Error 500”，而是“网络开小差了，稍后再试”）？

    🎯 每组选出“最佳体验改进点”和“最需要修复的问题”，上台分享2分钟。

    👍 好例子： 保存按钮按下有颜色变化，松手弹出“保存成功”，1秒后自动消失；列表加载时有转圈动画。
    👎 反面例子： 点击按钮没任何反应，过了3秒突然跳转；列表加载时白屏，用户以为卡死了。
