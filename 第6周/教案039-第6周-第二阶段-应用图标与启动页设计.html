🧩 知识讲解 · 应用图标的门道

1. 应用图标——App的“脸面”

📱 趣味小故事： 你有没有因为一个App的图标太丑而不想下载它？或者因为图标很酷就忍不住点进去？

鸿蒙应用图标规范：

  • 📐 尺寸要求： 需要提供多种分辨率的图标，适配不同设备（手机、平板、手表）。常见尺寸：48x48、72x72、96x96、144x144、192x192像素。
  • 🎨 设计原则： 简洁明了、识别度高、风格统一。建议使用矢量图形，避免过多文字。
  • 🖼️ 格式： 推荐使用PNG格式（支持透明背景）或SVG矢量图。

📁 图标存放位置： entry/src/main/resources/base/media/ 目录下。

2. 启动页——App的“迎宾员”

🚪 为什么需要启动页？

  • ✨ 提升用户体验：在App加载时展示品牌形象，避免白屏等待。
  • 🎯 品牌展示：展示Logo、Slogan，加深用户印象。
  • ⏳ 加载缓冲：在后台初始化数据、加载配置。

鸿蒙中实现启动页的两种方式：

  1. 使用Ability+页面跳转：创建一个单独的启动页面，延迟跳转到主页面（今天我们就用这个！）。
  2. 使用窗口Stage模型配置：更高级的方式，适合复杂场景。

3. 主题色——App的“气质”

🎨 什么是主题色？

主题色是App的主色调，会影响到按钮、导航栏、选中状态等组件的颜色。统一的主题色让App看起来更专业！

⚙️ 如何配置？

在 color.json 或 theme.json 中定义颜色值，然后在组件中使用 $r('app.color.primary') 引用。

⚙️ 技能实训 · 今日动手任务清单

✅ 本次实训目标：为资产管理系统设计应用图标、启动页和主题色

🔧 实训一：准备应用图标素材

  • 设计/下载图标： 可以使用在线工具（如Canva、IconFont）设计一个简洁的资产管理图标，或下载一个免费图标。推荐尺寸：512x512px。
  • 生成多尺寸图标： 使用在线图标生成器（如https://icon.wuruihong.com/）将大图标一键生成多尺寸PNG文件。
  • 复制到项目： 将生成的图标文件（如 icon.png）复制到 entry/src/main/resources/base/media/ 目录下。
  • 📸 截图打卡： 将media文件夹中的图标文件截图，发到集训队群。

    • 💡 没有设计基础怎么办？ 可以用纯色背景+文字“资产管家”作为临时图标，重点学会替换流程，后续可以再优化设计！

    🔧 实训二：修改配置文件，替换应用图标

  • 打开 entry/src/main/module.json5 配置文件。
  • 找到 icons 配置项，修改为你的图标文件名：
    "icons": { "foreground": "$media:icon", // 前景图（用于自适应图标） "background": "$media:icon_bg" // 背景图 }

    如果只有一张图，可以简单配置："icon": "$media:icon"

  • 保存并重新运行App，查看手机/模拟器桌面图标是否已更新。

    • 📌 如果图标没有变化，尝试卸载App重新安装，或者清理项目（Build → Clean Project）。

    🔧 实训三：创建启动页（Splash Screen）

  • 新建启动页面的ets文件： 在 entry/src/main/ets/pages/ 下新建 SplashPage.ets。
  • 编写启动页代码：
    // SplashPage.ets @Entry @Component struct SplashPage { @State progress: number = 0 aboutToAppear() { // 模拟加载进度，2秒后跳转到主页面 setTimeout(() => { router.replaceUrl({ url: 'pages/Index' }) }, 2000) } build() { Column() { // 启动页Logo Image($r('app.media.icon')) .width(120) .height(120) .margin({ top: 150 }) // App名称 Text('资产管家') .fontSize(28) .fontWeight(FontWeight.Bold) .margin({ top: 20 }) .fontColor($r('app.color.primary')) // Slogan Text('轻松管理 · 一目了然') .fontSize(16) .fontColor('#666666') .margin({ top: 8 }) // 加载指示器 LoadingProgress() .width(40) .height(40) .margin({ top: 60 }) .color($r('app.color.primary')) } .width('100%') .height('100%') .backgroundColor('#FFFFFF') } }
  • 修改启动顺序： 打开 entry/src/main/ets/pages/Index.ets，这是之前的首页。我们需要将启动页设为入口页面。
  • 配置路由： 确保 module.json5 中的 mainElement 指向启动页，或者直接在启动页加 @Entry 注解，将首页的 @Entry 注释掉。
  • 更简单的方法：在启动页加 @Entry，并把 Index.ets 的 @Entry 注释掉，这样App启动时先加载启动页。
    • // 修改后 Index.ets 头部（注释掉@Entry） // @Entry @Component struct Index { // ... 原有代码 }

    🎯 运行测试： 重新运行App，应该先看到启动页停留2秒，然后自动跳转到资产列表主页。

    🔧 实训四：配置应用主题色

  • 打开 entry/src/main/resources/base/element/color.json 文件。
  • 添加或修改主题色定义：
    { "color": [ { "name": "primary", "value": "#0078D4" // 鸿蒙蓝，也可以改成自己喜欢的颜色 }, { "name": "primary_dark", "value": "#005A9E" }, { "name": "background", "value": "#F5F5F5" } ] }
  • 在组件中使用主题色： 比如修改按钮颜色 .backgroundColor($r('app.color.primary'))。
  • 全局主题配置（进阶）： 可以在 EntryAbility.ets 中通过 window.setWindowBackgroundColor 设置全局背景色。

    • 🌈 小组任务： 每组选择一个主题色（如科技蓝、活力橙、清新绿），统一修改App中所有页面的按钮、标题栏颜色，让视觉风格一致。
