🧩 知识讲解 · 让数据开口说话

1. 为什么需要图表？—— 一图胜千言

📈 趣味引入： 给你一串数字：“电脑15台，显示器22台，键盘30个，鼠标28个”——你能一眼看出哪个最多吗？如果变成柱状图，最高的柱子就是答案！

图表三兄弟：

  • 📊 柱状图： 比大小，看数量差异（适合资产分类统计）。
  • 📉 折线图： 看趋势，比如每月资产增长情况。
  • 🥧 饼图： 看占比，比如各类资产占总资产的比例。

今天我们重点攻克柱状图，因为资产管理系统最常用！

2. 鸿蒙中如何画图？—— 借助开源图表库

🔧 HarmonyOS没有内置图表组件，但我们可以用优秀的开源库 ohos-mpchart（基于Android MPAndroidChart移植）。

三步走：

  1. 添加依赖： 在 oh-package.json5 中配置库的地址。
  2. 在XML布局中放置图表组件。
  3. 在ArkTS代码中设置数据并渲染。

💬 就像搭积木一样简单，老师会带着大家一步一步配置！

3. 柱状图核心API速览

📦 数据集 BarDataSet

存放柱子的数值列表和样式（颜色、名称）。

📊 数据对象 BarData

管理一个或多个BarDataSet，设置整体属性。

🎨 X轴 XAxis / Y轴 YAxis

设置坐标轴标签、位置、字体大小等。

🖱️ 交互 点击监听

图表支持点击柱子回调，获取被点击的数值和索引。

别担心记不住，跟着实训代码走一遍就理解了！

⚙️ 技能实训 · 手把手实现资产统计柱状图

✅ 本次实训目标：从零到一，让App显示资产分类统计柱状图

🔧 实训一：集成图表库（15分钟）

  • 打开项目工程，找到 oh-package.json5 文件。
  • 在 dependencies 中添加依赖：
    "dependencies": { "ohos-mpchart": "3.0.0-rc.1" }
  • 点击右上角 “Sync Now” 同步依赖，等待下载完成。
  • 在需要展示图表的页面 .ets 文件中导入组件：
    import { BarChart } from 'ohos-mpchart'; import { BarData, BarDataSet, BarEntry, XAxis, YAxis, AxisBase } from 'ohos-mpchart';
  • ✅ 验证成功： 没有报错即说明集成完成！

    • 💡 如果下载失败，老师会提供离线包或备用镜像，不要着急～

    🔧 实训二：准备模拟资产数据（10分钟）

  • 在页面状态变量中定义资产分类和数量：
    @State categories: string[] = ['电脑', '显示器', '键盘', '鼠标', '打印机']; @State counts: number[] = [15, 22, 30, 28, 8];
  • 将数据转换成图表组件需要的 BarEntry 格式：
    private getBarEntries(): BarEntry[] { let entries: BarEntry[] = []; for (let i = 0; i < this.counts.length; i++) { entries.push(new BarEntry(i, this.counts[i])); } return entries; }
  • 理解每个BarEntry对应一个柱子，x为索引（0,1,2...），y为数值。

    • 🔧 实训三：在页面中放置图表并显示数据（25分钟）

  • 在build()方法中添加柱状图组件，并设置id：
    build() { Column() { BarChart({ id: 'assetBarChart' }) .width('100%') .height(300) .onBarClick((index: number, entry: BarEntry) => { console.log(`点击了第${index}个柱子，数值: ${entry.getY()}`); }) } .padding(20) }
  • 在 aboutToAppear() 生命周期中配置图表数据和样式：
    aboutToAppear() { let barChart = this.$r('assetBarChart') as BarChart; let entries = this.getBarEntries(); let dataSet = new BarDataSet(entries, '资产数量'); dataSet.setColors([ '#FF6B6B', '#4ECDC4', '#45B7D1', '#96CEB4', '#FFEAA7' ]); let barData = new BarData(dataSet); barChart.setData(barData); // 配置X轴 let xAxis = barChart.getXAxis(); xAxis.setPosition(XAxis.Position.BOTTOM); xAxis.setGranularity(1); xAxis.setLabelCount(this.categories.length); xAxis.setValueFormatter((value: number, axis: AxisBase) => { return this.categories[value]; }); // 配置Y轴 let leftAxis = barChart.getAxisLeft(); leftAxis.setGranularity(1); leftAxis.setAxisMinimum(0); barChart.getAxisRight().setEnabled(false); barChart.getDescription().setEnabled(false); barChart.animateY(1000); }
  • 运行程序，查看效果！ 应该能看到彩色的柱状图，底部X轴显示资产分类名称。

    • 🔥 如果图表没显示，检查：① 是否正确调用 setData ② 是否调用了 invalidate() 刷新（图表库会自动刷新，一般不用） ③ 查看控制台是否有错误。

    🔧 实训四：从数据库动态加载数据（挑战任务 20分钟）

    任务说明： 之前我们使用的是静态模拟数据，现在要改为从资产数据库中读取真实的分类统计。

    SQL思路： 统计每种分类的资产数量，例如：

    SELECT category, COUNT(*) as count FROM assets GROUP BY category;

    在ArkTS中执行查询，将结果填充到 categories 和 counts 数组中，然后刷新图表。

  • 编写数据库查询函数，使用关系型数据库 relationalStore 执行分组统计SQL。
  • 查询结果存入数组，并更新 categories 和 counts。
  • 重新调用配置图表的代码，刷新界面（可以将配置代码封装成一个 refreshChart() 方法）。
    • // 伪代码示例 async loadDataFromDB() { let predicates = new relationalStore.RdbPredicates('assets'); let resultSet = await this.rdbStore.query(predicates); let categoryMap = new Map(); while (resultSet.goToNextRow()) { let category = resultSet.getString(resultSet.getColumnIndex('category')); categoryMap.set(category, (categoryMap.get(category) || 0) + 1); } this.categories = Array.from(categoryMap.keys()); this.counts = Array.from(categoryMap.values()); this.refreshChart(); }

    🏆 完成此任务的同学，说明你已经掌握了图表+数据库的联动，离优秀作品又近一步！
