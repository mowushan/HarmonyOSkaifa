🔰 第二关 · 状态管理连连练（15分钟）

🎯 场景： 父组件有一个计数器count，子组件需要显示并修改它。要求使用@State和@Link。

任务：

  1. 创建父组件 CounterParent，包含一个数字显示和“重置”按钮。
  2. 创建子组件 CounterChild，包含“+1”和“-1”按钮，并能同步修改父组件的count。
  3. 当父组件点击“重置”时，count归零，子组件显示同步归零。
// 父组件
@Component
struct CounterParent {
    @State count: number = 0

    build() {
        Column() {
            Text(`当前计数：${this.count}`).fontSize(24)
            Button('重置').onClick(() => { this.count = 0 })
            CounterChild({ count: $count })  // 使用$传递@Link
        }
    }
}

// 子组件
@Component
struct CounterChild {
    @Link count: number

    build() {
        Row({ space: 20 }) {
            Button('-1').onClick(() => { this.count-- })
            Button('+1').onClick(() => { this.count++ })
        }
    }
}

✅ 通关标准： 父子组件数据双向同步，重置功能正常。

🔰 第三关 · 列表渲染与页面跳转（20分钟）

🎯 任务： 实现“资产列表页”，展示至少3条假数据（资产名称、价格），点击任意一项跳转到“资产详情页”，并传递资产名称参数。

要求：

  • 使用List + ForEach渲染列表项。
  • 详情页接收参数并显示“您点击了：XXX”。
  • 使用router.pushUrl进行跳转。
// 列表页
import router from '@ohos.router'

@Entry
@Component
struct AssetList {
    @State assets: Array<{name: string, price: number}> = [
        { name: '笔记本电脑', price: 5999 },
        { name: '投影仪', price: 3200 },
        { name: '实训开发板', price: 899 }
    ]

    build() {
        Column() {
            List() {
                ForEach(this.assets, (item: {name: string, price: number}, index: number) => {
                    ListItem() {
                        Column() {
                            Text(item.name).fontSize(18)
                            Text(`¥${item.price}`).fontSize(14).fontColor(Color.Gray)
                        }
                        .onClick(() => {
                            router.pushUrl({
                                url: 'pages/AssetDetail',
                                params: { assetName: item.name }
                            })
                        })
                    }
                })
            }
        }
    }
}

// 详情页
@Entry
@Component
struct AssetDetail {
    private assetName: string = router.getParams()?.['assetName'] || '未知资产'

    build() {
        Column() {
            Text(`你点击了：${this.assetName}`).fontSize(24)
            Button('返回').onClick(() => { router.back() })
        }
    }
}

✅ 通关标准： 列表显示正确，点击跳转并正确显示传递的参数。

🔰 第四关 · 综合项目：资产录入+列表联动（25分钟）

🎯 终极挑战： 将资产录入页和资产列表页串联成一个小型App。

需求：

  1. 首页为资产列表页，右上角有一个“+”按钮，点击跳转到资产录入页。
  2. 资产录入页包含：资产名称(TextInput)、资产价格(TextInput)、购买日期(TextInput)，以及“保存”按钮。
  3. 点击“保存”后，将新资产添加到列表页的数据源中，并自动返回列表页刷新显示。
  4. 提示：可以使用全局状态管理（如AppStorage）或者通过router带回参数+回调刷新。

参考实现思路（通过router返回时传递新数据）：

// 列表页增加接收返回数据的逻辑
onPageShow() {
    // 每次显示页面时，检查是否有新数据
    const newAsset = AppStorage.get<string>('newAsset')
    if (newAsset) {
        this.assets.push(JSON.parse(newAsset))
        AppStorage.delete('newAsset')
    }
}

// 录入页保存时
Button('保存').onClick(() => {
    const newItem = { name: this.name, price: this.price }
    AppStorage.setOrCreate('newAsset', JSON.stringify(newItem))
    router.back()
})

✅ 通关标准： 能成功添加新资产并立即在列表页看到新增项。

@Entry
@Component
struct BuggyPage {

    build() {
        Column() {
            Text(this.message)
                .fontSize(30)
            Button("跳转")
                .onClick(() => {
                    router.push("pages/TargetPage")
                })
            ChildComponent({ msg: this.message })
        }
    }
}

@Component
struct ChildComponent {
    @Link msg: string
    build() {
        Text(msg)
    }
}

🐛 错误提示： 1. router.push 需要换成 router.pushUrl；2. 子组件接收父组件的普通字符串应该用@Prop而不是@Link；3. 传递参数应该用 $message 或直接传值，但@Link要求变量是引用类型。

⚡ 快问快答 · 积分赛

❓ 问题1

ArkTS中，哪个装饰器用于定义组件内部响应式变量？

✅ 答案：@State

❓ 问题2

页面跳转时传递参数，应该使用 router 的哪个方法？

✅ 答案：router.pushUrl()，并在params中携带参数。

❓ 问题3

ForEach循环渲染时，第二个参数是什么？作用是什么？

✅ 答案：index（索引），用于标识唯一性。

❓ 问题4

@Prop和@Link的区别是什么？

✅ 答案：@Prop是单向数据流，子组件不可修改父组件；@Link是双向同步。
