🧩 知识讲解 · API就是“数据快递员”

1. API是什么？—— 餐厅点餐的比喻

🍽️ 趣味比喻： 你去餐厅吃饭，不需要直接冲进厨房。你只需要告诉服务员（API）你想吃什么，服务员去厨房（服务器）把菜（数据）端给你。

关键角色：

  • 📱 客户端（你的App）：你本人，想获取数据。
  • 🔗 API接口：服务员，传递你的请求。
  • 🗄️ 服务器：厨房，提供数据。

📣 在鸿蒙中，我们使用 http 模块来扮演“服务员”的角色！

2. 常用HTTP方法 · 餐厅菜单

📖 GET 查

获取数据，就像“看菜单”。不改变服务器状态。

✏️ POST 增

提交新数据，就像“点新菜”。

🔧 PUT 改

更新数据，就像“修改订单”。

🗑️ DELETE 删

删除数据，就像“退菜”。

✅ 今天重点： GET请求 —— 从服务器获取资产数据！

3. JSON格式 · 数据的“普通话”

// 服务器返回的资产数据示例（JSON格式） { "code": 200, "message": "success", "data": [ { "id": 1, "name": "笔记本电脑", "price": 5999 }, { "id": 2, "name": "投影仪", "price": 3200 } ] }

💡 JSON就是“键值对”组成的文本，就像学生证：{"姓名":"张三","学号":"001"}，简单易懂！

⚙️ 技能实训 · 让鸿蒙应用联网获取数据

✅ 本次实训目标：完成一个完整的API调用Demo，成功解析并显示数据

🔧 实训一：配置网络权限（必须第一步）

  • 打开鸿蒙项目的 src/main/module.json5 文件
  • 在 module 节点下添加网络权限：
    // 在 module.json5 的 requestPermissions 数组中添加 { "name": "ohos.permission.INTERNET" }
  • 保存文件，如果IDE提示同步，点击“Sync Now”

    • ⚠️ 忘记加权限 = 网络请求一定会失败！ 这是99%新手踩的第一个坑，老师帮大家检查。

    🔧 实训二：编写API工具类（完整代码）

    新建文件 common/HttpUtil.ets，封装网络请求逻辑：

    // HttpUtil.ets - 网络请求工具类 import http from '@ohos.net.http'; export class HttpUtil { // 发送GET请求 static async get(url: string): Promise<object> { let httpRequest = http.createHttp(); try { let response = await httpRequest.request(url, { method: http.RequestMethod.GET, connectTimeout: 10000, // 10秒超时 readTimeout: 10000 }); // 解析JSON字符串 let result = JSON.parse(response.result as string); return result; } catch (err) { console.error('请求失败: ' + JSON.stringify(err)); throw err; } finally { httpRequest.destroy(); } } }

    🔧 实训三：创建免费测试API（模拟云端资产数据）

  • 打开浏览器，访问在线JSON服务： https://jsonplaceholder.typicode.com/posts （免费测试接口）
  • 也可以使用Mock API： https://run.mocky.io/v3/你的自定义ID
  • 推荐： 老师提前准备一个模拟资产数据的API地址，例如：
    https://api.npoint.io/1a2b3c4d5e6f // 示例地址，实际课堂上老师提供真实地址
  • 测试API是否可用： 在浏览器直接打开该地址，应该能看到JSON数据。

    🔧 实训四：在页面中调用API并显示数据

  • 创建页面 pages/AssetList.ets，代码如下：
    • // AssetList.ets - 资产列表页面（联网版） import { HttpUtil } from '../common/HttpUtil'; @Entry @Component struct AssetList { @State assets: Array<AssetItem> = []; @State loading: boolean = true; aboutToAppear() { this.fetchAssets(); } async fetchAssets() { this.loading = true; try { // 替换为你的真实API地址 let apiUrl = 'https://api.npoint.io/你的资产数据ID'; let response = await HttpUtil.get(apiUrl); // 假设返回格式: { code:200, data:[...] } if (response.code === 200) { this.assets = response.data; } } catch (error) { console.error('加载失败', error); } finally { this.loading = false; } } build() { Column() { if (this.loading) { LoadingProgress().width(50).height(50) } else { List() { ForEach(this.assets, (item: AssetItem) => { ListItem() { Row() { Text(item.name).fontSize(18).fontWeight(FontWeight.Bold) Blank() Text(`¥${item.price}`).fontColor('#0078d4') }.width('100%').padding(12) } }) } } }.width('100%').height('100%').padding(16) } } // 数据模型 interface AssetItem { id: number; name: string; price: number; }

    💡 老师分步讲解：

    1. aboutToAppear：页面加载时自动调用网络请求。
    2. @State：数据变化自动刷新界面。
    3. LoadingProgress：加载时显示转圈动画，提升用户体验。

    🔧 实训五：运行调试 + 错误处理增强

  • 修改API地址为真实地址，点击运行，观察模拟器/真机是否显示资产列表。
  • 故意把API地址写错，观察控制台错误输出，学会看报错信息。
  • 添加友好错误提示：
    @State errorMsg: string = ''; // 在catch块中添加 this.errorMsg = '网络开小差了，请稍后重试'; // 在build中显示错误信息 if (this.errorMsg) { Text(this.errorMsg).fontColor('#ff4444') }
  • 小组互相检查： 每个人的代码都能成功获取并显示数据吗？

    • 🐛 常见bug及解决：

    • ❌ 网络请求失败 → 检查module.json5是否添加INTERNET权限
    • ❌ 数据不显示 → 打印response看JSON结构，调整解析字段
    • ❌ 模拟器无网络 → 关闭代理或重启模拟器
