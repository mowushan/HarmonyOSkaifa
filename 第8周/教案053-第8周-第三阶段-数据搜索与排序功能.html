🧩 知识讲解 · 搜索与排序的奥秘

1. 数据搜索 —— 从“大海捞针”到“精准定位”

🔍 趣味引入： 想象一下，如果资产列表里有1000条数据，你要找到“笔记本电脑”得翻多久？搜索功能就是帮你“瞬间定位”的魔法！

搜索的两种方式：

  • 精确搜索： 输入内容必须完全匹配，比如搜索“华为笔记本”只会出现完全相同的。
  • 模糊搜索： 输入“华为”，所有包含“华为”的记录都会出现（包括“华为手机”、“华为平板”）。

ArkTS实现核心： 使用数组的 filter() 方法 + 字符串的 includes() 方法。

// 模糊搜索示例 let assets = [ { name: "华为笔记本", price: 5999 }, { name: "苹果手机", price: 6999 }, { name: "华为平板", price: 3299 } ]; let searchText = "华为"; let result = assets.filter(item => item.name.includes(searchText)); // result 包含 "华为笔记本" 和 "华为平板"

2. 数据排序 —— 让杂乱的数据“排排坐”

📊 排序就像军训排队： 按身高从矮到高（升序）或从高到矮（降序）。

三种常见排序：

  • 按名称排序： 字母/拼音顺序 A→Z 或 Z→A
  • 按价格排序： 从小到大（升序）或从大到小（降序）
  • 按日期排序： 从旧到新或从新到旧
// 价格升序排序 assets.sort((a, b) => a.price - b.price); // 价格降序排序 assets.sort((a, b) => b.price - a.price); // 按名称升序（字符串比较） assets.sort((a, b) => a.name.localeCompare(b.name));

3. 搜索 + 排序 = 完美体验

⚠️ 关键点： 搜索和排序是链式操作！

正确流程：先搜索过滤 → 再对结果排序（而不是在原数据上排序）

// 正确姿势：先filter再sort let filtered = this.assets.filter(item => item.name.includes(this.searchKeyword)); let sorted = [...filtered].sort((a, b) => a.price - b.price);

🧪 实训任务一：添加搜索输入框（15分钟）

  • 在页面顶部添加TextInput组件，绑定一个 @State searchText: string = '' 变量。
  • 设置placeholder提示： “请输入资产名称搜索”。
  • 监听onChange事件： 当输入内容变化时，更新searchText并重新过滤数据。
  • 实现过滤计算属性：
    getFilteredAssets() { if (this.searchText === '') { return this.assets; } return this.assets.filter(item => item.name.toLowerCase().includes(this.searchText.toLowerCase()) ); }
  • 在build()中使用： 遍历 this.getFilteredAssets() 展示列表。

    • 🎯 检验标准： 输入“华为”，列表只显示华为笔记本和平板；输入“苹果”，只显示苹果手机。

    🧪 实训任务二：添加排序按钮（20分钟）

  • 在搜索框下方添加三个按钮： “按价格↑升序”、“按价格↓降序”、“按名称排序”。
  • 定义排序状态： @State sortType: string = 'none' (可选值: 'priceAsc', 'priceDesc', 'nameAsc')
  • 编写排序函数：
    getSortedAndFilteredAssets() { let filtered = this.getFilteredAssets(); let sorted = [...filtered]; switch (this.sortType) { case 'priceAsc': sorted.sort((a, b) => a.price - b.price); break; case 'priceDesc': sorted.sort((a, b) => b.price - a.price); break; case 'nameAsc': sorted.sort((a, b) => a.name.localeCompare(b.name)); break; default: break; } return sorted; }
  • 按钮点击事件： 修改sortType的值，页面自动重新渲染。
  • 高亮当前排序： 当前激活的按钮改变背景色（可选进阶）。

    • 💡 小技巧： 使用 [...filtered] 创建新数组再排序，避免修改原数组引发副作用。

    🧪 实训任务三：完整集成测试（20分钟）
