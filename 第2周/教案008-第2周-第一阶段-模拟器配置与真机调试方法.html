🧩 知识讲解 · 让程序“看见”设备

1. 模拟器 vs 真机 · 两个调试小帮手

📱 模拟器（Emulator）

✅ 优点：不用花钱买手机，随时创建各种型号，拍照、传感器都能模拟。

❌ 缺点：启动慢一点，某些硬件功能（如真实蓝牙）无法测试。

📲 真机（Real Device）

✅ 优点：100%真实体验，触摸、传感器、网络都真实。

❌ 缺点：需要一台鸿蒙设备（手机/平板），需要配置调试模式。

🎮 趣味比喻： 模拟器像“飞行模拟训练器”，在电脑上模拟开飞机；真机就是真正驾驶舱，感受真实气流！两个都要学会！

2. 真机调试三件套 · 打开“开发者模式”

  • 第一步：找到“关于手机” 连续点击“版本号”7次，直到提示“您已处于开发者模式”。
  • 第二步：开启USB调试 返回设置，进入“系统和更新”→“开发者选项”，打开“USB调试”开关。
  • 第三步：授权电脑连接 用USB线连接电脑，手机上弹出“允许USB调试”时，勾选“始终允许”并确认。

    • ⚠️ 特别提醒： 不同品牌手机路径略有差异，但核心都是“点版本号7次→开发者选项→USB调试”。如果找不到，可以问老师或百度“XX手机开启开发者模式”。

    3. 调试神器 · 让代码会“说话”

                // 鸿蒙中的日志输出（ArkTS）
            import hilog from '@ohos.hilog';
            
            // 输出一条调试信息
            hilog.info(0x0000, "MyApp", "按钮被点击了！");
            
            // 输出错误信息
            hilog.error(0x0000, "MyApp", "网络请求失败：%{public}s", errorMsg);

    📢 日志级别： DEBUG(调试) → INFO(信息) → WARN(警告) → ERROR(错误)。上课时我们先用最简单的方法：console.log() 也能打印！

    ⚙️ 技能实训 · 一步一步搭建调试环境

    ✅ 实训目标：让每个人都成功在模拟器或真机上运行Hello World程序

    🔧 实训一：创建并启动鸿蒙模拟器（全员必做）

  • 打开DevEco Studio，点击菜单栏“Tools” → “Device Manager”。
  • 点击“+”号，选择“New Emulator”，下载一个设备镜像（推荐选择“Phone”和“API 12”以上版本）。
  • 等待下载完成（约10-15分钟），此期间可以继续听老师讲解或看文档。
  • 下载完毕后，点击右侧“▶️”按钮启动模拟器，等待手机界面出现。
  • 🎉 验证成功： 模拟器屏幕上出现鸿蒙系统桌面，说明模拟器就绪！

    • 💡 小组互助： 下载慢的同学可以借用已下载同学的镜像缓存（老师可共享离线包）。先完成的同学去帮助未完成的组员。

    🔧 实训二：在模拟器上运行第一个应用（Hello World强化版）

  • 打开之前的Hello World项目（或新建一个Empty Ability项目）。
  • 在项目顶部工具栏，选择目标设备为刚才创建的模拟器（如“HUAWEI P60”）。
  • 点击绿色三角形“Run”按钮（或Shift+F10），观察编译过程。
  • 等待几秒，模拟器上自动安装并打开你的应用！
  • 📸 拍照打卡： 把模拟器上显示“Hello World”的界面拍下来发到集训队群，老师统计完成人数。

    • 🔥 恭喜！你已经成功在模拟器上运行了鸿蒙应用！接下来，如果你手边有鸿蒙设备（华为手机/平板），我们挑战真机调试！

    🔧 实训三（挑战任务）：真机调试实战

  • 准备工作： 一台鸿蒙设备（HarmonyOS 4.0或更高版本），数据线。
  • 按照前面“开发者模式”步骤，开启USB调试，并用数据线连接电脑。
  • 在DevEco Studio中，设备列表会多出一个“USB设备”，选中它。
  • 点击“Run”，如果第一次连接，手机会弹窗“允许USB调试”，勾选“始终允许”。
  • 观察真机： 应用自动安装并启动，触摸屏幕交互，感受真机手感！

    • 📱 没有真机的同学怎么办？ 可以小组共享一台设备轮流调试，也可以先用模拟器完成实训，课后借老师设备体验。重点：每个人都要理解流程！

    🔧 实训四：给程序加“监听器” · 学会调试输出

  • 在项目中找到 Index.ets 文件，在 build() 方法里找到 Text('Hello World')。
  • 在Text组件下方添加一个Button按钮，并绑定一个点击事件：
    • // 在Index.ets中添加代码
import hilog from '@ohos.hilog';

@Entry
@Component
struct Index {
  build() {
    Column() {
      Text('Hello World')
        .fontSize(30)
        .margin(20)
      
      Button('点我试试')
        .onClick(() => {
          // 输出日志到控制台
          console.log('按钮被点击了！');
          hilog.info(0x0000, 'MyTest', '用户点击了按钮');
        })
        .margin(20)
    }
    .width('100%')
    .height('100%')
  }
}
  • 再次运行到模拟器或真机，点击按钮，观察DevEco Studio下方的“Run”窗口，会看到打印的日志。
  • 🤔 思考： 如果程序出错，日志会告诉你错在哪一行，这就是调试的威力！

    • 🐛 小知识： “调试”就是给程序装上“摄像头”，看它每一步在做什么。日志就是最简单的摄像头！

    🔧 实训五：故意出错 & 看报错信息（不怕bug训练）

  • 故意把代码中的 console.log('按钮被点击了'); 后面的括号去掉一个。
  • 重新运行，观察DevEco Studio会怎么提示错误（红色波浪线、编译报错）。
  • 尝试修复： 根据错误提示，把括号补回去，再次运行成功。
  • 老师总结： 错误提示是程序给你的“线索”，学会看错误提示，就学会了调试的一半功夫！
