2. Codelabs —— 程序员最好的“陪练”

🎮 趣味类比： 就像王者荣耀里的“训练营”，每一步都有提示，跟着做就能学会连招！Codelabs就是鸿蒙开发的训练营。

为什么必用Codelabs？

  • ✅ 环境配置好的在线/离线项目，下载就能跑
  • ✅ 步骤拆解得非常细，连“点哪里”都告诉你
  • ✅ 覆盖从入门到进阶的100+实战案例

🔗 入口：开发者学堂首页 → 顶部菜单“Codelabs” → 筛选“HarmonyOS”

3. 学习规划心法 · 四阶段分解法

📅 我们的18周通关秘籍（再次强调）：

  • 🏁 第1-2周：环境 + ArkTS语法（对应课程《鸿蒙第一课》1-4章）
  • 🎨 第3-6周：ArkUI组件 + 布局（重点刷Codelabs“购物应用”“健康生活”界面部分）
  • 💾 第7-12周：数据存储 + 网络请求（Preferences、数据库、HTTP请求）
  • 🚀 第13-18周：服务卡片 + 分布式 + 项目打磨（必刷“分布式音乐播放器”“元服务”Codelabs）

🎯 今日任务： 每位同学登录账号，把这四个阶段的推荐课程和Codelabs加入“我的学习”，生成个人学习清单。

⚙️ 技能实训 · 动手成为资源达人

✅ 本次实训目标：登录开发者学堂，收藏核心资源，制定个人学习周计划

🔧 实训一：登录 & 个性化设置（10分钟）

  • 打开浏览器，访问 https://developer.huawei.com/consumer/cn/training/
  • 点击右上角“登录”，使用第一节课注册的华为账号登录。
  • 点击右上角头像 → “个人中心” → “我的学习”，查看目前是否有课程（若无则正常）。
  • 📸 打卡任务： 截屏“我的学习”页面（空页面也没关系），发到集训群，证明已登录成功。

    • 🔧 实训二：添加核心课程（15分钟）

  • 搜索并添加《鸿蒙第一课》： 在搜索框输入“鸿蒙第一课”，找到华为官方出品的系列课程（通常有8-10节），点击“加入学习”。
  • 搜索并添加《ArkTS入门》： 输入“ArkTS语言入门”，选择最新的版本课程，加入学习。
  • 搜索并添加《ArkUI开发基础》： 输入“ArkUI”，选择“声明式开发范式”相关课程，加入学习。
  • 验证： 回到“我的学习”，应该能看到至少3门课程显示“学习中”。

    • 💡 小技巧： 可以给课程打标签“第一阶段”“第二阶段”，方便后续查找。老师巡场，帮助找不到的同学。

    🔧 实训三：收藏Codelabs实战项目（15分钟）

  • 点击顶部菜单“Codelabs”，进入实验室首页。
  • 在筛选器选择“HarmonyOS” + “入门级”。
  • 必收藏1： “构建你的第一个HarmonyOS应用”（Hello World项目）—— 下节课就要用！点击“查看详情”，然后点“收藏”。
  • 必收藏2： “使用ArkTS开发购物应用（界面篇）” —— 适合第二阶段学习布局和列表。
  • 必收藏3： “数据持久化之Preferences” —— 第三阶段核心。
  • 收藏后，在“个人中心” → “我的收藏”里可以看到所有项目。

    • 🔥 挑战任务（速度快的同学）： 多收藏一个“元服务卡片开发”项目，为第四阶段预热！

    🔧 实训四：制定个人学习周计划（20分钟）

    📅 周计划表示例

    第2周（本周）课后计划：
    • 周一：复习账号，浏览《鸿蒙第一课》第1-2节视频
    • 周二：无（今天课堂已学）
    • 周三：看《ArkTS入门》第1-2章
    • 周四：预习下节课内容，准备DevEco Studio

    📋 空白模板（现场填写）

    每位同学在纸上或手机备忘录中，写下：
    1. 本周剩余3天每天花20分钟看哪个课程？
    2. 周末打算挑战哪个Codelabs？
    3. 小组内互相检查计划，组长收集拍照发群。

  • 老师投影展示一个填写好的周计划样例（例如用Excel或手写拍照）。
  • 学生各自填写自己的第2-3周学习计划（包含每周观看视频/动手Codelabs的目标）。
  • 小组内互相分享计划，并互相提建议（比如计划是否可行）。
  • 每组推选一个“最靠谱计划”，上台分享（每人1分钟）。

    • 🎯 老师总结： 有计划才不会迷茫！每周日晚上，组长在群里提醒大家检查计划完成情况，养成自律习惯。

    🔧 实训五：文档中心初探（10分钟）

  • 打开华为开发者官网顶部“文档” → 选择“HarmonyOS”。
  • 浏览“开发指南”目录，找到“UI开发” → “声明式开发范式”。
  • 实战小任务： 用文档中心的搜索功能，搜“Button组件”，看看官方文档怎么描述Button的属性和事件。
  • 每人记下“文档中心”的URL，添加到浏览器书签。

    • 📚 以后写代码遇到问题，先查官方文档！比任何搜索引擎都准确。

    🎮 互动活动 · 资源寻宝大挑战

    「寻宝猎人」小组对抗赛

    规则： 每组领取一张“寻宝清单”，限时10分钟，在开发者学堂/文档中心找到答案，最快最全的小组获胜。

    寻宝清单（共5题）：

    1. 找到“鸿蒙第一课”总共有多少章节？（写数字）

    3. 文档中心里，“Button组件”有一个属性叫“type”，它的取值有哪些？（写出两个）
    4. 在开发者学堂中，找到一个可以考“鸿蒙应用开发”认证的入口名称。
    5. 收藏夹里已经有几个Codelabs项目？（现场数一数）
