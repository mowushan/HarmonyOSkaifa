🧩 知识讲解 · 鸿蒙工程就像一栋大楼

1. 工程目录全览 —— 房间分配大揭秘

🏠 趣味类比： 鸿蒙项目就像一栋三层小楼——

📦 MyApplication （项目根目录）
  ├── 📂 entry   （主模块，相当于“客厅+卧室”）
  │    ├── 📂 src/main
  │    │    ├── 📂 ets   （代码区：ArkTS脚本，客厅里放的家具）
  │    │    │    ├── 📂 pages  （页面文件夹）
  │    │    │    └── Index.ets  （首页）
  │    │    ├── 📂 resources  （资源文件夹：图片、文本、颜色）
  │    │    └── module.json5  （模块配置文件，门牌信息）
  │    └── 📂 build-profile.json5  （构建配置）
  └── 📂 oh-package.json5  （项目依赖清单）

重点记忆： 我们主要待在 entry/src/main/ets/pages 写界面，在 resources/base/element/string.json 改文本，在 resources/base/media 放图片！

2. EntryAbility —— 应用的“大管家”

👔 每个鸿蒙应用都有一个或多个Ability。EntryAbility是应用启动时第一个运行的Ability，相当于大楼的“前台接待”。

重要生命周期方法：

  • onCreate()：应用创建时执行（只执行一次）→ 适合初始化数据。
  • onWindowStageCreate()：窗口创建时执行 → 设置主页面。
  • onForeground()：应用切到前台时触发。

🎯 我们目前只需要知道：EntryAbility 通过 windowStage.loadContent('pages/Index') 加载首页 Index.ets。

3. 资源管理 —— 文本、图片集中营

📝 string.json

{ "string": [ { "name": "app_name", "value": "鸿蒙集训营" }, { "name": "hello_text", "value": "Hello 鸿蒙！" } ] }

引用方式：$r('app.string.app_name')

🖼️ media 文件夹

放图片（png/jpg），引用方式：$r('app.media.icon')（文件名icon.png）

颜色同理：color.json 里定义，用 $r('app.color.primary')

✅ 好处： 修改文案不用进代码，更换图标秒完成，国际化也方便！

⚙️ 技能实训 · 今日动手探索

✅ 实训目标：打开已有工程，导航目录，动手改文字/图标，创建新页面

🔧 实训一：打开上次的 HelloWorld 工程，认识项目结构

  • 启动 DevEco Studio，打开上次创建的“MyFirstHarmony”项目（若没有，则新建一个Empty Ability项目）。
  • 观察左侧“Project”视图，切换到“Project”文件视角（不是Android那种）。找到 entry/src/main/ets 和 entry/src/main/resources。
  • 老师引导： 依次点击这些文件夹，让学生说出每个文件夹可能装什么。（提问互动：你觉得图片应该放哪里？文字放哪里？）
  • 完成探索清单： 在纸上画出工程目录树，同桌互相检查是否正确。

    • 💡 老师巡场技巧： 鼓励学生多点击，并强调“不用怕弄坏，代码都在Git里备份过”。

    🔧 实训二：修改应用名称和首页文本（资源引用初体验）

  • 修改应用名： 打开 entry/src/main/resources/base/element/string.json，找到 "app_name" 的 value，改成 "鸿蒙探索营-王小乐"（可加自己名字）。
  • 观察效果： 重新运行模拟器（点击绿色三角），应用标题栏名字变了！
  • 修改首页欢迎语： 在 string.json 中增加一条：
    { "name": "welcome_msg", "value": "我终于看懂了鸿蒙工程！" }
  • 打开 pages/Index.ets，将原来 Text('Hello World') 改为 Text($r('app.string.welcome_msg'))，运行查看效果。
  • 挑战： 试着在 resources 下新建一个 color.json，定义 title_color，并在 Text 上加上 .fontColor($r('app.color.title_color'))。

    • 🎨 通过这个实训，学生将理解“资源与代码分离”的思想，以后换皮肤、换语言再也不怕！

    🔧 实训三：替换应用图标 —— 让桌面与众不同

  • 准备一张 512x512 的png图片（或使用老师下发的鸿蒙logo，也可以简单用画图做一个图标）。
  • 将图片重命名为 app_icon.png，复制到 entry/src/main/resources/base/media 文件夹下。
  • 打开 entry/src/main/module.json5，找到 "icon": "$media:app_icon" 那一行（注意原来的图标名），确认新图标文件名一致，或修改为 "$media:app_icon"。
  • 重新运行应用（最好卸载旧应用再安装），查看手机桌面/模拟器桌面图标是否更换成功。

    • 📸 小组任务： 每组设计一个创意图标（可用AI生成或手绘后拍照），统一更新到项目中，互相展示“独家定制版”应用。

    🔧 实训四（核心挑战）：新建第二个页面并完成页面跳转

  • 在 ets/pages 目录下右键 → New → ArkTS File，命名为 AboutPage.ets。
  • 编写简单页面内容： 
    @Entry
@Component
struct AboutPage {
  build() {
    Column() {
      Text('关于本应用')
        .fontSize(30)
        .margin({ top: 50 })
      Text('鸿蒙备赛集训队')
        .fontSize(20)
        .margin(20)
      Button('返回首页')
        .onClick(() => {
          router.back();   // 返回上一页
        })
    }
    .width('100%')
    .height('100%')
  }
}
  • 在 Index.ets 中添加一个按钮，点击跳转到 AboutPage：
    需要导入 router: import router from '@ohos.router';
    按钮的 onClick: router.pushUrl({ url: 'pages/AboutPage' })
  • 运行调试： 点击按钮，看是否能跳转到关于页面，并能返回。

    • 🚀 通过创建新页面和跳转，学生将深刻理解“模块化管理”和“页面路由”的基本概念，为后期资产管理系统多页面开发打好基础。
