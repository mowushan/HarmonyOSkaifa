🧩 知识讲解 · 用类来描绘世界

1. 为什么需要数据模型？—— 把“杂物”变成“抽屉”

📦 生活类比： 假设你要整理一堆资产信息（名称、价格、购买日期、存放位置）。如果没有数据模型，这些信息就像散落一地的衣服；有了实体类，就像给每个资产配了一个“专用抽屉”，所有属性整整齐齐。

数据模型的好处：

  • 📁 结构化：相关数据绑定在一起，代码更清晰。
  • 🔄 可复用：定义一次，到处创建对象。
  • 🛡️ 安全性：通过封装控制数据访问权限。
  • 🔌 可扩展：未来增加新属性只需要改类定义。

2. ArkTS中的类（class）—— 对象的“生产图纸”

// 最简单的实体类定义
class Asset {
  // 1. 属性定义
  name: string = ''
  price: number = 0
  purchaseDate: string = ''
  
  // 2. 构造函数（方便创建对象时直接赋值）
  constructor(name: string, price: number, purchaseDate: string) {
    this.name = name
    this.price = price
    this.purchaseDate = purchaseDate
  }
  
  // 3. 方法（可选，后面再学）
  getDisplayInfo(): string {
    return `${this.name} - ¥${this.price}`
  }
}

// 创建对象实例
let laptop = new Asset('MacBook Pro', 12999, '2025-03-01')
console.log(laptop.name)        // 输出: MacBook Pro
console.log(laptop.getDisplayInfo())  // 输出: MacBook Pro - ¥12999

重点理解：

  • 类 = 设计图（定义有什么属性和方法）
  • 对象 = 根据设计图造出来的具体东西（如“我的这台电脑”）
  • this = 指向“当前对象自己”

3. 访问修饰符与封装 —— 给数据加上“保护罩”

🔒 public（公开）

默认修饰符，属性/方法可以在任何地方访问。

class Person {
  public name: string = ''
}

🔐 private（私有）

只能在类内部访问，外部无法直接修改。配合getter/setter使用。

class BankAccount {
  private balance: number = 0
  
  getBalance(): number {
    return this.balance
  }
  
  deposit(amount: number) {
    if (amount > 0) this.balance += amount
  }
}

🔔 实训重点： 资产管理系统中，价格不应该被随意修改成负数，所以我们要用private + setter来做保护！

    🔧 实训二：封装优化（加入private + getter/setter）（20分钟）

  • 将price属性改为private，防止外部直接赋值负数。
  • 添加getPrice()和setPrice()方法： 
    private _price: number = 0

getPrice(): number {
  return this._price
}

setPrice(value: number): void {
  if (value < 0) {
    console.error('价格不能为负数！')
    return
  }
  this._price = value
}
  • 将status改为private，添加getStatus()和setStatus()，限制只能设置“正常/维修/报废”三种状态。
  • 在构造函数中调用setter，确保创建对象时也会触发校验。

    • ⚠️ 常见错误： 忘记修改构造函数中的赋值方式，直接this._price = price是可以的，但建议统一使用setter。为了简洁，我们可以保留直接赋值，但外部必须用setter。

    🔧 实训三：添加辅助方法（15分钟）

getShortDesc(): string {
