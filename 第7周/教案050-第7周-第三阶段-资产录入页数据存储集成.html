🧩 知识讲解 · 数据持久化两种武器

1. Preferences（用户首选项）—— 轻量级“便签本”

📝 趣味类比： Preferences就像你的手机备忘录，适合保存“设置项”“用户偏好”“少量配置信息”。

特点： 键值对存储，读写快，适合保存几百KB以内的数据。例如：用户主题颜色、是否开启通知、上次登录用户名。

在我们的资产系统中： 可以用Preferences保存“默认资产分类”、“上次录入的时间”等轻量数据。

2. 关系型数据库（relationalStore）—— 强大的“Excel表格本”

🗄️ 趣味类比： 数据库就像Excel的多张表格，可以存放成百上千条资产记录，还能按条件查询、排序。

核心概念： 表(Table)、行(Row)、列(Column)。每一行是一条资产，每一列是资产属性（名称、价格、购买日期等）。

在我们资产系统中： 所有录入的资产都会存到数据库里，后续资产列表页就从数据库读取展示。

🔧 实训一：创建数据库管理类（15分钟）

  • 在项目中的 entry/src/main/ets/common/ 目录下新建文件 DatabaseHelper.ets。
  • 编写数据库初始化代码，创建资产表 ASSET_TABLE，包含 id, name, price, category, purchaseDate 字段。
  • 编写插入数据的方法 insertAsset()。
    • // DatabaseHelper.ets
import relationalStore from '@ohos.data.relationalStore';
import { BusinessError } from '@ohos.base';

const STORE_CONFIG: relationalStore.StoreConfig = {
  name: 'AssetDB.db', // 数据库文件名
  securityLevel: relationalStore.SecurityLevel.S1
};

const CREATE_TABLE_SQL = `
  CREATE TABLE IF NOT EXISTS ASSET_TABLE (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    price REAL,
    category TEXT,
    purchaseDate TEXT
  )
`;

export class DatabaseHelper {
  private rdbStore: relationalStore.RdbStore | null = null;

  // 初始化数据库
  async initDatabase(context: Context): Promise<void> {
    try {
      this.rdbStore = await relationalStore.getRdbStore(context, STORE_CONFIG);
      await this.rdbStore.executeSql(CREATE_TABLE_SQL);
      console.info('数据库初始化成功，表已创建');
    } catch (err) {
      let error = err as BusinessError;
      console.error(`数据库初始化失败: ${error.message}`);
    }
  }

  // 插入资产数据
  async insertAsset(asset: AssetData): Promise<number> {
    if (!this.rdbStore) {
      throw new Error('数据库未初始化');
    }
    const valueBucket: relationalStore.ValuesBucket = {
      'name': asset.name,
      'price': asset.price,
      'category': asset.category,
      'purchaseDate': asset.purchaseDate
    };
    return await this.rdbStore.insert('ASSET_TABLE', valueBucket);
  }
}

// 资产数据接口
export interface AssetData {
  name: string;
  price: number;
  category: string;
  purchaseDate: string;
}

    🔧 实训二：在录入页面中集成数据库（25分钟）

  • 打开资产录入页面的.ets文件（如 AssetAddPage.ets）。
  • 引入 DatabaseHelper，并获取全局数据库实例。
  • 给“保存”按钮绑定点击事件，在回调中收集表单数据。
  • 进行数据校验（非空、价格为正数等）。
  • 调用数据库插入方法，成功后弹出提示并跳转回列表页。
    • // AssetAddPage.ets 关键代码片段
import router from '@ohos.router';
import promptAction from '@ohos.promptAction';
import { DatabaseHelper, AssetData } from '../common/DatabaseHelper';
import { BusinessError } from '@ohos.base';

@Entry
@Component
struct AssetAddPage {
  @State assetName: string = '';
  @State assetPrice: string = '';
  @State category: string = '';
  @State purchaseDate: string = '';
  private dbHelper: DatabaseHelper = (AppStorage.get('dbHelper') as DatabaseHelper);

  // 保存资产
  async saveAsset() {
    // 1. 表单验证
    if (!this.assetName.trim()) {
      await this.showToast('请输入资产名称');
      return;
    }
    const priceNum = parseFloat(this.assetPrice);
    if (isNaN(priceNum) || priceNum <= 0) {
      await this.showToast('请输入有效的资产价格');
      return;
    }
    if (!this.category) {
      await this.showToast('请选择资产分类');
      return;
    }

    // 2. 组装数据
    const asset: AssetData = {
      name: this.assetName,
      price: priceNum,
      category: this.category,
      purchaseDate: this.purchaseDate || new Date().toISOString().split('T')[0]
    };

    try {
      // 3. 插入数据库
      const id = await this.dbHelper.insertAsset(asset);
      console.info(`插入成功，id=${id}`);
      await this.showToast('资产保存成功！');
      // 4. 清空表单或返回上一页
      router.back(); // 返回资产列表页
    } catch (err) {
      let error = err as BusinessError;
      console.error(`保存失败: ${error.message}`);
      await this.showToast('保存失败，请重试');
    }
  }

  async showToast(msg: string) {
    await promptAction.showToast({ message: msg, duration: 2000 });
  }

  build() {
    Column() {
      // ... 原有的表单UI组件（TextInput、Picker等）
      Button('保存资产')
        .onClick(() => this.saveAsset())
        .margin(20)
    }
    .padding(16)
  }
}

    • 🐛 常见问题排查：

    • ❌ 报错“数据库未初始化”：检查EntryAbility中是否调用了initDatabase，并通过AppStorage传递实例。
    • ❌ 插入无反应：在insertAsset前后加console.log，检查控制台输出。
    • ❌ 中文乱码：确保字符串编码正确，鸿蒙默认支持UTF-8。
