🧩 知识讲解 · Preferences轻量级存储

1. 为什么需要数据存储？—— 让应用“长记性”

🤔 思考场景： 你做了一个“夜间模式”开关，用户打开后把界面变暗。但是应用一关闭再打开，开关又回到关闭状态——用户会很抓狂！

解决方案： 把用户的设置“存起来”，下次启动时“读出来”。这就是数据持久化！

鸿蒙提供了三种存储方式：

  • 📝 Preferences（首选项）：轻量级，适合存配置、开关状态、用户信息等小数据（今天学这个！）
  • 🗄️ 关系型数据库：适合存结构化数据，如资产列表（下周学）
  • 📁 分布式数据库：适合跨设备同步数据（第四阶段学）

2. Preferences原理 —— 键值对小本本

🔑 键（Key）

数据的“名字”，必须是字符串，唯一标识。比如 "user_name"、"night_mode"。

📦 值（Value）

数据的“内容”，支持字符串、数字、布尔值等。比如 "张三"、true。

// 键值对示例 { "user_name": "张三", "night_mode": true, "font_size": 16, "last_login": "2026-04-12" }

📌 核心API：

// 1. 获取Preferences实例 let preferences = await dataPreferences.getPreferences(context, "my_settings"); // 2. 写入数据（同步或异步） await preferences.put("user_name", "张三"); await preferences.put("night_mode", true); // 3. 保存（必须调用！） await preferences.flush(); // 4. 读取数据 let userName = await preferences.get("user_name", "默认值"); let nightMode = await preferences.get("night_mode", false); // 5. 删除数据 await preferences.delete("user_name"); // 6. 清空所有数据 await preferences.clear();

⚠️ 重要提醒： put()只是把数据放到内存，必须调用flush()才能保存到磁盘文件！很多新手会忘记这一步，导致数据没存进去。

3. 应用场景 —— 我们的资产管理系统需要存什么？

👤 用户登录状态

记住“已登录”，下次打开应用直接进入主页。

🌙 夜间模式开关

用户偏好，保存true/false，启动时读取应用。

🔊 音效开关

操作音效是否开启。

📱 最后打开的资产分类

记住用户上次查看的是哪个分类，提升体验。

⚙️ 技能实训 · 手把手实现Preferences存储

✅ 本次实训目标：实现一个“用户设置页面”，能保存和读取夜间模式开关、用户名

📁 准备工作

1. 打开鸿蒙项目（之前创建的“AssetManager”或新建项目）。

2. 在 entry/src/main/ets/pages 目录下，新建一个页面：SettingsPage.ets。

3. 在 Index.ets 中添加跳转按钮，方便测试。

🧪 实训任务一：获取Preferences实例（10分钟）

  • 在SettingsPage.ets中导入Preferences模块：
    import dataPreferences from '@ohos.data.preferences';
  • 创建获取Preferences的函数（异步）：
    async getPreferences() { let context = getContext(this); let preferences = await dataPreferences.getPreferences(context, "asset_settings"); return preferences; }
  • 教师演示： 解释context是什么（应用上下文），"asset_settings"是存储文件名。

    • 🧪 实训任务二：写入数据 —— 保存用户设置（15分钟）

  • 在页面中添加状态变量：
    @State nightMode: boolean = false; @State userName: string = "";
  • 实现保存夜间模式的方法：
    async saveNightMode(isNight: boolean) { let preferences = await this.getPreferences(); await preferences.put("night_mode", isNight); await preferences.flush(); this.nightMode = isNight; }
  • 实现保存用户名的方法：
    async saveUserName(name: string) { let preferences = await this.getPreferences(); await preferences.put("user_name", name); await preferences.flush(); this.userName = name; }
  • 💡 老师强调： flush()一定不能少！可以类比为“点击保存按钮”。

    • 🧪 实训任务三：读取数据 —— 恢复用户设置（15分钟）

  • 实现读取数据的方法（在页面加载时调用）：
    async loadSettings() { let preferences = await this.getPreferences(); let savedNightMode = await preferences.get("night_mode", false); let savedUserName = await preferences.get("user_name", ""); this.nightMode = savedNightMode; this.userName = savedUserName; }
  • 在aboutToAppear()生命周期中调用：
    aboutToAppear() { this.loadSettings(); }
  • 📌 注意： get()的第二个参数是默认值，如果key不存在则返回默认值，避免崩溃。

    • 🧪 实训任务四：构建完整的设置页面UI（20分钟）

  • 编写UI，包含开关和输入框：
    build() { Column() { // 夜间模式开关 Row() { Text("夜间模式") .fontSize(16) .width('70%') Toggle({ type: ToggleType.Switch, isOn: this.nightMode }) .onChange((isOn: boolean) => { this.saveNightMode(isOn); }) } .width('100%') .padding(12) .justifyContent(FlexAlign.SpaceBetween) // 用户名输入 Row() { Text("用户名") .fontSize(16) .width('30%') TextInput({ placeholder: "请输入用户名", text: this.userName }) .width('65%') .onChange((value: string) => { this.saveUserName(value); }) } .width('100%') .padding(12) // 显示当前状态 Text(`当前夜间模式：${this.nightMode ? "开启" : "关闭"}`) .margin({ top: 20 }) Text(`当前用户名：${this.userName || "未设置"}`) } .padding(20) }
  • 挑战： 添加一个“清除所有设置”按钮，调用clear()方法。

    • 🧪 实训任务五：验证数据持久化（10分钟）

  • 在模拟器或真机上运行应用。
  • 打开设置页面，打开夜间模式开关，输入用户名。
  • 完全关闭应用（从最近任务中划掉）。
  • 重新打开应用，进入设置页面，观察开关和用户名是否恢复！
  • 成功标志： 关闭前开启的夜间模式，重新打开后依然是开启状态！
