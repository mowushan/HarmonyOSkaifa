🧩 知识讲解 · 从静态页面到动态交互

1. 为什么需要状态管理？—— 静态UI vs 动态UI

🔄 生活类比： 静态UI像一张打印的照片，永远不变。动态UI像手机屏幕，你点一下，数字增加，文字变化，照片切换——这就是“状态”在起作用！

传统命令式编程： 你手动告诉UI“把这个文本改成新值”。
鸿蒙声明式编程： 你只需要说“这个变量是状态，它变了UI自动变”。
✨ @State就是实现这种魔法的装饰器！

2. @State 语法与工作原理

@Component
struct Counter {
    // @State 装饰的变量，一旦改变，build方法会重新执行
    @State count: number = 0

    build() {
        Column() {
            Text(`点击次数: ${this.count}`)
                .fontSize(24)
                .margin(20)
            Button('点我 +1')
                .onClick(() => {
                    // 修改状态，UI自动刷新
                    this.count++
                })
        }
        .width('100%')
        .height('100%')
        .justifyContent(FlexAlign.Center)
    }
}

📌 关键规则：

  • @State只能用在struct（组件）内部的成员变量。
  • 状态变量变化必须在UI线程（比如按钮的onClick回调里）。
  • UI中直接使用 this.变量名 即可自动订阅变化。

3. 状态提升与组件通信（预热）

👪 父子组件

父组件用@State，子组件用@Prop接收（单向），或@Link双向同步。

📦 简单案例

父组件定义资产列表，子组件展示每一项，点击“借用”按钮时父组件的状态更新，列表重新渲染。

🧠 今天我们先专注单个组件内的@State，下节课再深入学习@Prop和@Link。但思维可以先建立：状态放在最需要共享的父组件中。

    // 计数器完整代码（包含加/减）
@Component
struct Counter {
    @State count: number = 0

    build() {
        Column({ space: 20 }) {
            Text(`当前计数: ${this.count}`)
                .fontSize(32)
                .fontWeight(FontWeight.Bold)

            Row({ space: 30 }) {
                Button('➕ 增加')
                    .onClick(() => { this.count++ })
                    .backgroundColor('#0078D4')
                Button('➖ 减少')
                    .onClick(() => {
                        if (this.count > 0) this.count--
                    })
                    .backgroundColor('#E64340')
            }
        }
        .width('100%')
        .height('100%')
        .justifyContent(FlexAlign.Center)
    }
}

    🔧 实训二：资产状态切换小模块（25分钟）

  • 在同一个项目里新建一个组件文件 AssetStatus.ets，定义一个卡片组件，展示资产名称和“状态”按钮。
  • 组件内部用@State isBorrowed: boolean = false 表示是否已借出。
  • UI展示： 如果isBorrowed为true，显示红色文字“已借出”，按钮为“归还”；否则显示绿色“可借用”，按钮为“借用”。
  • 点击按钮切换状态，文字和按钮样式都要跟着变（条件渲染可以用if或者三元运算符）。
  • 在主页面引入该组件，并传props（资产名称），做3个不同的资产卡片测试。
    • // AssetStatus.ets
@Component
export struct AssetStatus {
    @State isBorrowed: boolean = false
    private assetName: string = ''

    aboutToAppear() {
        // 假设从父组件传进来，这里用参数接收
    }

    build() {
        Row() {
            Text(this.assetName)
                .fontSize(18)
                .layoutWeight(1)
            Button(this.isBorrowed ? '归还' : '借用')
                .onClick(() => {
                    this.isBorrowed = !this.isBorrowed
                })
                .backgroundColor(this.isBorrowed ? '#E64340' : '#28A745')
            Text(this.isBorrowed ? '已借出' : '可借用')
                .fontColor(this.isBorrowed ? '#E64340' : '#28A745')
                .margin({ left: 12 })
        }
        .width('100%')
        .padding(12)
        .backgroundColor('#F5F5F5')
        .borderRadius(16)
        .margin({ bottom: 10 })
    }
}

    🔧 注意：组件内部状态变化只会刷新该组件自身，不会影响其他组件，这正是@State的封装好处！
