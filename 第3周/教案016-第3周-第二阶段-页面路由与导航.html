🧩 知识讲解 · 路由的魔法世界

1. 什么是页面路由？—— App里的“导航员”

🗺️ 生活类比： 你去商场，想知道“电影院怎么走”，需要看指示牌或问保安——这就是“路由”的作用。在鸿蒙应用里，router模块就是那个指路人。

两个核心概念：

  • 📄 页面（Page）：每个.ets文件就是一个页面，比如资产列表页、资产详情页。
  • 🔁 路由（Router）：从一个页面跳转到另一个页面的机制。

🎯 今天的终极目标： 点击列表中的某一项，跳转到详情页，并且能携带资产名称、数量等信息。

2. 鸿蒙路由核心API —— 三招搞定跳转

📌 pushUrl()

把新页面“压”到页面栈顶部，用户可点击返回按钮回到上一页。
场景： 列表→详情（用户想返回列表）

📌 replaceUrl()

替换当前页面，无法返回上一页。
场景： 启动页→首页（不希望回启动页）

📌 back()

返回上一页，可携带数据回传。
场景： 编辑资产后返回列表并刷新

📦 参数传递： 跳转时可以在params对象里放任何数据，目标页面用router.getParams()接收。

3. 代码结构预览（别怕，等下手把手写）

// 在列表页跳转
import router from '@ohos.router';

Button('查看详情')
  .onClick(() => {
    router.pushUrl({
      url: 'pages/AssetDetailPage',
      params: { id: 1, name: '笔记本电脑', quantity: 5 }
    });
  })

// 在详情页接收参数
import router from '@ohos.router';

@Entry
@Component
struct AssetDetailPage {
  @State assetName: string = '';
  @State quantity: number = 0;

  aboutToAppear() {
    const params = router.getParams();
    if (params) {
      this.assetName = params['name'];
      this.quantity = params['quantity'];
    }
  }
  // ... 页面内容
}

⚙️ 技能实训 · 手把手实现资产详情跳转

✅ 实训目标：从资产列表页点击任意资产 → 跳转到详情页并显示信息

🔧 实训准备：创建详情页

  • 在项目中的pages目录下，新建一个ArkTS文件，命名为 AssetDetailPage.ets。
  • 快速生成页面模板： 输入“@Entry”会自动提示，选择完整组件模板。
  • 添加基础UI： 一个Column容器，里面放两个Text组件，分别显示“资产名称：”和“数量：”，用@State变量绑定。
    • // AssetDetailPage.ets
import router from '@ohos.router';

@Entry
@Component
struct AssetDetailPage {
  @State assetName: string = '加载中...';
  @State assetQuantity: number = 0;

  aboutToAppear() {
    // 获取路由跳转时传递的参数
    const params = router.getParams();
    if (params && params['name']) {
      this.assetName = params['name'];
      this.assetQuantity = params['quantity'];
    }
  }

  build() {
    Column({ space: 20 }) {
      Text(`资产名称：${this.assetName}`)
        .fontSize(20)
        .fontWeight(FontWeight.Bold)
      Text(`数量：${this.assetQuantity}`)
        .fontSize(18)
      Button('返回列表')
        .onClick(() => {
          router.back();
        })
        .margin({ top: 30 })
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
    .padding(20)
  }
}

    🔧 实训二：在资产列表页添加跳转逻辑

  • 打开之前的 AssetListPage.ets（资产列表页），先导入router模块：import router from '@ohos.router';
  • 找到渲染列表项的ForEach，为每个列表项添加点击事件（.onClick）。
  • 在点击事件中调用router.pushUrl，携带当前资产的name和quantity。
    • // AssetListPage.ets 关键代码片段
import router from '@ohos.router';

// 假设资产数据
@State assetList: Array = [
  { id: 1, name: '笔记本电脑', quantity: 5 },
  { id: 2, name: '投影仪', quantity: 2 },
  { id: 3, name: '实训台', quantity: 10 }
];

build() {
  List() {
    ForEach(this.assetList, (item: AssetItem) => {
      ListItem() {
        Text(`${item.name} (数量: ${item.quantity})`)
          .fontSize(18)
          .padding(12)
      }
      .onClick(() => {
        router.pushUrl({
          url: 'pages/AssetDetailPage',
          params: {
            name: item.name,
            quantity: item.quantity
          }
        });
      })
    })
  }
}

    🔧 实训三：运行并测试跳转

  • 点击DevEco Studio右上角的“运行”按钮，在模拟器或真机上启动应用。
  • 点击列表中的任意一项，观察是否跳转到详情页，并且显示正确的资产名称和数量。
  • 点击详情页的“返回列表”按钮，验证能否回到列表页。

    • 💡 常见错误排雷：

    • ❌ 页面找不到？检查url路径是否正确，必须是“pages/文件名”（不带.ets）。
    • ❌ 参数获取不到？检查params对象里的key是否和接收时一致。
    • ❌ 返回没反应？检查是否调用了router.back()，且没有写错方法名。

    🔧 实训四（挑战）：带数据返回 + 刷新列表

    ⭐ 进阶挑战： 在详情页增加一个“增加数量”按钮，点击后数量+1，返回列表页时列表页的数字同步更新。

    实现思路：
