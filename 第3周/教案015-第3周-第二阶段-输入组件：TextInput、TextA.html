🧩 知识讲解 · 让应用听懂用户的话

1. TextInput · 单行输入小能手

⌨️ 场景导入： 想象你正在开发一个“资产录入”页面，需要用户填写资产名称、型号、购买价格。这些信息都是一行就能写完的，用TextInput最合适！

常用属性（记重点就行）：

  • placeholder：占位提示文字（灰色显示，比如“请输入资产名称”）
  • type：输入类型（普通文本、数字、密码等）
  • maxLength：最大字符数限制
  • onChange：内容变化时触发的事件
// 最简单的TextInput示例 TextInput({ placeholder: '请输入资产名称' }) .onChange((value: string) => { console.log('用户输入了：', value); })

2. TextArea · 多行输入大肚量

📝 什么时候用？ 当用户需要写备注、详细描述、留言时，TextArea会提供多行输入区域，还能自动换行。

常用属性：

  • placeholder：占位提示
  • maxLines：最多显示几行
  • onChange：内容变化回调
TextArea({ placeholder: '请输入备注信息（最多100字）' }) .maxLines(5) .onChange((value: string) => { console.log('备注内容：', value); })

💬 小技巧：TextArea默认会自动增高，设置maxLines可以限制高度。

3. @State + 输入框 = 实时反馈

🔄 核心理解： 在鸿蒙ArkUI中，我们用@State装饰器标记变量。当这个变量变化时，用到它的UI组件会自动重新渲染！

联动流程： 用户在TextInput输入 → onChange触发 → 更新@State变量 → 界面其他位置（比如Text组件）立刻显示新内容。

@State assetName: string = '' // 在build()中 TextInput({ placeholder: '资产名称' }) .onChange((value) => { this.assetName = value }) Text('当前输入：' + this.assetName) // 会实时更新

3. 或者在原有页面中直接添加以下代码，本节课推荐新建页面，保持专注。

🔧 实训任务一：搭建基础框架 + 单行TextInput（15分钟）

  • 创建页面结构： 使用Column容器垂直排列组件，设置padding和间距。
  • 定义状态变量： @State assetName: string = ''，用于存储资产名称。
  • 添加TextInput组件： 设置placeholder为“资产名称”，绑定onChange更新assetName。
  • 添加Text组件显示实时内容： 在下方显示“您输入的资产名称：xxx”。
  • 运行到模拟器/真机，尝试输入文字，观察下方文字实时变化。
    • @Entry @Component struct InputDemo { @State assetName: string = '' build() { Column({ space: 20 }) { Text('📦 资产信息收集器') .fontSize(24) .fontWeight(FontWeight.Bold) TextInput({ placeholder: '资产名称' }) .width('90%') .onChange((value: string) => { this.assetName = value }) Text(`✨ 当前资产名称：${this.assetName || '未输入'}`) .fontSize(16) .fontColor('#0078d4') } .padding(20) .width('100%') .height('100%') } }

    ✅ 小挑战： 给TextInput加上边框圆角、背景色，让它更好看！参考代码：.backgroundColor('#f5f5f5').borderRadius(12)

    🔧 实训任务二：添加数字价格输入（限制输入数字）

  • 新增状态变量： @State price: string = ''（注意价格用字符串暂存，后续可转数字）
  • 添加第二个TextInput： 设置placeholder为“购买价格”，.type(InputType.Number) 让键盘弹出数字键盘。
  • 绑定onChange更新price。
  • 添加显示区域： 显示“价格：xxx 元”。

    • 🔢 InputType.Number可以让手机弹出数字键盘，用户体验更好！记得加.type(InputType.Number)

    🔧 实训任务三：加入TextArea多行备注（20分钟）

  • 新增状态变量： @State remark: string = ''
  • 添加TextArea组件： placeholder为“备注信息（最多80字）”，设置.maxLength(80)和.maxLines(4)。
  • 绑定onChange更新remark。
  • 增加实时显示备注区域： 显示“备注内容：” + remark，并显示字数统计（当前字数/80）。
    • TextArea({ placeholder: '备注信息（最多80字）' }) .width('90%') .maxLength(80) .maxLines(4) .onChange((value: string) => { this.remark = value }) Text(`📝 备注：${this.remark || '无'}`) Text(`字数：${this.remark.length}/80`)

    💡 老师指导：让学生观察字数统计如何随输入变化，加深对@State驱动UI的理解。

    🔧 实训任务四：综合展示卡片 + 提交按钮（15分钟）

  • 在页面底部增加“预览卡片”区域： 用白色卡片展示所有已输入的信息（资产名称、价格、备注）。
  • 添加“提交”按钮： 点击时弹出一个提示框，显示“已收集信息：xxx”。
  • 扩展挑战： 增加清空按钮，一键清空所有输入框（将三个状态变量设为空字符串）。
    • // 预览卡片 Column() { Text('📋 资产信息预览').fontSize(18).fontWeight(FontWeight.Bold) Divider().margin(10) Text(`资产名称：${this.assetName || '未填写'}`) Text(`购买价格：${this.price || '未填写'} 元`) Text(`备注：${this.remark || '未填写'}`) } .width('90%') .backgroundColor('#f9f9f9') .borderRadius(16) .padding(15) // 提交按钮 Button('提交信息') .onClick(() => { AlertDialog.show({ message: `已收集：\n资产：${this.assetName}\n价格：${this.price}\n备注：${this.remark}`, confirm: { value: '确定', action: () => {} } }) }) // 清空按钮 Button('清空所有') .backgroundColor('#f4a261') .onClick(() => { this.assetName = '' this.price = '' this.remark = '' })
