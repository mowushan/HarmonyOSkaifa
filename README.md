📋 每次课作业包含内容
表格
项目	要求
源码文件	保存本次课编写的所有.ets 文件，代码要有必要注释，命名清晰（如 asset_input_page.ets ）

✅ 每周提交规范
每周五前完成当周作业 — 将当周文件夹（如 “第 3 周”）打包为 zip 压缩包。
命名格式 — 姓名_第X周实训作业.zip（例：王嘉鑫_第 4 周实训作业.zip）
提交方式 — 发给各组组长汇总，组长统一交给林泳免老师。
检查标准 — 代码能正常运行、截图清晰展示实训要求、文件夹结构规范。

⚠️ 重要提醒
代码要能运行 — 提交前务必在 DevEco Studio 中测试，确保无语法错误。
截图要清晰 — 使用系统截图工具，展示关键界面和交互效果。
文件夹命名规范 — 严格按照 “第 X 次课 - 教案标题” 格式，方便老师快速定位。
按时提交 — 每周五截止，逾期将影响平时成绩和备赛进度。
❓ 常见问题
Q：如果某次课没有编写代码，还需要保存什么？
A：如第一课主要为理论 + 账号注册，只需保存截图（如注册成功页面、官网收藏截图）即可。
Q：截图需要放在 Word 里吗？
A：不需要，直接放图片在 “截图” 文件夹中，文件名说明内容即可。
Q：源码可以多个文件放在一起吗？
A：建议按项目模块分开存放，如果某次课只修改了单个文件，保留该文件即可。
Q：作业提交后老师会反馈吗？
A：林老师会抽查部分作业，并在下次课集中点评共性问题。
